Bloque 1 · Variables

¿Qué es una Variable?

🗃️ Definición

Espacio en la memoria que almacena un valor. Se identifica con un nombre único y puede cambiar durante la ejecución del programa.

Pensalo como una caja etiquetada donde guardás un dato.

💡 Permiten

  • Guardar y reutilizar información
  • Realizar operaciones con datos
  • Interactuar con el usuario
  • Hacer el código más flexible y dinámico
let nombreUsuario = "Sofía"; // variable de texto
let edad = 25; // variable numérica
const PI = 3.1416; // constante
Bloque 1 · Variables

Reglas y Convenciones de Nombres

✅ Reglas (obligatorias)

  • Deben empezar con letra, $ o _
  • No pueden empezar con un número
  • Sin espacios ni caracteres especiales
  • No pueden ser palabras reservadas (let, if...)

🎨 Convenciones (buenas prácticas)

// camelCase → variables y funciones
let nombreUsuario = "Sofía";
// PascalCase → clases
class UsuarioRegistrado { }
// UPPER_CASE → constantes globales
const MAX_INTENTOS = 3;
Bloque 1 · Declaración

let · const · var

let

Variable que puede cambiar de valor. Alcance de bloque. La opción principal para variables mutables.

let contador = 0;
contador = 1; // ✅ OK

const

No puede reasignarse. Alcance de bloque. Usarla por defecto cuando el valor no cambia.

const PI = 3.1416;
PI = 3; // ❌ Error!

var

Forma antigua. Alcance de función, no de bloque. No recomendada en código moderno.

var nombre = "Ana";
// ⚠️ Evitar usar var
Regla de oro: usá const por defecto. Solo usá let cuando necesitás reasignar el valor.
Bloque 1 · Tipos de datos

Tipos de Datos Simples (Primitivos)

string

Texto

Secuencia de caracteres entre comillas.

let msj = "Hola";
let nom = 'Sofía';
let txt = `Mundo`;
number

Número

Enteros y decimales. Un solo tipo para ambos.

let edad = 25;
let precio = 99.99;
let inf = Infinity;
boolean

Lógico

Solo dos valores posibles.

let activo = true;
let verificado = false;
// Solo true o false
// typeof → conocer el tipo de una variable
typeof "Hola" // "string"
typeof 42 // "number"
typeof true // "boolean"
Bloque 1 · Tipos complejos

Tipos de Datos Complejos

Array

Colección ordenada

const frutas = ["manzana", "pera"];
const nums = [1, 2, 3];
console.log(frutas[0]); // "manzana"
Object

Par clave-valor

const usuario = {
  nombre: "Sofía",
  edad: 25
};
console.log(usuario.nombre);
Los tipos complejos pueden modificar su contenido aunque sean const. const impide reasignar la referencia, no modificar el objeto.
Bloque 1 · Scope

Scope (Ámbito de Variables)

🌍 Global

Fuera de funciones. Accesible desde cualquier parte del código.

let nombre = "Ana";

function saludar() {
  console.log(nombre); // ✅
}

🏠 Local (función)

Dentro de una función. Solo existe ahí.

function fn() {
  let msg = "Hola";
  console.log(msg); // ✅
}
console.log(msg); // ❌

🔲 Bloque (let/const)

Dentro de {}. Solo existe dentro de ese bloque.

if (true) {
  let x = 10;
  console.log(x); // ✅
}
console.log(x); // ❌
Usá let y const en lugar de var para aprovechar el scope de bloque y evitar errores.
Bloque 1 · Operadores

Operadores Aritméticos

let a = 10, b = 3;

console.log(a + b); // 13 → suma
console.log(a - b); // 7 → resta
console.log(a * b); // 30 → multiplicación
console.log(a / b); // 3.33 → división
console.log(a % b); // 1 → módulo (resto)
console.log(a ** b); // 1000 → exponente (10³)

// Incremento y decremento
let x = 5;
console.log(x++); // 5 → postfijo: devuelve LUEGO incrementa
console.log(++x); // 7 → prefijo: incrementa LUEGO devuelve
El % (módulo) es muy útil: sirve para saber si un número es par (n % 2 === 0).
Bloque 1 · Precedencia

Precedencia de Operadores

⚠️ Sin paréntesis

let r = 5 + 3 * 2;
// Primero * → 5 + 6 = 11
console.log(r); // 11

✅ Con paréntesis

let r = (5 + 3) * 2;
// Primero () → 8 * 2 = 16
console.log(r); // 16
( )

1º Paréntesis

Mayor prioridad

**

2º Exponente

* /

%

4º Módulo

+ -

5º Suma / Resta

Menor prioridad

Bloque 1 · Comparación

Operadores de Comparación

5 == "5" // true → igual valor (sin verificar tipo) ⚠️
5 === "5" // false → igual valor Y tipo ✅ usar siempre
10 != 5 // true → diferente valor
10 !== "10" // true → diferente valor O tipo
7 > 5 // true → mayor que
3 < 8 // true → menor que
6 >= 6 // true → mayor o igual que
4 <= 5 // true → menor o igual que
Siempre usá === en lugar de ==. El triple igual evita conversiones de tipo implícitas que generan bugs difíciles de detectar.
Bloque 2 · Strings

Strings y Concatenación

+ Concatenación clásica

let nom = "Sofía";
let edad = 25;
let msj = "Hola, " + nom + ". Tenés " + edad + " años.";
// Hola, Sofía. Tenés 25 años.

✨ Template literals (ES6+)

let nom = "Sofía";
let edad = 25;
let msj = `Hola, ${nom}. Tenés ${edad} años.`;
// Hola, Sofía. Tenés 25 años. ✅ Mejor
// Caracteres especiales dentro de strings
let frase = "Ella dijo: \"Hola\""; // comillas escapadas
let otro = 'It\'s a test'; // apostrofe escapado

Bloque 3 · Condicionales

¿Qué son las Condicionales?

🎯 Definición

Instrucciones que ejecutan diferentes bloques de código dependiendo de si una condición es verdadera o falsa.

Son fundamentales para controlar el flujo del programa y tomar decisiones en el código.

🌧️ Ejemplo de la vida real

"Si está lloviendo → llevar paraguas.
Si no → salir sin paraguas."

Cada día tomamos decisiones basadas en condiciones. Las condicionales llevan esa lógica al código.

Permiten controlar el flujo del programa · Son esenciales en validaciones y lógica de negocio.
Bloque 3 · if...else

Sentencia if...else

// Estructura básica
if (condición) {
  // se ejecuta si la condición es true
} else {
  // se ejecuta si la condición es false
}

// Ejemplo: saludo según la hora
const hora = 14;

if (hora < 12) {
  console.log("Buenos días");
} else if (hora < 18) {
  console.log("Buenas tardes"); // ← se ejecuta
} else {
  console.log("Buenas noches");
}
Bloque 3 · switch

Sentencia switch

const dia = 3;

switch (dia) {
  case 1:
    console.log("Lunes"); break;
  case 2:
    console.log("Martes"); break;
  case 3:
    console.log("Miércoles"); break; // ← se ejecuta
  default:
    console.log("Otro día");
}
El break es obligatorio — sin él el código sigue ejecutando los siguientes casos (fall-through).
Bloque 3 · Ternario

Operador Ternario

condición ? valor_si_true : valor_si_false
// if/else tradicional:
let acceso;
if (edad >= 18) { acceso = "Permitido"; } else { acceso = "Denegado"; }

// Ternario — mismo resultado en una línea:
const acceso = edad >= 18 ? "Permitido" : "Denegado";

// Ternario encadenado (con moderación):
const saludo = hora < 12 ? "Mañana"
              : hora < 18 ? "Tarde"
              : "Noche";

Bloque 4 · Operadores lógicos

Operadores Lógicos

&& AND

Ambas condiciones deben ser verdaderas.

true && true // true
true && false // false
false && true // false

|| OR

Al menos una condición debe ser verdadera.

true || false // true
false || false // false
false || true // true

! NOT

Invierte el valor booleano.

!true // false
!false // true
!(5 > 3) // false
// Ejemplo combinado: control de acceso
const edad = 20; const tieneLicencia = true;
if (edad >= 18 && tieneLicencia) {
  console.log("Podés conducir");
}
Bloque 4 · Buenas prácticas

Condiciones de Borde

⚠️ ¿Qué son?

Situaciones extremas o inusuales que pueden hacer que un programa falle si no se manejan correctamente.

Ejemplo: un ascensor de 500 kg. ¿Qué pasa si alguien sube con exactamente 500? ¿Con 501? ¿Con 0?

✅ Buenas prácticas

  • Definir valores máximos y mínimos
  • Validar antes de calcular
  • Considerar entradas vacías, null, undefined
  • Probar el código con casos extremos
// Condición de borde: división por cero
const divisor = 0;
if (divisor === 0) {
  console.error("Error: no se puede dividir por cero.");
} else {
  console.log(10 / divisor);
}
Bloque 4 · Diagramas de flujo

Diagramas de Flujo

📊 ¿Qué son?

Representación gráfica de los pasos para resolver un problema. Permiten planificar el código antes de escribirlo.

🔷 Símbolos principales

  • Óvalos → Inicio y Fin
  • Rectángulos → Procesos / acciones
  • Rombos → Decisiones (sí/no)
  • Flechas → Flujo de ejecución
  • Círculos → Conectores
Antes de escribir código → diseñar el diagrama de flujo. Ahorrás tiempo y evitás errores lógicos.
Desafío · Predecir salida

⚡ Desafío 1: ¿Qué imprime?

const usuario = "admin";
const contrasena = "1234";

if (usuario === "admin" && contrasena === "1234") {
  console.log("Acceso concedido.");
} else if (usuario === "admin") {
  console.log("Contraseña incorrecta.");
} else {
  console.log("Acceso denegado.");
}
Opciones: 1️⃣ "Acceso concedido."  ·  2️⃣ "Contraseña incorrecta."  ·  3️⃣ "Acceso denegado."
Desafío · Predecir salida

⚡ Desafío 2: ¿Qué imprime?

const temperatura = 35;

if (temperatura < 0) {
  console.log("Hace mucho frío.");
} else if (temperatura < 15) {
  console.log("Hace frío.");
} else if (temperatura < 28) {
  console.log("Clima agradable.");
} else {
  console.log("Hace mucho calor.");
}
💡 ¿Qué pasaría si temperatura = 20?  ·  Opciones: 1️⃣ Mucho frío   2️⃣ Frío   3️⃣ Agradable   4️⃣ Mucho calor
