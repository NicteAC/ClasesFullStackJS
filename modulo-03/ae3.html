¿Por qué usar SASS?

❌ CSS Tradicional

  • Repetición constante de valores
  • Difícil de mantener en proyectos grandes
  • Sin lógica ni cálculos
  • Archivos monolíticos

✅ Con SASS

  • Variables centralizadas
  • Código modular y organizado
  • Funciones y operaciones
  • Parciales importables

Instalación de SASS

# Instalar SASS globalmente
npm install -g sass

# Verificar instalación
sass --version

# Compilar un archivo
sass input.scss output.css

# Modo watch (compilación automática)
sass --watch input.scss output.css
💡 Recomendación: Instala la extensión Live Sass Compiler en VS Code para compilación automática

Mixins: Código Reutilizable

// Definir mixin
@mixin flex-center {
    display: flex;
    justify-content: center;
    align-items: center;
}

// Mixin con parámetros
@mixin button-variant($bg-color, $text-color: white) {
    background: $bg-color;
    color: $text-color;
    &:hover { background: darken($bg-color, 10%); }
}

// Usar mixins
.container { @include flex-center; }
.btn-primary { @include button-variant($primary); }
.btn-danger { @include button-variant(#dc3545); }

Mixin para Media Queries

@mixin respond-to($breakpoint) {
    @if $breakpoint == mobile {
        @media (max-width: 480px) { @content; }
    } @else if $breakpoint == tablet {
        @media (max-width: 768px) { @content; }
    } @else if $breakpoint == desktop {
        @media (max-width: 1200px) { @content; }
    }
}

.header {
    padding: 2rem;
    @include respond-to(mobile) { padding: 1rem; }
    @include respond-to(tablet) { padding: 1.5rem; }
}

Funciones Personalizadas

Crear funciones

@function rem($pixels) {
    @return $pixels / 16 * 1rem;
}

@function col-width($cols, $total: 12) {
    @return percentage($cols / $total);
}

    font-size: rem(24);
}

.col-4 {
    width: col-width(4);
}

Sistema de Grillas con SASS

@use 'sass:math';

// Variables
$grid-columns: 12;
$gutter: 1.5rem;

// Función para calcular ancho
@function col-width($cols) {
    @return percentage(math.div($cols, $grid-columns));
}

// Generar clases con bucle
@for $i from 1 through $grid-columns {
    .col-#{$i} {
        width: col-width($i);
        padding: 0 math.div($gutter, 2);
    }
}

.row {
    display: flex;
    flex-wrap: wrap;
    margin: 0 -(math.div($gutter, 2));
}

Sistema de Design Tokens

// _tokens.scss - Mapas de tokens
$colors: (
    'primary': ('base': #2D5BFF, 'light': #5C82FF, 'dark': #1A3DB8),
    'secondary': ('base': #00D4AA, 'light': #33DFBB, 'dark': #00A385)
);

$spacing: ('xs': 0.25rem, 'sm': 0.5rem, 'md': 1rem, 'lg': 1.5rem, 'xl': 2rem);

// Función de acceso
@function color($name, $shade: 'base') {
    @return map-get(map-get($colors, $name), $shade);
}

@function space($size) {
    @return map-get($spacing, $size);
}

// Uso
.btn { background: color('primary'); padding: space('sm') space('md'); }

SASS Moderno: @use y @forward

Forma antigua (@import)

// Problemas:
// - Variables globales
// - Colisiones de nombres
// - Importaciones duplicadas

@import 'variables';
@import 'mixins';

Forma moderna (@use)

// Beneficios:
// - Namespace automático
// - Sin colisiones
// - Importación única

@use 'variables' as v;
@use 'mixins' as m;

.btn {
    color: v.$primary;
    @include m.flex-center;
}

Depuración en SASS

@debug

Imprime en consola durante compilación

@debug "Valor: #{$var}";

@warn

Muestra advertencia sin detener

@warn "Deprecado!";

@error

Detiene compilación con error

@error "Valor inválido";
