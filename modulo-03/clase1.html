Fundamentos · JavaScript

¿Qué es JavaScript?

⚡

Ligero y dinámico

Lenguaje interpretado que se ejecuta directamente en el navegador, sin necesidad de compilar.

🌐

Universal

Compatible con todos los navegadores modernos. También corre en servidores con Node.js.

🎯

El trío web

HTML estructura · CSS diseño · JS interactividad

JS permite modificar el contenido de una página en tiempo real, sin recargar.

Fundamentos · Entorno

Cliente vs Servidor

🖥️

Lado del Cliente

El código JS corre en el navegador del usuario. Interactúa con el DOM y la interfaz.

Ejemplo: cambiar el color de un botón al hacer clic, mostrar/ocultar un menú.

⚙️

Lado del Servidor

Con Node.js, JS corre en el servidor. Manipula datos e interactúa con bases de datos.

Ejemplo: responder una solicitud HTTP con datos en formato JSON.

En este curso aprenden los dos lados: front-end con JS en el navegador y back-end con Node.js.
Fundamentos · Motor

¿Qué es un Motor de JavaScript?

⚙️ ¿Qué hace?

Componente del navegador que interpreta y ejecuta el código JavaScript, transformándolo en instrucciones que la máquina puede entender.

V8 → Chrome · Edge · Node.js
SpiderMonkey → Firefox
JavaScriptCore → Safari

🔄 Fases de ejecución

  • Parseo: convierte el código en un Árbol de Sintaxis Abstracta (AST)
  • Compilación JIT: optimiza y compila en tiempo de ejecución
  • Ejecución: el código corre e interactúa con el navegador y el DOM
Fundamentos · Límites

JavaScript en el Navegador

✅ Puede

  • Ejecutarse en tiempo real y mostrar resultados instantáneos
  • Interactuar con las APIs del navegador
  • Mostrar alertas, registrar en consola
  • Acceder a información básica del dispositivo y navegador
  • Modificar el DOM de la página

🚫 No puede

  • Acceder directamente al sistema de archivos del usuario
  • Ejecutar comandos del sistema operativo
  • Acceder a recursos de otros dominios sin permiso (CORS)
  • Realizar acciones maliciosas sin autorización explícita

Consola

La Consola de JavaScript

🖨️ Métodos principales

console.log()Mensajes informativos
console.warn()Advertencias
console.error()Errores críticos
console.table()Arrays/objetos como tabla

🐛 Depuración

  • console.group() agrupa mensajes relacionados
  • console.trace() muestra la pila de llamadas
  • debugger; pausa la ejecución en ese punto
  • clear(); limpia la consola
Consola · Código

Métodos de Consola en Acción

// Mensajes básicos
console.log("Mensaje informativo");
console.warn("⚠️ Advertencia: revisar este valor");
console.error("❌ Error crítico detectado");

// Tabla para visualizar datos
const usuarios = [
  { nombre: "Ana", rol: "Dev" },
  { nombre: "Luis", rol: "QA" }
];
console.table(usuarios);

// Agrupar mensajes relacionados
console.group("Inicio del proceso");
console.log("Paso 1 completado");
console.log("Paso 2 completado");
console.groupEnd();

// Pausar ejecución para depurar
debugger; // detiene aquí si DevTools está abierto
console.log("Después del debugger");

Entorno de Trabajo · Integración

Conectar JavaScript con HTML

📄 Script interno

JS directamente dentro del HTML con <script>. Útil para pruebas rápidas.

<body>
  <!-- contenido -->
  <script>
    console.log("Hola!");
  </script>
</body>

📂 Archivo externo ✅

JS en un archivo .js separado. Recomendado para proyectos reales.

<body>
  <!-- contenido -->
  <script src="script.js"></script>
</body>
Siempre colocar el <script> al final del body para que el HTML cargue antes de ejecutar JS.
Entorno de Trabajo · Estructura

Estructura Base HTML + JS

<!-- index.html -->
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Mi Proyecto</title>
</head>
<body>
  <h1>Mi primer proyecto JS</h1>
  <p id="mensaje"></p> <!-- JS lo llenará -->

  <!-- Script al final del body ✅ -->
  <script src="script.js"></script>
</body>
</html>
