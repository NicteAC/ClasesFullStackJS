🧩 OOCSS: Object-Oriented CSS

Tratar estilos como objetos reutilizables

1️⃣ Separar Estructura de Skin

Estructura: layout, tamaño, spacing

Skin: colores, bordes, sombras

2️⃣ Evitar Dependencia Contextual

Un botón se ve igual en el header que en el sidebar

🎯 Beneficio: Cambias colores sin tocar estructura, y viceversa

⏱️ 3 min

📊 OOCSS en Práctica

❌ Sin OOCSS

.error-button {
  padding: 10px 20px;
  border-radius: 8px;
  background: red;
}

.success-button {
  padding: 10px 20px;  /* repetido */
  border-radius: 8px;  /* repetido */
  background: green;
}

✅ Con OOCSS

/* Estructura */
.button {
  padding: 10px 20px;
  border-radius: 8px;
}

/* Skins */
.button--error { background: red; }
.button--success { background: green; }

🔄 Cambiar padding de TODOS los botones = 1 solo lugar

🏷️ Estados SMACSS: Prefijo is- (is-active, is-open) y has- (has-error)

⏱️ 5 min

📂 Estructura de Carpetas SMACSS

scss/

Estados con JavaScript

// Agregar estado
element.classList.add('is-active');

// Toggle estado
element.classList.toggle('is-open');

Los estados is- y has- son controlados por JS, nunca hardcodeados en HTML.

⚠️ Errores Comunes y Soluciones

1. Menú mobile no aparece

.nav__menu.is-open { display: flex; }
@include mobile { &.is-open { display: flex; } }

is-open debe estar DENTRO del media query

2. Menú a 100vh del viewport

.nav { display: flex; }
.nav { position: sticky; top: 0; }

Nav necesita position para que top:100% funcione

3. Mixin no acepta múltiples props

@mixin transition($prop: all)
@mixin transition($props...)

Usar varargs (...) para múltiples valores

4. Menú se cierra al abrir

toggle.addEventListener('click', () => {})
toggle.addEventListener('click', (e) => { e.stopPropagation(); })

stopPropagation evita que el evento cierre el menú
