Bloque 1 · Fundamentos

¿Qué es una Función?

📦 Definición

Bloque de código reutilizable que realiza una tarea específica. Se define una vez y se puede llamar cuantas veces sea necesario.

Al definirla no se ejecuta. Se ejecuta cuando la invocamos.

🍰 Analogía: receta de cocina

  • Ingredientes → parámetros de entrada
  • Pasos → proceso interno de la función
  • Torta → resultado (return)

La misma receta (función) puede usarse con distintos ingredientes (argumentos).

Sin funciones, deberíamos repetir el mismo código en cada lugar donde lo necesitamos. Con funciones, lo escribimos una vez y lo reutilizamos.
Bloque 1 · Ventajas

¿Por qué usar Funciones?

🧩

Modularidad

Cada función realiza una tarea específica y bien definida.

♻️

Reutilización

Se pueden llamar múltiples veces desde cualquier parte del código.

🔧

Mantenimiento

Modificar una función actualiza su comportamiento en todos los lugares donde se usa.

🐛

Menos errores

Código más estructurado, fácil de testear y depurar.

👁️

Legibilidad

Nombre descriptivo que comunica qué hace el bloque de código.

Bloque 1 · Tipos

Funciones Declaradas

// Declaración: define la función
function saludar(nombre) {
  console.log(`¡Hola, ${nombre}!`);
}

// Invocación: ejecuta la función
saludar("Ana"); // ¡Hola, Ana!
saludar("Luis"); // ¡Hola, Luis!

// Hoisting: se puede llamar ANTES de declararla
despedir("Mía"); // ✅ Funciona gracias al hoisting

function despedir(nombre) {
  console.log(`Hasta luego, ${nombre}!`);
}
El hoisting eleva la declaración al inicio del archivo. Permite llamar la función antes de que aparezca en el código.
Bloque 1 · Tipos

Funciones Anónimas y Arrow

👤 Función Anónima

Sin nombre, asignada a una variable. No hay hoisting.

const sumar = function(a, b) {
  return a + b;
};
console.log(sumar(3, 4)); // 7

// Uso típico: callback en setTimeout
setTimeout(function() {
  console.log("3 segundos");
}, 3000);

⚡ Arrow Function (ES6)

Sintaxis concisa. Sin this propio. No hay hoisting.

// Con cuerpo
const multiplicar = (a, b) => {
  return a * b;
};

// Una línea: return implícito
const restar = (a, b) => a - b;

// Un solo parámetro: sin paréntesis
const doble = n => n * 2;

Bloque 1 · Parámetros y retorno

Parámetros y Valor de Retorno

📥 Parámetros vs Argumentos

// "a" y "b" son PARÁMETROS (en la declaración)
function sumar(a, b) {
  return a + b;
}

// 5 y 3 son ARGUMENTOS (al invocar)
sumar(5, 3); // 8

📤 return — devolver un valor

// Sin return: solo ejecuta
function mostrar(n) {
  console.log(n); // muestra, no guarda
}

// Con return: valor reutilizable
function calcular(n) {
  return n * 2; // devuelve el valor
}
const resultado = calcular(5); // 10
return detiene la ejecución de la función. Todo código después de un return en la misma función es inalcanzable.

Errores Comunes con Funciones

❌ Olvidar paréntesis

  console.log("Hola");
}

// ❌ No invoca la función
saludar; // solo referencia

// ✅ Invoca la función
saludar(); // ejecuta

❌ Olvidar return

function doble(n) {
  n * 2; // ❌ calcula pero no retorna
}
console.log(doble(5)); // undefined

function doble(n) {
  return n * 2; // ✅
}

function sumar(a, b) {

sumar(5); // b = undefined → NaN

Bloque 3 · Scope

Variables Locales vs Globales

🌍 Variable Global

Declarada fuera de cualquier función. Accesible desde cualquier parte del código.

let contador = 0; // global

function incrementar() {
  contador++; // ✅ accede
}
console.log(contador); // ✅

🏠 Variable Local

Declarada dentro de una función. Solo existe dentro de esa función.

function calcular() {
  let resultado = 42; // local
  console.log(resultado); // ✅
}
console.log(resultado); // ❌ Error
⚠️ Variables globales: modificables desde cualquier parte → bugs difíciles de rastrear. Usarlas con cuidado y solo cuando sea necesario.
Bloque 3 · Parámetros

Parámetros con Valores por Defecto

// Sin valor por defecto: undefined si no se pasa
function saludar(nombre) {
  console.log(`Hola, ${nombre}!`);
}
saludar(); // "Hola, undefined!" ⚠️

// Con valor por defecto (ES6)
function saludar(nombre = "invitado") {
  console.log(`Hola, ${nombre}!`);
}
saludar(); // "Hola, invitado!" ✅
saludar("Ana"); // "Hola, Ana!"

// Combinar parámetros obligatorios y opcionales
function descuento(precio, porcentaje = 10) {
  return precio - (precio * porcentaje / 100);
}
console.log(descuento(100)); // 90 (10% por defecto)
console.log(descuento(100, 20)); // 80
Bloque 3 · Operador Rest

Operador Rest (...)

// Recibir número indefinido de argumentos como array
function sumarTodo(...numeros) {
  return numeros.reduce((acc, n) => acc + n, 0);
}
console.log(sumarTodo(1, 2, 3)); // 6
console.log(sumarTodo(10, 20, 30, 40)); // 100

// Combinando fijos + rest (rest debe ir al final)
function presentar(nombre, ...hobbies) {
  console.log(`${nombre} disfruta: ${hobbies.join(", ")}`);
}
presentar("Ana", "leer", "correr", "cocinar");
// "Ana disfruta: leer, correr, cocinar"
El operador rest (...) siempre debe ser el último parámetro de la función. Captura todos los argumentos restantes en un array.
Bloque 4 · Callbacks

Parámetros como Funciones (Callbacks)

// Una función recibe otra función como parámetro
function procesar(numero, operacion) {
  return operacion(numero); // llama al callback
}

console.log(procesar(5, n => n * 2)); // 10
console.log(procesar(5, n => n ** 2)); // 25

// Ya los usaron: map() y filter() reciben callbacks
const nums = [1,2,3,4,5];
const pares = nums.filter(n => n % 2 === 0); // callback
const dobles = nums.map(n => n * 2); // callback
Los callbacks son la base de la programación asíncrona en JavaScript: setTimeout, fetch, eventos del DOM.
Bloque 4 · Funciones anidadas

Funciones Anidadas

// Función dentro de otra función
function calcularArea(base, altura) {

  // Función interna: solo accesible aquí
  function validar(valor) {
    return typeof valor === "number" && valor > 0;
  }

  if (!validar(base) || !validar(altura)) {
    return "Valores inválidos";
  }
  return base * altura;
}

console.log(calcularArea(5, 3)); // 15
console.log(calcularArea(-1, 3)); // "Valores inválidos"
Las funciones anidadas encapsulan lógica sin contaminar el scope global. La función interna solo existe dentro de la externa.
