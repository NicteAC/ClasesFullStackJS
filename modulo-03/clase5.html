Bloque 1 · Objetos

¿Qué es un Objeto?

📦 Definición

Estructura de datos que almacena información en pares clave-valor. Puede tener propiedades (datos) y métodos (funciones).

Es más flexible que los tipos primitivos: puede contener múltiples valores organizados en un solo contenedor.

💡 Vida real → código

Un auto tiene: marca, modelo, año (propiedades) y puede arrancar, frenar (métodos). En código:

const auto = {
  marca: "Toyota",
  anio: 2023,
  arrancar: function() {
    console.log("Vroom!");
  }
};
Los objetos agrupan datos relacionados en un solo lugar → código más organizado y legible.
Bloque 1 · Propiedades y métodos

Propiedades y Métodos

const persona = {
  // Propiedades (pares clave-valor)
  nombre: "Carlos",
  edad: 25,
  profesion: "Desarrollador",

  // Método (función dentro del objeto)
  presentarse: function() {
    return `Hola, soy ${this.nombre} y trabajo como ${this.profesion}.`;
  }
};

console.log(persona.nombre); // "Carlos"
console.log(persona.presentarse()); // "Hola, soy Carlos..."
this dentro de un método se refiere al propio objeto. this.nombre es equivalente a persona.nombre.
Bloque 1 · Crear y modificar

Crear, Modificar y Eliminar Propiedades

// Crear con literal de objeto {}
const perro = { nombre: "Rex", raza: "Labrador" };

// Modificar una propiedad existente
perro.nombre = "Max";

// Agregar nueva propiedad dinámicamente
perro.edad = 3;
console.log(perro); // { nombre:"Max", raza:"Labrador", edad:3 }

// Eliminar una propiedad
delete perro.raza;
console.log(perro); // { nombre:"Max", edad:3 }
Bloque 1 · Notación

Notación de Punto vs Corchetes

🔵 Notación de punto obj.clave

Más común y legible. Usar siempre que sea posible.

console.log(persona.nombre); // ✅
console.log(persona.edad); // ✅

🟡 Notación de corchetes obj["clave"]

Necesaria para: claves con espacios, caracteres especiales o acceso dinámico.

// Clave con espacios
obj["nombre completo"];

// Acceso dinámico con variable
const campo = "nombre";
console.log(persona[campo]); // ✅
Usá punto por defecto · Usá corchetes para acceso dinámico o claves con caracteres especiales.

Bloque 2 · Math

Objeto Math — Métodos Principales

// Redondeo
Math.round(5.7) // 6 → más cercano
Math.ceil(5.1) // 6 → siempre hacia arriba
Math.floor(5.9) // 5 → siempre hacia abajo
Math.trunc(5.9) // 5 → elimina decimales (sin redondear)

// Matemáticas
Math.abs(-8) // 8 → valor absoluto
Math.pow(2, 10) // 1024 → potencia
Math.sqrt(16) // 4 → raíz cuadrada
Math.max(3,7,1) // 7 → máximo
Math.min(3,7,1) // 1 → mínimo

// Número aleatorio entre min y max (entero)
function random(min, max) {
  return Math.floor(Math.random() * (max - min + 1)) + min;
}

Bloque 3 · String.replace

Reemplazo de Texto

const frase = "Hola mundo hola";

// Primera ocurrencia
frase.replace("hola", "chau"); // "Hola mundo chau" (solo 1ª)

// Todas las ocurrencias (regex /g)
frase.replace(/hola/gi, "chau"); // "chau mundo chau" (/gi = global + ignore case)

// ES2021: replaceAll()
frase.replaceAll("hola", "chau"); // sin regex, reemplaza todas
MétodoReemplazaMayúsculas/minúsculas
replace("a","b")Solo la primeraDistingue
replace(/a/g,"b")TodasDistingue
replace(/a/gi,"b")TodasIgnora
replaceAll("a","b")TodasDistingue
Bloque 3 · String comparación

Comparar Cadenas Correctamente

// Básica: sensible a mayúsculas
"hola" === "Hola" // false ⚠️

// Ignorando mayúsculas
"hola".toLowerCase() === "Hola".toLowerCase() // true ✅

// Con espacios y mayúsculas: trim() + toLowerCase()
" Hola ".trim().toLowerCase() === "hola" // true ✅

// Comparación natural (orden de palabras, acentos)
["banana", "Árbol", "manzana"].sort((a,b) => a.localeCompare(b));
// ["Árbol", "banana", "manzana"] — ordenado correctamente
Siempre aplicar .trim().toLowerCase() antes de comparar texto ingresado por el usuario.
Bloque 3 · String.match

Extraer Datos con String.match()

const texto = "Pedido 3 productos por $150.50 y 2 por $80";

// Extraer todos los números enteros
const enteros = texto.match(/\d+/g);
console.log(enteros); // ["3","150","50","2","80"]

// Convertir a números
const nums = enteros.map(Number);
console.log(nums); // [3, 150, 50, 2, 80]

// Extraer decimales y negativos
const precios = texto.match(/-?\d+(\.\d+)?/g);
console.log(precios); // ["3","150.50","2","80"]
/\d+/g — \d es dígito, + es uno o más, g es global (todas las coincidencias).
Bloque 3 · Funciones anidadas

Funciones Anidadas

¿Cuándo usarlas?

  • La lógica interna solo se usa dentro de la función externa
  • Para mejorar la organización y legibilidad
  • Para evitar contaminar el scope global
  • Para facilitar el uso de closures

🔑 Regla de alcance léxico

La función interna puede acceder a las variables de la función externa. La función externa NO puede acceder a las de la interna.

function calcularArea(base, altura) {
  function validar(v) { return typeof v === "number" && v > 0; }
  if (!validar(base) || !validar(altura)) return "Valores inválidos";
  return base * altura;
}
Bloque 3 · Closures

Closures — Funciones que "Recuerdan"

// Un closure ocurre cuando una función interna
// se retorna y "recuerda" el ámbito donde fue creada

function crearContador() {
  let cuenta = 0; // variable privada

  return function() { // función interna retornada
    cuenta++;
    console.log(`Conteo: ${cuenta}`);
  };
}

const contador = crearContador(); // guarda la función interna
contador(); // Conteo: 1
contador(); // Conteo: 2
contador(); // Conteo: 3
El closure permite crear variables privadas que solo la función interna puede acceder y modificar.
