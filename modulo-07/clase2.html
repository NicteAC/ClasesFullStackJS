¿Qué es una API?

Una API (Application Programming Interface) es un conjunto de reglas que permite que dos sistemas de software se comuniquen entre sí.

🔀

Separación

El frontend se ocupa de la UI, el backend de los datos y la lógica. La API los conecta.

🌐

Flexibilidad

APIs públicas (OpenWeather, Reqres) y privadas (requieren auth). Formato estándar: JSON sobre HTTP.

🔒

Seguridad

Control de acceso, monitoreo de uso y protección de datos mediante tokens y permisos.

Comunicación Cliente / Servidor

Toda petición HTTP tiene tres partes clave:

📋

Header

Meta-información: tipo de contenido (Content-Type), autorización (Authorization: Bearer ...).

📦

Body

El contenido principal. En POST/PUT contiene los datos enviados. En la respuesta, los datos que vuelven (JSON).

🚦

Status

Código numérico del resultado: 200 OK, 201 Created, 404 Not Found, 500 Server Error.

Códigos de estado HTTP

CódigoSignificadoCuándo ocurre
200OKLa petición fue exitosa
201CreatedSe creó un recurso nuevo (POST exitoso)
400Bad RequestDatos mal formados o inválidos
401UnauthorizedFalta token o token inválido
404Not FoundEl recurso no existe
500Internal Server ErrorError del lado del servidor

CRUD, Web Services y REST

CRUD son las 4 operaciones básicas sobre datos. REST es el estilo de arquitectura que usa verbos HTTP para ejecutarlas.

GET

Leer datos. Obtener lista de usuarios.

POST

Crear datos. Registrar un usuario nuevo.

PUT / PATCH

Actualizar datos existentes total o parcialmente.

DELETE

Eliminar un recurso específico.

Un Web Service expone estas operaciones mediante URLs (endpoints). Ejemplo: GET /api/users devuelve la lista de usuarios.

JSON — El formato de los datos

JSON (JavaScript Object Notation) es el formato estándar que usan las APIs para enviar y recibir datos.

Ejemplo de JSON

{
  "id": 1,
  "name": "Juan Pérez",
  "email": "juan@mail.com",
  "active": true,
  "roles": ["admin", "editor"],
  "address": {
    "city": "Santiago",
    "zip": "8320000"
  }
}

Características

Claves siempre entre comillas dobles.

Valores: string, number, boolean, array, object, null.

Sintaxis casi idéntica a objetos JS — fácil de parsear.

Más liviano que XML. Estándar de la industria.

Promesas y Callbacks

Las llamadas a API son asíncronas: el código no espera la respuesta. Necesitamos mecanismos para manejar eso.

Callback (el problema)

fetchData(function(error, data) {
  if (error) {
    console.error(error)
    return
  }
  // Callback dentro de callback
  fetchMore(data.id, function(err, more) {
    // Callback hell...
  })
})

Promesa (la solución)

fetchData()
  .then(function(data) {
    console.log('Datos:', data)
    return fetchMore(data.id)
  })
  .then(function(more) {
    console.log('Más datos:', more)
  })
  .catch(function(error) {
    console.error('Error:', error)
  })

Async / Await

Sintaxis moderna que hace que el código asíncrono se lea como si fuera síncrono.

async fetchData() {
  try {
    var response = await axios.get('https://reqres.in/api/users')
    this.users = response.data.data
    console.log('Usuarios cargados:', this.users)
  } catch (error) {
    console.error('Error:', error)
    this.errorMsg = 'No se pudieron cargar los datos'
  } finally {
    this.loading = false
  }
}

try ejecuta el código, catch atrapa errores, finally se ejecuta siempre.

Axios — Consumiendo APIs

Librería basada en promesas para hacer peticiones HTTP. Se instala con npm install axios.

⚡

Basado en Promesas

Código limpio con .then/.catch o async/await. Sin XMLHttpRequest manual.

⚙️

Configuración global

URL base, headers por defecto e instancias reutilizables.

🔄

Interceptores

Interceptar peticiones y respuestas para lógica centralizada (tokens, errores).

🌍

Compatible

Funciona en navegador y Node.js. Transforma JSON automáticamente.

Axios GET en Vue

<template>
  <div>
    <p v-if="loading">Cargando...</p>
    <p v-if="error" class="error">{{ error }}</p>
    <ul>
      <li v-for="user in users" :key="user.id">
        {{ user.first_name }} {{ user.last_name }}
      </li>
    </ul>
  </div>
</template>

<script>
import axios from 'axios'

export default {
  data() {
    return { users: [], loading: true, error: '' }
  },
  created() {
    axios.get('https://reqres.in/api/users')
      .then(function(response) {
        this.users = response.data.data
      }.bind(this))
      .catch(function(err) {
        this.error = 'Error al cargar usuarios'
      }.bind(this))
      .finally(function() {
        this.loading = false
      }.bind(this))
  }
}
</script>

Axios POST — Crear datos

<template>
  <form @submit.prevent="crearUsuario">
    <input v-model="nombre" placeholder="Nombre" />
    <input v-model="trabajo" placeholder="Trabajo" />
    <button type="submit">Crear</button>
  </form>
</template>

<script>
import axios from 'axios'

export default {
  data() {
    return { nombre: '', trabajo: '' }
  },
  methods: {
    crearUsuario() {
      axios.post('https://reqres.in/api/users', {
        name: this.nombre,
        job: this.trabajo
      })
      .then(function(res) {
        console.log('Creado:', res.data)
        alert('Usuario creado con id: ' + res.data.id)
      })
      .catch(function(err) {
        console.error(err)
      })
    }
  }
}
</script>

Manejo de errores con Axios

Axios distingue 3 tipos de error según dónde ocurrió el problema:

error.response

El servidor respondió con un código de error (4xx, 5xx). Tenemos status y data.

error.request

La petición se envió pero no hubo respuesta. Problemas de red o servidor caído.

error.message

Error al configurar la petición. URL mal escrita, error de código.

Autenticación con JWT

JSON Web Token: un token firmado que el servidor genera al hacer login y el cliente envía en cada petición.

1. Login (user + pass)
→
2. Servidor genera JWT
→
3. Cliente guarda token
→
4. Token en cada request
→
5. Servidor valida

El JWT tiene 3 partes: Header (algoritmo), Payload (datos del usuario) y Signature (firma de verificación).

JWT con Axios en Vue

<script>
import axios from 'axios'

export default {
  data() {
    return { email: '', password: '', token: '' }
  },
  methods: {
    login() {
      axios.post('https://reqres.in/api/login', {
        email: this.email,
        password: this.password
      })
      .then(function(res) {
        this.token = res.data.token
        localStorage.setItem('token', this.token)
      }.bind(this))
      .catch(function(err) {
        alert('Login fallido')
      })
    },
    fetchProtected() {
      var token = localStorage.getItem('token')
      axios.get('https://reqres.in/api/users', {
        headers: { 'Authorization': 'Bearer ' + token }
      })
      .then(function(res) {
        console.log('Datos protegidos:', res.data)
      }.bind(this))
    }
  }
}
</script>

Encriptación en el cliente

Capa adicional de seguridad para datos sensibles antes de enviarlos al servidor.

🔐

¿Qué encriptar?

Contraseñas antes del envío, tokens temporales, datos sensibles en tránsito.

⚠️

Limitaciones

No reemplaza HTTPS ni la validación del servidor. Es una capa extra, no la única.

🛠️

Herramientas

Librerías como CryptoJS para AES, SHA-256. Se usan en métodos o funciones auxiliares.

Instancia global de Axios

Centralizamos configuración para no repetir la URL base y los headers en cada componente.

// src/services/api.js
import axios from 'axios'

var api = axios.create({
  baseURL: 'https://reqres.in/api',
  headers: {
    'Content-Type': 'application/json'
  }
})

// Interceptor: agregar token automáticamente
api.interceptors.request.use(function(config) {
  var token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = 'Bearer ' + token
  }
  return config
})

// Interceptor de respuesta: manejo global de errores
api.interceptors.response.use(
  function(response) { return response },
  function(error) {
    if (error.response && error.response.status === 401) {
      alert('Sesión expirada')
    }
    return Promise.reject(error)
  }
)

export default api
