Pruebas unitarias — Características

Verifican que una unidad de código (función, método, componente) funciona correctamente de forma aislada.

Ventajas

Rápidas (sin navegador). Precisas (saben exactamente qué falló). Fáciles de automatizar. Ideales para TDD.

Limitaciones

No detectan errores de integración entre componentes. No validan la experiencia del usuario real.

TDD — Desarrollo dirigido por tests

Se escribe la prueba antes que el código. Parece al revés, pero funciona.

🔴 Rojo
Escribir test que falla
→
🟢 Verde
Código mínimo para pasar
→
🔵 Refactorizar
Mejorar sin romper tests
→
🔴 Repetir

Beneficio: cada funcionalidad nace cubierta por tests. El código es más enfocado y menos propenso a errores.

Vue Test Utils

Librería oficial para testear componentes Vue. Monta componentes sin navegador.

🔌

mount / shallowMount

mount renderiza todo. shallowMount reemplaza hijos por stubs.

🔍

find / findAll

Buscar elementos en el DOM renderizado por selector CSS.

⚡

trigger / setValue

Simular clicks, inputs, submits y otros eventos del usuario.

Mocks y Stubs — ¿Qué son?

Stub

Reemplaza una función por una versión falsa que devuelve un resultado fijo. Ejemplo: simular respuesta de API sin hacer la petición real.

Mock

Igual que un stub pero además registra cómo fue llamado: cuántas veces, con qué argumentos. Permite verificar interacciones.

// Mock con Jest
const mockFn = jest.fn()
mockFn('hola')

expect(mockFn).toHaveBeenCalled()
expect(mockFn).toHaveBeenCalledWith('hola')
expect(mockFn).toHaveBeenCalledTimes(1)

Pruebas E2E — Características

Validan flujos completos como lo haría un usuario real: clicks, inputs, navegación.

Ventajas

Alta fidelidad al comportamiento real. Detectan errores de integración. Validan UX completa.

Limitaciones

Más lentas. Más frágiles ante cambios de UI. Requieren servidor corriendo. Mantenimiento constante.

Cypress vs Nightwatch

Cypress

Interfaz visual, ve tests en tiempo real. Capturas y videos automáticos. Documentación excelente. Curva baja.

Limitado a navegadores Chromium.

Nightwatch

Basado en Selenium/WebDriver. Multi-browser (Firefox, Safari, Edge). Integración CI/CD robusta.

Más complejo de configurar.

# Agregar Cypress al proyecto

# Ejecutar en modo interactivo
npm run test:e2e

# Estructura generada:

Anatomía de un test Cypress

describe('Pokédex', () => {
  it('carga la lista de pokémon', () => {
    cy.visit('/')
    cy.contains('Pokédex')
    cy.get('[data-cy="pokemon-card"]').should('have.length.greaterThan', 0)
  })

  it('agrega un pokémon a favoritos', () => {
    cy.visit('/')
    cy.get('[data-cy="fav-btn"]').first().click()
    cy.get('[data-cy="fav-count"]').should('contain', '1')
  })

  it('busca un pokémon por nombre', () => {
    cy.visit('/')
    cy.get('[data-cy="search"]').type('pikachu')
    cy.get('[data-cy="pokemon-card"]').should('have.length', 1)
  })
})

Ejercicios de testing

Ejercicio 1 — Pruebas unitarias básicas (~25 min)

Crea TodoItem.vue con prop texto (String, required) y botón "Eliminar" que emite evento. Escribe 3 tests con Jest + Vue Test Utils:

1. Renderiza el texto de la prop.
2. El botón "Eliminar" existe.
3. Emite eliminar al hacer click.

Ejercicio 2 — TDD: construye desde el test (~25 min)

Escribe primero los tests para un componente Buscador.vue que aún no existe:

1. Renderiza un input.
2. Al escribir, emite buscar con el texto.
3. El input tiene placeholder "Buscar...".

Corre los tests (fallan = rojo). Crea el componente para que pasen (verde).

Ejercicio 3 — Prueba E2E con Cypress (~20 min)

Escribe un test E2E que:

1. Visite tu app en /.
2. Verifique el título principal.
3. Interactúe con al menos 2 elementos (input + botón).
4. Verifique que el resultado aparece en la UI.

Bonus — Mock de API (~10 min)

Test unitario para un componente que usa Axios:

1. jest.mock('axios')
2. axios.get.mockResolvedValue({...})
3. Montar componente y verificar que renderiza los datos mockeados.
