Backend tradicional vs BaaS

Tradicional (monolítico)

Toda la lógica, rutas, BD y seguridad en un solo servidor que tú administras.

✅ Control total

✅ Personalización completa

❌ Mayor complejidad

❌ Administrar servidores

BaaS (Firebase, Supabase)

Servicios en la nube con auth, BD, hosting y funciones listas para usar.

✅ Rápido de implementar

✅ Escalabilidad automática

❌ Menor personalización

❌ Dependencia del proveedor

¿Qué es Firebase?

Plataforma de Google que ofrece backend completo como servicio. Auth, base de datos, hosting, storage y funciones en la nube.

🔑

Authentication

Login con email/password, Google, Facebook, GitHub. Manejo automático de tokens y sesiones.

🗄️

Cloud Firestore

Base de datos NoSQL en tiempo real. Colecciones y documentos con sincronización instantánea.

🌐

Hosting

Deploy de sitios estáticos con SSL gratis, CDN global y despliegue por línea de comandos.

⚡

Cloud Functions

Código del lado del servidor que se ejecuta en respuesta a eventos, sin administrar servidores.

Más servicios de Firebase

📁

Storage

📊

Analytics

Métricas de uso, comportamiento de usuarios y eventos en tiempo real.

🐛

Crashlytics

Reportes de errores y fallos en tiempo real para debug rápido.

var db = getFirestore(app)

export { auth, db }

Firebase Authentication

Soporta múltiples métodos de login. Hoy trabajaremos con email y contraseña.

API modular (v12): Las funciones se importan individualmente y reciben auth como primer argumento.

Registro

createUserWithEmailAndPassword(auth, email, pass)

Login

signInWithEmailAndPassword(auth, email, pass)

Logout

signOut(auth)

Registro y Login en Vue

<script>
import { auth } from '../firebase'
import {
  createUserWithEmailAndPassword,
  signInWithEmailAndPassword,
  signOut
} from 'firebase/auth'

export default {
  data() {
    return { email: '', password: '', error: '', user: null }
  },
  methods: {
    registrar() {
      var self = this
      createUserWithEmailAndPassword(auth, self.email, self.password)
        .then(function(result) {
          self.user = result.user
          self.error = ''
        })
        .catch(function(err) { self.error = err.message })
    },
    login() {
      var self = this
      signInWithEmailAndPassword(auth, self.email, self.password)
        .then(function(result) {
          self.user = result.user
          self.error = ''
        })
        .catch(function(err) { self.error = err.message })
    },
    logout() {
      var self = this
      signOut(auth).then(function() { self.user = null })
    }
  }
}
</script>

Detectar estado de autenticación

onAuthStateChanged se ejecuta cada vez que el usuario inicia o cierra sesión.

import { auth } from './firebase'
import { onAuthStateChanged } from 'firebase/auth'

export default {
  data() {
    return { user: null }
  },
  created() {
    var self = this
    onAuthStateChanged(auth, function(user) {
      if (user) {
        self.user = { uid: user.uid, email: user.email }
      } else {
        self.user = null
      }
    })
  }
}

Mismo patrón modular: importar onAuthStateChanged y pasar auth como primer argumento.

Proteger rutas con Vue Router

Usamos meta.requiresAuth y un navigation guard para bloquear acceso.

// router/index.js
import { auth } from '../firebase'

var routes = [
  { path: '/', component: Home },
  { path: '/login', component: Login },
  { path: '/dashboard', component: Dashboard,
    meta: { requiresAuth: true }
  }
]

router.beforeEach(function(to, from, next) {
  var requiresAuth = to.matched.some(function(record) {
    return record.meta.requiresAuth
  })
  // auth.currentUser funciona igual en v12
  var currentUser = auth.currentUser

  if (requiresAuth && !currentUser) {
    next('/login')
  } else {
    next()
  }
})

Cloud Firestore

Base de datos NoSQL en la nube organizada en colecciones y documentos.

Estructura

📁 colección: tareas
  📄 documento: { titulo, completada }
  📄 documento: { titulo, completada }
📁 colección: usuarios
  📄 documento: { nombre, email }

API modular (v12)

getDocs(collection(db, 'tareas'))

addDoc(collection(db, 'tareas'), {...})

deleteDoc(doc(db, 'tareas', id))

updateDoc(doc(db, 'tareas', id), {...})

Leer datos de Firestore

<script>
import { db } from '../firebase'
import { collection, getDocs } from 'firebase/firestore'

export default {
  data() {
    return { tareas: [], loading: true }
  },
  created() {
    var self = this
    getDocs(collection(db, 'tareas'))
      .then(function(snapshot) {
        var lista = []
        snapshot.forEach(function(docu) {
          lista.push({
            id: docu.id,
            titulo: docu.data().titulo,
            completada: docu.data().completada
          })
        })
        self.tareas = lista
      })
      .catch(function(err) {
        console.error('Error:', err)
      })
      .finally(function() {
        self.loading = false
      })
  }
}
</script>

Crear y eliminar en Firestore

import { db } from '../firebase'
import { collection, addDoc, deleteDoc, doc } from 'firebase/firestore'

methods: {
  agregar() {
    var self = this
    addDoc(collection(db, 'tareas'), {
      titulo: self.nuevaTarea,
      completada: false
    })
    .then(function(docRef) {
      self.tareas.push({
        id: docRef.id,
        titulo: self.nuevaTarea,
        completada: false
      })
      self.nuevaTarea = ''
    })
    .catch(function(err) { console.error(err) })
  },
  eliminar(id) {
    var self = this
    deleteDoc(doc(db, 'tareas', id))
      .then(function() {
        self.tareas = self.tareas.filter(function(t) {
          return t.id !== id
        })
      })
      .catch(function(err) { console.error(err) })
  }
}

Vuex + Firebase

Centralizar el estado del usuario autenticado en el store.

// store/index.js
import { auth } from '../firebase'
import { signInWithEmailAndPassword, signOut } from 'firebase/auth'

export default new Vuex.Store({
  state: { user: null, error: null },
  mutations: {
    SET_USER(state, user) { state.user = user },
    SET_ERROR(state, error) { state.error = error }
  },
  actions: {
    login(context, payload) {
      return signInWithEmailAndPassword(auth, payload.email, payload.password)
        .then(function(result) {
          context.commit('SET_USER', {
            uid: result.user.uid,
            email: result.user.email
          })
        })
        .catch(function(err) {
          context.commit('SET_ERROR', err.message)
        })
    },
    logout(context) {
      return signOut(auth).then(function() {
        context.commit('SET_USER', null)
      })
    }
  },
  getters: {
    isAuthenticated(state) { return !!state.user }
  }
})

Reglas de seguridad en Firestore

Controlan quién puede leer y escribir cada colección.

// Solo usuarios autenticados pueden leer y escribir
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /tareas/{tareaId} {
      allow read, write: if request.auth != null;
    }
  }
}

Importante: nunca dejar las reglas en modo test en producción. Siempre restringir acceso según autenticación.
