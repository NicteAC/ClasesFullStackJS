¿Qué es un componente?

Una pieza independiente de la interfaz con su propio template, lógica y estilos, todo en un archivo .vue.

<template>
  <div class="saludo">
    <h2>{{ mensaje }}</h2>
  </div>
</template>

<script>
export default {
  data() {
    return {
      mensaje: '¡Hola desde mi componente!'
    }
  }
}
</script>

<style scoped>
.saludo { color: navy; }
</style>

¿Por qué componentizar?

♻️

Reutilización

Usa el mismo componente en distintas partes sin duplicar código.

📦

Modularización

Código organizado, fácil de leer, mantener y escalar.

👥

Trabajo en equipo

Cada dev trabaja en componentes distintos sin conflictos.

Jerarquía padre → hijo

Los componentes forman un árbol. El padre contiene hijos; cada hijo puede contener más hijos.

// App.vue (Padre)
<template>
  <div>
    <HeaderApp />
    <ListaTareas />
    <FooterApp />
  </div>
</template>

<script>
import HeaderApp from './components/HeaderApp.vue'
import ListaTareas from './components/ListaTareas.vue'
import FooterApp from './components/FooterApp.vue'

export default {
  components: { HeaderApp, ListaTareas, FooterApp }
}
</script>

Props — Datos del padre al hijo

Las props permiten enviar datos del componente padre al hijo, en flujo unidireccional.

Padre

<template>
  <Tarjeta
    titulo="Aprende Vue"
    :likes="42"
  />
</template>

Hijo (Tarjeta.vue)

<template>
  <div>
    <h3>{{ titulo }}</h3>
    <span>{{ likes }} likes</span>
  </div>
</template>

<script>
export default {
  props: ['titulo', 'likes']
}
</script>

Validación de props

Podemos tipar, requerir y validar cada prop para hacer componentes más robustos.

export default {
  props: {
    titulo: {
      type: String,
      required: true
    },
    likes: {
      type: Number,
      default: 0
    },
    tipo: {
      type: String,
      default: 'info',
      validator(value) {
        return ['info', 'success', 'error'].includes(value)
      }
    }
  }
}

$emit — Eventos hijo → padre

El hijo notifica al padre que algo ocurrió usando this.$emit('nombre-evento').

Hijo (Boton.vue)

<template>
  <button @click="notificar">
    Click aquí
  </button>
</template>

<script>
export default {
  methods: {
    notificar() {
      this.$emit('clicked', 'dato extra')
    }
  }
}
</script>

Padre

<template>
  <Boton
    @clicked="manejarClick"
  />
</template>

<script>
export default {
  methods: {
    manejarClick(dato) {
      console.log('Recibido:', dato)
    }
  }
}
</script>

Slots — Contenido flexible

Los slots permiten inyectar contenido desde el padre dentro de la estructura del hijo.

Hijo (Panel.vue)

<template>
  <div class="panel">
    <div class="header">
      <slot name="header">
        Título por defecto
      </slot>
    </div>
    <div class="body">
      <slot></slot>
    </div>
  </div>
</template>

Padre

<Panel>
  <template v-slot:header>
    Mi título personalizado
  </template>

  <p>Contenido del panel</p>
</Panel>

Componentes dinámicos

Con <component :is="..."> renderizamos distintos componentes en el mismo lugar.

<template>
  <div>
    <button @click="vista = 'TabInicio'">Inicio</button>
    <button @click="vista = 'TabPerfil'">Perfil</button>

    <keep-alive>
      <component :is="vista"></component>
    </keep-alive>
  </div>
</template>

<script>
import TabInicio from './TabInicio.vue'
import TabPerfil from './TabPerfil.vue'

export default {
  components: { TabInicio, TabPerfil },
  data() { return { vista: 'TabInicio' } }
}
</script>

Ciclo de vida de un componente

Cada componente pasa por etapas desde que nace hasta que se destruye. Vue nos da hooks para intervenir en cada momento.

beforeCreate
→
created
→
beforeMount
→
mounted
→
beforeUpdate
→
updated
→
beforeDestroy
→
destroyed

Hooks del ciclo de vida

HookMomentoUso típico
beforeCreateAntes de inicializar dataConfig temprana
createdData y eventos listosLlamadas a API, lógica inicial
beforeMountAntes de insertar en el DOMAjustes pre-render
mountedYa insertado en el DOMAcceso al DOM, librerías externas
beforeUpdateAntes de re-renderValidaciones
updatedDespués de re-renderMonitoreo de cambios
beforeDestroyAntes de destruirLimpiar listeners, timers
destroyedComponente destruidoConfirmación de limpieza

¿Cuándo usar cada hook?

created

Fetch de datos, inicialización de lógica. La data ya existe pero el DOM aún no.

mounted

Integrar Chart.js, manipular el DOM, iniciar suscripciones WebSocket.

beforeDestroy

Limpiar setInterval, removeEventListener, cerrar conexiones.

updated

Reaccionar a cambios del DOM ya renderizado. Evitar lógica pesada aquí.

DEMO EN VIVO

Demo: Hooks en acción

Componente con un setInterval en mounted y limpieza en beforeDestroy.

<script>
export default {
  data() {
    return { segundos: 0, intervalo: null }
  },
  created() {
    console.log('created: data lista, DOM no existe aún')
  },
  mounted() {
    console.log('mounted: DOM listo')
    this.intervalo = setInterval(() => {
      this.segundos++
    }, 1000)
  },
  beforeDestroy() {
    console.log('beforeDestroy: limpiando...')
    clearInterval(this.intervalo)
  }
}
</script>

Estilos encapsulados — scoped

El atributo scoped limita los estilos al componente actual, evitando colisiones.

Sin scoped (global)

<style>
h2 { color: red; }
/* Afecta a TODOS los h2 */
</style>

Con scoped (aislado)

<style scoped>
h2 { color: red; }
/* Solo afecta a ESTE componente */
</style>

Class Binding — :class

Asigna clases CSS dinámicamente según condiciones reactivas.

<template>
  <!-- Con objeto -->
  <div :class="{ activo: esActivo, error: tieneError }">
    Ejemplo 1
  </div>

  <!-- Con array -->
  <div :class="[claseBase, esGrande ? 'grande' : 'chico']">
    Ejemplo 2
  </div>
</template>

<script>
export default {
  data() {
    return {
      esActivo: true,
      tieneError: false,
      claseBase: 'tarjeta',
      esGrande: true
    }
  }
}
</script>

Style Binding — :style

Aplica estilos en línea reactivamente con objetos JavaScript.

<template>
  <div :style="{ color: colorTexto, fontSize: tamano + 'px' }">
    Texto dinámico
  </div>

  <!-- Con array de objetos -->
  <div :style="[estiloBase, estiloExtra]">
    Combinado
  </div>
</template>

<script>
export default {
  data() {
    return {
      colorTexto: '#2D5BFF',
      tamano: 24,
      estiloBase: { fontWeight: 'bold' },
      estiloExtra: { backgroundColor: '#f0f0f0' }
    }
  }
}
</script>
