ACTIVIDAD 1 — 60 minutos

Correcciones finales y pulido

Instrucciones

1. Revisa los bugs pendientes de la clase anterior y corrígelos.
2. Incorpora el feedback del instructor.
3. Limpia el código: quita console.log, código comentado y funciones sin usar.
4. Mejora la UX: agrega loading, mensajes de error y estados vacíos donde falte.
5. Verifica que la navegación funcione completa.

Prioridades

1. Crítico: Lo que está roto. Si no funciona, arréglalo primero.

2. Feedback: Lo que el instructor señaló.

3. UX: Loading, errores, estados vacíos.

4. Código: Limpieza, nombres claros, estructura.

ACTIVIDAD 2 — 30 minutos

README y documentación final

El README debe tener

1. Nombre del proyecto y descripción breve.
2. Problemática que resuelve.
3. Tecnologías: Vue 2, Vuex, Axios, Firebase, etc.
4. Instalación: npm install → npm run serve.
5. Funcionalidades implementadas.
6. Capturas de pantalla (opcional pero recomendado).
7. Link al deploy si existe.

Decisiones técnicas (agregar al README)

Agrega una sección breve explicando:

🔧 ¿Por qué elegiste esas tecnologías?
📦 ¿Cómo organizaste el proyecto?
🤔 ¿Qué fue lo más difícil?
💡 ¿Qué harías diferente?

Esto demuestra criterio profesional en tu portafolio.

ACTIVIDAD 3 — 30 minutos

Deploy y publicación

Subir a GitHub

1. Crea un repositorio en GitHub.
2. git init → git add . → git commit -m "entrega final"
3. git remote add origin URL
4. git push -u origin main
5. Verifica que el README se vea bien en GitHub.

Deploy (opcional pero recomendado)

Netlify: Conecta GitHub → detecta Vue → deploy automático.

Vercel: Importar repo → configura build → deploy.

Firebase Hosting: firebase init → firebase deploy

Un link en vivo vale más que mil líneas de código.

ACTIVIDAD 4 — 20 minutos

Validación final

Prueba completa como usuario

1. Abre tu app desde cero (o desde el link de deploy).
2. Recorre cada funcionalidad de inicio a fin.
3. Abre la consola (F12) y verifica cero errores.
4. Prueba en otro tamaño de pantalla.
5. Revisa que el README en GitHub se muestre bien.
