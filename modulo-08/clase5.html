Checklist de entrega final

  • Perfil de GitHub con foto, bio y README de perfil
  • Proyecto en repositorio limpio con README completo
  • README con: descripción, tecnologías, instalación, funcionalidades, capturas
  • Deploy funcionando con link público (GitHub Pages o Netlify)
  • Link del deploy en el README y en la bio del perfil
  • Repositorio pinneado en el perfil de GitHub
  • Producto funcional de inicio a fin sin errores en consola
  • Código limpio: sin console.log, sin código comentado
  • Proyecto agregado a LinkedIn (al final de la clase)
ACTIVIDAD 1 — 60 minutos

Completar y pulir el portafolio

Si te faltó de la clase anterior

1. Completar el README de perfil.
2. Terminar el deploy en GitHub Pages o Netlify.
3. Agregar capturas al README del proyecto.
4. Subir el código limpio al repositorio.

Si ya terminaste todo

1. Corregir últimos bugs y warnings de consola.
2. Mejorar UX: loading, errores, estados vacíos.
3. Agregar sección de decisiones técnicas al README.
4. Pinear el repo en tu perfil de GitHub.

Prioridades

1. Deploy funcionando con link.

2. README completo con capturas.

3. Perfil de GitHub profesional.

4. Pulido de bugs y UX.

Si el deploy funciona y el README está completo, ya tienes portafolio.

DESCANSO

Receso — 20 minutos

20:00
Listo para comenzar

Cómo agregar un proyecto a LinkedIn

Sección "Proyectos"

Perfil → Agregar sección → Proyectos. Nombre del proyecto, descripción breve (2-3 líneas), link al deploy, link al repositorio.

Qué escribir

"App web desarrollada con Vue.js, Vuex y Axios que consume la PokéAPI. Incluye sistema de favoritos con persistencia y búsqueda en tiempo real."

Tip: También pueden hacer un post de LinkedIn mostrando el proyecto: "Acabo de terminar este proyecto con Vue.js..." con captura y link. Genera visibilidad.

ACTIVIDAD 2 — 25 minutos

Agregar proyecto a LinkedIn

Instrucciones

1. Abre tu perfil de LinkedIn.
2. Agrega la sección "Proyectos" si no la tienes.
3. Agrega tu proyecto con nombre, descripción y links.
4. Actualiza tu headline si es necesario: "Desarrollador Front-end | Vue.js".
5. Opcional: escribe un post breve compartiendo tu proyecto.

Ejemplo de descripción

"Aplicación web tipo Pokédex desarrollada con Vue.js 2, Vuex para manejo de estado y Axios para consumo de la PokéAPI. Incluye sistema de favoritos con persistencia en localStorage, búsqueda en tiempo real con getters de Vuex y deploy en GitHub Pages."

Tecnologías: Vue.js, Vuex, Axios, PokéAPI, GitHub Pages

ACTIVIDAD 3 — 30 minutos

Validación final y entrega

Prueba completa

1. Abre el link de deploy y recorre toda la app.
2. Abre la consola (F12) y verifica cero errores.
3. Revisa tu perfil de GitHub: ¿se ve profesional?
4. Revisa el README: ¿tiene capturas, link, tecnologías?
5. Revisa LinkedIn: ¿el proyecto está agregado?
