🎨

YouTube

Para demos en video, walkthroughs, tutoriales. Útil si quieren explicar el funcionamiento del producto.

GitHub profesional — Perfil

Tu perfil de GitHub es tu CV técnico. Lo primero que ve un reclutador.

Lo básico

Foto profesional (o avatar decente). Bio con qué haces: "Front-end developer | Vue.js". Ubicación. Link a LinkedIn si tienen.

README de perfil

Repo con tu mismo nombre de usuario. El README.md aparece en tu perfil. Agrega: quién eres, qué tecnologías manejas, proyectos destacados.

# Para crear el README de perfil:
# 1. Crear repo con tu nombre de usuario (ej: juanperez/juanperez)
# 2. Agregar README.md con tu presentación
# 3. El contenido aparece automáticamente en tu perfil

GitHub Pages — Deploy gratuito

Hosting gratuito para sitios estáticos directo desde un repositorio de GitHub.

# Pasos para activar GitHub Pages:

1. Sube tu proyecto a un repositorio en GitHub
2. Ve a Settings → Pages
3. En "Source" selecciona la rama main y carpeta / (root)
4. Click en Save
5. En unos segundos aparece el link:
   https://tu-usuario.github.io/nombre-repo

# Para proyectos Vue (necesitan build):
npm run build
# Subir la carpeta dist/ o configurar GitHub Actions

ACTIVIDAD 1 — 40 minutos

Perfil de GitHub profesional

Instrucciones

1. Actualiza tu perfil de GitHub: foto, bio, ubicación, link a LinkedIn.
2. Crea el repo con tu nombre de usuario para el README de perfil.
3. Escribe el README: quién eres, qué tecnologías manejas, proyectos destacados.
4. Sube tu proyecto principal a un repositorio limpio.
5. Completa el README del proyecto: descripción, tecnologías, instalación, funcionalidades.

Resultado esperado

Al terminar deberías tener:

✅ Perfil de GitHub con foto y bio profesional.
✅ README de perfil visible en tu página de GitHub.
✅ Proyecto subido con README completo.
✅ Código limpio y organizado en el repo.

ACTIVIDAD 2 — 45 minutos

Deploy en GitHub Pages

Instrucciones

1. Haz npm run build de tu proyecto Vue.
2. Sube la carpeta dist/ a un repositorio o rama.
3. Activa GitHub Pages en Settings → Pages.
4. Espera el link público y verifica que funcione.
5. Agrega el link del deploy al README del proyecto.

Alternativa: Si GitHub Pages da problemas con Vue Router, usa Netlify: conecta el repo → deploy automático.

Resultado esperado

✅ Proyecto desplegado con link público.
✅ Link funcionando y accesible.
✅ README actualizado con el link al deploy.

Opcional pero recomendado:
⭐ Agregar el link al perfil de GitHub (bio).
⭐ Pinear el repositorio en tu perfil.

ACTIVIDAD 3 — 25 minutos

Capturas y presentación visual

Instrucciones

1. Toma 3-4 capturas de pantalla de las vistas principales de tu app.
2. Agrégalas al README del proyecto (subirlas a la carpeta del repo o usar enlaces de imagen).
3. Opcional: sube las capturas a Behance con descripción del proyecto.
4. Opcional: graba un video corto (1-2 min) mostrando el flujo principal y súbelo a YouTube como "no listado".

¿Por qué capturas?

Un README con capturas se ve 10x más profesional que uno sin ellas. El reclutador entiende qué hace tu app sin necesidad de instalarla.

Tip: En Chrome, F12 → Toggle device toolbar te permite simular móvil para capturas responsive.
