Bloque 1 · Algoritmos

¿Qué es un Algoritmo?

📋

Definición

Conjunto finito de pasos ordenados y definidos para resolver un problema.

🔑

Propiedades

  • Entrada / Salida
  • Determinismo
  • Finitud
  • Efectividad
💡

Pensamiento Algorítmico

Habilidad para descomponer problemas complejos en pasos manejables.

Bloque 1 · Componentes

Componentes Esenciales

E

Entrada

Datos que el algoritmo necesita para funcionar.

P

Proceso

Pasos y reglas que manipulan los datos.

S

Salida

Resultados generados tras el procesamiento.

V

Variables

Espacios de almacenamiento con valores dinámicos.

Números · Texto · Booleanos — los tres tipos básicos de variables.
Bloque 1 · Estructuras de Control

Estructuras de Control

➡️

Secuencia

Las instrucciones se ejecutan en el orden en que aparecen. Paso a paso, sin saltos.

🔀

Condicionales

Ejecutan diferentes acciones según una condición.
Si … Entonces … Sino

🔁

Repetitivas

Repiten un bloque varias veces.
Mientras · Para · Repetir-Hasta

Dominar estas tres estructuras es la base para cualquier lenguaje de programación.
Bloque 1 · Pseudocódigo

¿Qué es el Pseudocódigo?

📝 Definición

Forma intermedia entre el lenguaje natural y la programación real. Describe la lógica sin depender de sintaxis específica.

🎯 Propósito

  • Representa ideas estructuradas
  • Sin errores de sintaxis
  • Puente al código real
  • Trabajo colaborativo
"Si tu lógica está bien en pseudocódigo, convertirla a código será mucho más fácil."
Bloque 1 · Pseudocódigo

Reglas del Pseudocódigo

ElementoDescripciónEjemplo
Inicio / FinDelimitan el algoritmoInicio … Fin
Palabras claveAcciones básicasLeer · Mostrar · Si · Mientras · Para
AsignaciónGuardar un valorx ← 5 o x = 5
SangríasBloques de código4 espacios por nivel
NombresDescriptivos y sin ambigüedadtotalVentas

Ejemplo: ¿Es Mayor de Edad?

  Leer edad

    Mostrar "Mayor de edad"

    Mostrar "Menor de edad"
  Fin Si

PSeInt – Tu Laboratorio de Algoritmos

🖥️ ¿Qué es?

Herramienta educativa para desarrollar lógica algorítmica con pseudocódigo. Permite escribir, ejecutar y depurar algoritmos sin lenguaje formal.

⚙️ Características

  • Pseudocódigo en español
  • Ejecución paso a paso
  • Estado de variables en vivo
  • Editor de diagramas de flujo
  • Descarga gratis en SourceForge
Modo "Paso a paso" → detecta bucles infinitos y errores lógicos antes de escribir código real.

Bloque 1 · Comparativa

Pseudocódigo vs Diagrama de Flujo

📝 Pseudocódigo

  • Naturaleza textual y detallada
  • Fácil transición a código real
  • Ideal para lógica compleja
  • Énfasis en estructura de control

🗺️ Diagrama de Flujo

  • Vista visual del proceso
  • Detecta errores rápidamente
  • Comunicación con no técnicos
  • Sigue el "camino" del algoritmo
Usar ambas herramientas juntas mejora significativamente la comprensión.

Bloque 3 · Modelos de Prompt

Modelo RTF

🎭

R – Rol

"Actúa como un especialista en…"

📋

T – Tarea

"Escribe / Crea / Genera / Explica…"

📐

F – Formato

"Muestra como tabla / lista / JSON…"

Ejemplo: "Actúa como especialista en testing. Escribe un caso de prueba para un formulario de login. Muéstrame en formato de tabla con columnas: Caso, Entrada y Resultado."
Bloque 3 · Modelos de Prompt

Modelo Gherkin (BDD)

🟢

Given – Dado

Provee el contexto del escenario

🔵

When – Cuando

Especifica los pasos a seguir

🟡

Then – Entonces

Define el resultado esperado

Ideal para automatización de pruebas y Behavior-Driven Development.
Bloque 3 · Fórmula CLEAR

Fórmula CLEAR

( Acción + Contexto + Dolor + Restricciones )
/ Herramienta * Formato
  • Acción: "Genera", "Valida", "Escribe"…
  • Contexto: Describe el caso de uso
  • Dolor: El objetivo específico
  • Restricciones: Limitaciones técnicas
  • Herramienta: Framework/lenguaje
  • Formato: Cómo quieres el resultado

Sobriedad Digital

Construyendo un futuro tecnológico sostenible

Bloque 4 · Concepto

¿Qué es la Sobriedad Digital?

📖 Definición

Estrategia de limitar el consumo de energía, datos y hardware a lo estrictamente necesario para alcanzar los objetivos deseados.

❓ La pregunta clave

"¿Es necesario hacer esto? ¿Hay una forma más sencilla o ligera?"

No se trata de frenar la innovación, sino de encauzarla para que sus beneficios no sean opacados por impactos ambientales.
Bloque 4 · Comparativa

Green IT vs. Sobriedad Digital

ConceptoEnfoquePregunta guía
🌿 Green IT Eficiencia de sistemas existentes "¿Cómo hago lo mismo con menos recursos?"
♻️ Sobriedad Digital Pertinencia y suficiencia "¿Es necesario hacer esto?"
La sobriedad digital también plantea la pertinencia de ciertas funcionalidades: no producir, no almacenar, no transmitir cuando no aporta valor.
Bloque 4 · Importancia

¿Por qué Importa?

🌍

Ambiental

El sector digital supera a la aviación en emisiones de GEI. Reducir la transferencia de datos y prolongar el hardware es crucial.

🤝

Social

Interfaces ligeras mejoran la inclusión digital y reducen la brecha de acceso tecnológico.

💰

Económica

Menos recursos implican menos OPEX y CAPEX, y mayor resiliencia ante cambios normativos.

Bloque 4 · Principios

5 Principios de Diseño Sostenible

1

Eficiencia

Algoritmos de baja complejidad, lazy loading, caché.

2

Sostenibilidad

Lenguajes eficientes, energías renovables en servidores.

3

Accesibilidad

Interfaces ligeras para dispositivos de gama baja.

4

Privacidad

Recopilación mínima de datos y cifrado E2E.

5

Seguridad

Mínimo privilegio y pruebas continuas.
