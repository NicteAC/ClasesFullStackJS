Bloque 1 · ¿Qué es una API?

¿Qué es una API?

📋 Definición

Application Programming Interface: conjunto de reglas que permite la comunicación entre sistemas.

Permite acceder a funcionalidades o datos sin conocer los detalles internos de la implementación.

💡 Analogía

Como el menú de un restaurante: no necesitás saber cómo se cocina el plato. Solo pedís lo que querés y te lo traen en el formato esperado.

La cocina = el servidor. El menú = la API. El resultado = el JSON.

En desarrollo web: el cliente pide datos → el servidor responde con JSON → el cliente los muestra en pantalla.

Bloque 1 · AJAX

AJAX y la Comunicación Cliente-Servidor

1

Cliente pide

El navegador envía una solicitud HTTP al servidor con método GET, POST, etc.

2

Servidor procesa

El servidor recibe la solicitud, consulta la base de datos y prepara la respuesta.

3

Respuesta JSON

El servidor responde con datos en formato JSON (o XML). El cliente los recibe.

4

UI se actualiza

JavaScript procesa el JSON y actualiza el DOM sin recargar la página.

AJAX (Asynchronous JavaScript and XML): técnica que permite actualizar partes de la página sin recargarla completa. XHR y Fetch son las herramientas que lo implementan.
Bloque 1 · XHR vs Fetch

XHR vs Fetch API

📟 XHR — Forma tradicional

const xhr = new XMLHttpRequest();
xhr.open("GET", "https://api.example.com/data");
xhr.onload = function() {
  if (xhr.status === 200) {
    const datos = JSON.parse(xhr.responseText);
    console.log(datos);
  }
};
xhr.onerror = () => console.error("Error");
xhr.send();

✅ Fetch — Forma moderna

fetch("https://api.example.com/data")
  .then(r => r.json())
  .then(datos => console.log(datos))
  .catch(e => console.error(e));

// O con async/await
async function obtener() {
  const r = await fetch("...");
  const datos = await r.json();
}
Usar Fetch para código moderno con Promesas. XHR solo cuando se necesita compatibilidad con IE.

Bloque 2 · Fetch y DOM

Fetch + DOM: Mostrar Datos en Pantalla

// Versión con .then()
document.getElementById("btnCargar").addEventListener("click", () => {
  fetch("https://jsonplaceholder.typicode.com/users")
    .then(r => { if (!r.ok) throw new Error(r.status); return r.json(); })
    .then(users => {
      const lista = document.getElementById("lista");
      lista.innerHTML = "";
      users.forEach(u => {
        const div = document.createElement("div");
        div.innerHTML = `<h3>${u.name}</h3><p>${u.email}</p>`;
        lista.appendChild(div);
      });
    })
    .catch(e => console.error("Error:", e.message));
});
Bloque 2 · async/await

La Misma Función con async/await

async function cargarUsuarios() {
  const lista = document.getElementById("lista");
  lista.innerHTML = "<p>Cargando...</p>"; // loader simple

  try {
    const r = await fetch("https://jsonplaceholder.typicode.com/users");
    if (!r.ok) throw new Error(`Error HTTP: ${r.status}`);

    const users = await r.json();
    lista.innerHTML = "";

    users.forEach(u => {
      const card = document.createElement("div");
      card.className = "user-card";
      card.innerHTML = `<h3>${u.name}</h3><p>${u.email}</p>`;
      lista.appendChild(card);
    });
  } catch (e) {
    lista.innerHTML = `<p style="color:red;">Error: ${e.message}</p>`;
  }
}
document.getElementById("btnCargar").addEventListener("click", cargarUsuarios);

Bloque 3 · APIs de Terceros

¿Qué son las APIs de Terceros?

🌍 Definición

Servicios desarrollados por empresas externas que exponen sus funcionalidades a través de una API.

Permiten integrar funcionalidades complejas en nuestra app sin desarrollarlas desde cero: clima, mapas, pagos, traducción.

🏢 Ejemplos reales

  • OpenWeather — datos del clima en tiempo real
  • Google Maps — mapas, rutas, lugares
  • Stripe — pagos con tarjeta de crédito
  • GitHub API — repositorios, usuarios, commits
  • Spotify API — canciones, álbumes, playlists
Bloque 3 · API Key OpenWeather

Obtener API Key — OpenWeather

1

Ir a openweathermap.org → Sign In → Create an Account. Completar nombre, email y contraseña.

2

Confirmar el email recibido en la bandeja de entrada para activar la cuenta.

3

Iniciar sesión → clic en el nombre de usuario (arriba a la derecha) → My API Keys.

4

Copiar la key del campo Default. Es un string como a1b2c3d4e5f6789...

5

⏳ Esperar 10 min a 2 horas para que se active. Probar en el navegador con: api.openweathermap.org/data/2.5/weather?q=Santiago&appid=TU_KEY&units=metric

Bloque 3 · OpenWeather

Anatomía de la URL de OpenWeather

https://api.openweathermap.org/data/2.5/weather
  ?q=Santiago ← nombre de la ciudad
  &appid=TU_API_KEY ← tu clave de autenticación
  &units=metric ← °C (metric) / °F (imperial)
  &lang=es ← descripción en español
El JSON de respuesta tiene: data.main.temp · data.weather[0].description · data.wind.speed · data.name · data.sys.country

Errores y Límites

Status codes · Manejo de errores · Rate limiting

Bloque 4 · Status codes

Códigos de Estado HTTP

200

OK — La solicitud fue exitosa. Datos disponibles.

201

Created — Recurso creado correctamente (POST exitoso).

400

Bad Request — La solicitud tiene errores de formato o datos inválidos.

401

Unauthorized — API Key faltante o incorrecta.

403

Forbidden — Autenticado pero sin permiso para ese recurso.

404

Not Found — La ciudad o recurso no existe.

429

Too Many Requests — Superaste el límite de peticiones (rate limiting).

500

Internal Server Error — Problema en el servidor de la API.

Bloque 4 · Manejo de errores

Manejo de Errores en APIs

async function buscarClima(ciudad) {
  const resultado = document.getElementById("resultado");
  const url = `https://api.openweathermap.org/data/2.5/weather?q=${ciudad}&appid=${API_KEY}&units=metric&lang=es`;

  try {
    const r = await fetch(url);

    // Manejar errores específicos por código
    if (r.status === 404) { resultado.innerHTML = "<p>❌ Ciudad no encontrada</p>"; return; }
    if (r.status === 401) { resultado.innerHTML = "<p>🔑 API Key inválida</p>"; return; }
    if (r.status === 429) { resultado.innerHTML = "<p>⏳ Límite de peticiones alcanzado</p>"; return; }
    if (!r.ok) throw new Error(`Error ${r.status}`);

    const data = await r.json();
    resultado.innerHTML = `<h2>${data.name}</h2><p>${data.main.temp}°C · ${data.weather[0].description}</p>`;

  } catch (e) {
    resultado.innerHTML = `<p>❌ Error de conexión: ${e.message}</p>`;
  }
}
Bloque 4 · Límites

Límites y Restricciones en las APIs

⏱️ Tipos de restricciones

  • Rate limiting: máximo de peticiones por tiempo (ej. 60/hora)
  • Límite de datos: archivos no mayores a X MB
  • Endpoints bloqueados: features solo en plan pago
  • CORS: acceso restringido por dominio de origen
GitHub sin auth: 60 req/hora · Con OAuth: 5000 req/hora

🔄 Estrategia de reintento

async function fetchConReintento(url, intentos = 3) {
  for (let i = 0; i < intentos; i++) {
    try {
      const r = await fetch(url);
      if (r.ok) return r.json();
    } catch (e) {
      if (i === intentos - 1) throw e;
      await new Promise(r => setTimeout(r, 1000));
    }
  }
}
