Bloque 1 · ES6+

¿Qué es ES6+?

📜 Historia rápida

  • 1995: JavaScript creado por Brendan Eich
  • 1997: Estandarizado como ECMAScript (ES1)
  • 2009 (ES5): strict mode, JSON.parse, Object.create
  • 2015 (ES6): Gran salto — let, const, clases, arrow functions, módulos…
  • ES6+: actualizaciones anuales desde 2016

⚡ Principales mejoras ES6

  • let y const — scope de bloque
  • Arrow functions =>
  • Template literals
  • Destructuring
  • Módulos import/export
  • Clases y herencia
  • Promesas y async/await
ES6 fue lanzado en 2015 y marcó un antes y un después. Desde entonces: actualizaciones anuales con mejoras adicionales.
Bloque 1 · Compatibilidad

Compatibilidad en Navegadores

✅ Soporte actual

La mayoría de navegadores modernos soportan ES6+ completamente: Chrome, Firefox, Safari, Edge.

Características como let, const, arrow functions y promesas tienen soporte universal en navegadores modernos.

⚠️ Problema con IE

Internet Explorer no soporta ES6. Algunos dispositivos antiguos también pueden tener problemas.

Para verificar soporte: caniuse.com y MDN Web Docs.

Demo: ir a caniuse.com y buscar arrow functions — ver soporte por navegador y versión.
Bloque 1 · Herramientas

Babel — Transpilador ES6 → ES5

🔄 ¿Qué hace?

Convierte código ES6+ a ES5 compatible con navegadores antiguos. Permite usar todas las nuevas características sin preocuparse por compatibilidad.

Demo en línea: babeljs.io/repl

💻 Ejemplo de transpilación

// ES6 (input)
const sumar = (a, b) => a + b;
↓ Babel
// ES5 (output)
var sumar = function(a, b) {
  return a + b;
};

Bloque 2 · ES6 práctico

Variables, Arrow Functions y Template Literals

// let: reasignable · const: no reasignable · scope de bloque
let edad = 25; edad = 26; // ✅
const PI = 3.1416; // PI = 3 → ❌ Error

// Arrow function: sintaxis corta, sin this propio
const sumar = (a, b) => a + b; // retorno implícito
const cuadrado = n => n ** 2; // un parámetro

// Parámetros por defecto
const saludar = (nombre = "invitado") => `Hola, ${nombre}!`;

// Template literals: backticks + ${ expresión }
const nombre = "Ana"; const edad2 = 25;
console.log(`${nombre} tiene ${edad2} años`); // "Ana tiene 25 años"
Bloque 2 · ES6 práctico

Destructuring · Spread · Rest

// DESTRUCTURING de objetos
const usuario = { nombre: "Carlos", edad: 30, pais: "Chile" };
const { nombre, edad } = usuario; // extrae solo nombre y edad

// DESTRUCTURING de arrays
const [primero, segundo] = [10, 20, 30];

// SPREAD: expandir / combinar
const nums = [1, 2, 3];
const mas = [...nums, 4, 5]; // [1,2,3,4,5]
const copia = { ...usuario, activo: true }; // nuevo obj con propiedad extra

// REST: agrupar argumentos variables
const sumarTodo = (...n) => n.reduce((a,b) => a+b, 0);
console.log(sumarTodo(1,2,3,4)); // 10
Bloque 2 · ES6 práctico

Módulos · Set · Map · Async/Await

// MÓDULOS: export en saludo.js
export const saludar = () => "¡Hola!";
// import en app.js
import { saludar } from './saludo.js';

// SET: colección de valores únicos
const set = new Set([1,2,3,3]); // Set {1,2,3}

// MAP: pares clave-valor con cualquier tipo de clave
const mapa = new Map();
mapa.set("nombre", "Lucía");
console.log(mapa.get("nombre")); // "Lucía"

// ASYNC/AWAIT: asincronía legible
async function obtenerDatos() {
  try {
    const res = await fetch("https://api.example.com/data");
    const datos = await res.json();
    console.log(datos);
  } catch (e) { console.error(e); }
}

Bloque 3 · DOM

¿Qué es el DOM?

📄 Definición

Interfaz de programación que representa la estructura de un documento HTML como un árbol jerárquico. Permite a JavaScript acceder y modificar la página.

💡 Permite

  • Modificar contenido sin recargar la página
  • Agregar y eliminar elementos HTML dinámicamente
  • Cambiar estilos CSS en respuesta a eventos
  • Escuchar clics, teclas, scroll, formularios
// Árbol DOM: document → html → head/body → elementos
// Tipos de nodos:
// - Nodo raíz: document
// - Nodos elemento: <body>, <h1>, <p>
// - Nodos texto: el contenido dentro de los elementos
// - Nodos atributo: class, id, src, href
Bloque 3 · Jerarquía

Jerarquía y Relaciones del DOM

🔗 Relaciones entre nodos

  • parentNode → nodo padre
  • children → nodos hijos
  • nextElementSibling → hermano siguiente
  • previousElementSibling → hermano anterior

💻 Código

// HTML: <ul><li>A</li><li id="b">B</li><li>C</li></ul>
const b = document.getElementById("b");
b.parentNode; // <ul>
b.nextElementSibling; // <li>C</li>
b.previousElementSibling; // <li>A</li>
Bloque 3 · Selectores

Seleccionar Elementos del DOM

// Por ID → un solo elemento
document.getElementById("titulo");

// Por clase → colección HTMLCollection
document.getElementsByClassName("descripcion")[0];

// Por etiqueta → todos los <li>
document.getElementsByTagName("li");

// querySelector → primer elemento que coincide (CSS selector)
document.querySelector("#miId"); // por ID
document.querySelector(".miClase"); // por clase
document.querySelector("ul li:first-child"); // selector complejo

// querySelectorAll → NodeList con TODOS los que coinciden
document.querySelectorAll(".elemento")
  .forEach(el => console.log(el.textContent));
Preferir siempre querySelector y querySelectorAll — más flexibles y usan selectores CSS estándar.
Bloque 3 · Modificar

Modificar Contenido y Atributos

// CONTENIDO
elem.textContent = "Nuevo texto"; // solo texto (seguro)
elem.innerHTML = "<strong>Negrita</strong>"; // HTML (cuidado con XSS)

// ATRIBUTOS directamente
document.getElementById("img").src = "nueva.jpg";

// setAttribute / getAttribute (para cualquier atributo, incluyendo data-*)
elem.setAttribute("href", "https://ejemplo.com");
elem.setAttribute("target", "_blank");
const url = elem.getAttribute("href");

// ESTILOS y CLASES
elem.style.color = "red";
elem.classList.add("activo"); // agregar clase
elem.classList.remove("activo"); // quitar clase
elem.classList.toggle("oscuro"); // alternar clase
Bloque 3 · Agregar y eliminar

Agregar y Eliminar Elementos

// CREAR y AGREGAR
const parrafo = document.createElement("p");
parrafo.textContent = "Nuevo párrafo";
document.body.appendChild(parrafo); // al final del body

// Dentro de un contenedor específico
document.getElementById("contenedor").appendChild(parrafo);

// Insertar ANTES de otro elemento
const ref = document.getElementById("referencia");
document.body.insertBefore(parrafo, ref);

// ELIMINAR directamente
document.getElementById("miDiv").remove();

// Eliminar un hijo desde su padre
const padre = document.getElementById("lista");
padre.removeChild(document.getElementById("item2"));

Bloque 4 · Eventos

¿Qué son los Eventos?

🎯 Definición

Acciones o sucesos que ocurren en el navegador y pueden ser detectados y manejados con JavaScript. Permiten reaccionar a acciones del usuario.

📋 Tipos de eventos

  • Mouse: click, dblclick, mouseover, mouseout
  • Teclado: keydown, keyup, keypress
  • Formulario: submit, change, focus, blur, input
  • Ventana: load, resize, scroll
  • DOM: DOMContentLoaded
Bloque 4 · Manejo de eventos

Formas de Manejar Eventos

// ❌ Atributo HTML (no recomendado: mezcla HTML y JS)
// <button onclick="saludar()">Click</button>

// ⚠️ Propiedad del DOM (solo permite un manejador por evento)
document.getElementById("btn").onclick = function() {
  console.log("Click!");
};

// ✅ addEventListener (recomendado: múltiples manejadores, más control)
const btn = document.getElementById("btn");

btn.addEventListener("click", () => {
  console.log("Primer manejador");
});
btn.addEventListener("click", () => {
  console.log("Segundo manejador"); // ambos se ejecutan ✅
});
Bloque 4 · Modelo de eventos

Event Bubbling y Capturing

⬆️ Bubbling (default)

El evento se propaga desde el elemento objetivo hacia sus elementos padres. Click en <li> → sube a <ul> → sube a <div>…

// Detener la propagación
elem.addEventListener("click", (e) => {
  e.stopPropagation(); // no sube más
});

⬇️ Capturing

El evento se propaga desde el raíz hacia el elemento objetivo. Se activa con tercer parámetro true.

// Prevenir comportamiento default
form.addEventListener("submit", (e) => {
  e.preventDefault(); // no envía el form
});
stopPropagation() detiene el burbujeo · preventDefault() cancela la acción por defecto del navegador
Bloque 4 · Delegación

Delegación de Eventos

// ❌ Sin delegación: un listener por elemento
document.querySelectorAll("li").forEach(li => {
  li.addEventListener("click", (e) => e.target.remove());
}); // no funciona con elementos creados dinámicamente

// ✅ Con delegación: UN listener en el padre maneja todos los hijos
document.getElementById("lista").addEventListener("click", (e) => {
  if (e.target.tagName === "LI") { // filtrar el elemento correcto
    e.target.remove();
  }
});

// También funciona con clases
document.getElementById("lista").addEventListener("click", (e) => {
  if (e.target.classList.contains("eliminar")) {
    e.target.parentElement.remove();
  }
});
La delegación de eventos es esencial para elementos creados dinámicamente. Reduce listeners y mejora el rendimiento.
