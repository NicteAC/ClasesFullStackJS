¿Qué es un Arreglo?

Un arreglo es una estructura de datos que almacena múltiples valores en una secuencia indexada.

Características:

  • Los índices comienzan en 0
  • Son mutables (se pueden modificar)
  • Pueden contener cualquier tipo de dato
  • Organizados en posiciones numeradas

¿Para qué se usan?

  • Almacenar listas de usuarios
  • Gestionar carritos de compras
  • Procesar datos dinámicos
  • Iterar sobre elementos de forma eficiente

Arreglos vs Objetos en JavaScript

Arreglos

  • Índices numéricos secuenciales
  • Orden garantizado
  • Ideal para colecciones homogéneas
  • Métodos: push(), pop(), map(), filter()

Objetos

  • Claves de texto (strings)
  • Sin orden garantizado
  • Ideal para datos estructurados
  • Pares clave-valor

Dato clave: Los arreglos son un tipo especial de objeto en JavaScript

Ciclos de Repetición

Herramientas para ejecutar código repetidamente

Ciclo for

Ideal cuando conoces el número exacto de repeticiones

for (let i = 0; i < 5; i++) {
  console.log(i);
}

Ciclo while

Útil cuando no sabes cuántas repeticiones necesitas

let i = 0;
while (i < 5) {
  console.log(i);
  i++;
}

Métodos de Iteración: forEach() y map()

forEach()

Ejecuta una función para cada elemento. No retorna nada.

const frutas = ["🍎", "🍌", "🍊"];
frutas.forEach(fruta => {
  console.log(fruta);
});

map()

Transforma cada elemento y retorna un nuevo arreglo.

const nums = [1, 2, 3];
const dobles = nums.map(n => n * 2);
// [2, 4, 6]

Regla de oro: Usa map() cuando necesites transformar datos, forEach() solo para ejecutar acciones

¿Qué es una Función?

Un bloque de código reutilizable que realiza una tarea específica

Analogía: Receta de Cocina

  • Entrada: Ingredientes (parámetros)
  • Proceso: Pasos de la receta (código)
  • Salida: Plato terminado (return)

Ventajas

  • Reutilización de código
  • Facilita el mantenimiento
  • Código más legible
  • Reducción de errores

Tres Formas de Definir Funciones

1. Función Declarada

function saludar(nombre) {
  return `Hola, ${nombre}!`;
}

✓ Hoisting habilitado

2. Función Anónima

const saludar = function(nombre) {
  return `Hola, ${nombre}!`;
};

✓ Útil para callbacks

3. Arrow Function (función flecha) - ES6+

const saludar = (nombre) => `Hola, ${nombre}!`;

✓ Sintaxis corta, ideal para programación funcional

Parámetros y Valores de Retorno

Sin return

Solo ejecuta acciones

function saludar(nombre) {
  console.log("Hola " + nombre);
}
saludar("Ana"); // Imprime
let x = saludar("Ana");
// x es undefined

Con return

Devuelve un valor reutilizable

function sumar(a, b) {
  return a + b;
}
let resultado = sumar(5, 3);
// resultado es 8

Importante: Si no hay return, la función retorna undefined

Buenas Prácticas con Funciones

  • Nombres descriptivos: calcularTotal() mejor que calc()
  • Una sola responsabilidad: Cada función hace una cosa bien
  • Evitar funciones largas: Máximo 20-30 líneas
  • No modificar variables globales: Usa parámetros y return
  • Documentar parámetros: Qué espera recibir y qué devuelve
  • Validar entradas: Verificar que los parámetros sean válidos
