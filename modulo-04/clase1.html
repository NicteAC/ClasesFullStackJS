Bloque 1 · POO

¿Qué es la POO?

🎯 Definición

Paradigma de programación que organiza el código en objetos, modelando entidades del mundo real con sus datos y comportamientos.

Objetivo: acercar el software a cómo pensamos e interactuamos con el mundo real.

💡 ¿Por qué usarla?

  • Mejor organización del código
  • Mayor reutilización de funcionalidades
  • Facilita el mantenimiento a largo plazo
  • Escala mejor en proyectos grandes
  • Base de frameworks modernos: React, Angular, Vue
La POO es el paradigma dominante en el desarrollo de software moderno. JavaScript la soporta completamente desde ES6.
Bloque 1 · Paradigmas

Paradigmas de Programación

📋 Imperativa

Instrucciones paso a paso: "haz esto, luego esto". El más cercano a cómo piensa la máquina.

Ejemplos: C, Pascal

📢 Declarativa

Describe qué se quiere en lugar de cómo lograrlo. Más abstracto.

Ejemplos: SQL, HTML

λ Funcional

Basado en funciones matemáticas puras, sin efectos secundarios. Favorece la inmutabilidad.

Ejemplos: Haskell, Lisp

🧩 Orientada a Objetos

Modela entidades reales mediante clases y objetos. Organiza el código en torno a objetos que interactúan.

Ejemplos: Java, Python, JavaScript

Bloque 1 · Conceptos

Conceptos Clave de la POO

📋

Clase

Plantilla o molde para crear objetos. Define qué propiedades y métodos tendrán.

📦

Objeto

Instancia concreta de una clase. Tiene valores reales en sus propiedades.

🏷️

Propiedad

Variable dentro de un objeto. Describe sus características.

⚙️

Método

Función dentro de un objeto. Define sus comportamientos y acciones.

✨

Instancia

Cada objeto creado a partir de una clase es una instancia única de ella.

class Persona { // ← Clase (plantilla)
  constructor(nombre, edad) {
    this.nombre = nombre; // ← Propiedad
    this.edad = edad;
  }
  saludar() { return `Hola, soy ${this.nombre}`; } // ← Método
}
const persona1 = new Persona("Sofía", 25); // ← Instancia / Objeto
Bloque 1 · Los 4 pilares

Los 4 Pilares de la POO

🔒

Encapsulamiento

Protege los datos internos. Solo expone lo necesario.

🧬

Herencia

Una clase hija reutiliza propiedades y métodos de la clase padre.

🎭

Polimorfismo

Una clase hija redefine métodos del padre con su propia implementación.

🎯

Abstracción

Oculta la complejidad interna. Solo muestra lo esencial.

Ejemplo del auto: Encapsulamiento → motor oculto · Herencia → camión hereda de Vehículo · Polimorfismo → cada vehículo acelera diferente · Abstracción → solo ves el volante
Bloque 1 · Encapsulamiento

Encapsulamiento en JavaScript

class CuentaBancaria {
  #saldo = 0; // propiedad privada con #

  constructor(titular) {
    this.titular = titular;
  }

  depositar(monto) {
    if (monto > 0) this.#saldo += monto;
  }

  getSaldo() { // getter: única forma de leer el saldo
    return this.#saldo;
  }
}

const cuenta = new CuentaBancaria("Ana");
cuenta.depositar(1000);
console.log(cuenta.getSaldo()); // 1000
console.log(cuenta.#saldo); // ❌ SyntaxError: campo privado
Bloque 1 · Herencia y Polimorfismo

Herencia y Polimorfismo

class Vehiculo {
  constructor(marca) { this.marca = marca; }
  info() { return `Vehículo: ${this.marca}`; } // método base
}

class Auto extends Vehiculo { // ← herencia
  constructor(marca, puertas) {
    super(marca); // llama al constructor padre
    this.puertas = puertas;
  }
  info() { return `Auto: ${this.marca}, ${this.puertas} puertas`; } // ← polimorfismo
}

class Moto extends Vehiculo {
  constructor(marca, cc) { super(marca); this.cc = cc; }
  info() { return `Moto: ${this.marca}, ${this.cc}cc`; } // ← polimorfismo
}

const a = new Auto("Toyota", 4); a.info(); // "Auto: Toyota, 4 puertas"
const m = new Moto("Yamaha", 600); m.info(); // "Moto: Yamaha, 600cc"
Bloque 2 · Crear objetos

3 Formas de Crear Objetos en JavaScript

1️⃣ Literal {}

Más simple. No reutilizable para múltiples instancias.

const perro = {
  nombre: "Rex",
  ladrar() {
    return "¡Guau!";
  }
};

2️⃣ Constructor

Previa a ES6. Permite múltiples instancias con new.

function Perro(nombre) {
  this.nombre = nombre;
}
const p = new Perro("Rex");

3️⃣ Clase ES6 ✅

La forma moderna. Más legible y organizada.

class Perro {
  constructor(nombre) {
    this.nombre = nombre;
  }
}
const p = new Perro("Rex");
En código moderno: siempre usar clases ES6. Los otros dos métodos se ven en código legado.
Bloque 2 · Prototype

Prototype — Herencia en JavaScript

🔗 ¿Qué es?

Mecanismo interno de JavaScript que permite a los objetos heredar propiedades y métodos de otros objetos. Cada función tiene una propiedad prototype.

Las clases ES6 usan prototype internamente de forma automática.

💡 ¿Para qué sirve?

Permite que todos los objetos creados con una función constructora compartan métodos sin duplicarlos en memoria.

function Animal(nombre) {
  this.nombre = nombre;
}
// Método compartido vía prototype
Animal.prototype.hablar = function() {
  return `${this.nombre} habla`;
};

Bloque 3 · Prototype código

Prototype Manual vs Clase ES6

⚙️ Con prototype manual

function Empleado(nombre, cargo) {
  this.nombre = nombre;
  this.cargo = cargo;
}
// Método en prototype (compartido)
Empleado.prototype.info = function() {
  return `${this.nombre} - ${this.cargo}`;
};
const e1 = new Empleado("Ana", "Dev");

✅ Con clase ES6 (equivalente)

class Empleado {
  constructor(nombre, cargo) {
    this.nombre = nombre;
    this.cargo = cargo;
  }
  // JS lo agrega al prototype automáticamente
  info() {
    return `${this.nombre} - ${this.cargo}`;
  }
}
Bloque 3 · JSON

¿Qué es JSON?

📄 Definición

JavaScript Object Notation. Formato ligero de intercambio de datos. Es un string con estructura de objeto.

  • Se usa para enviar datos entre cliente y servidor
  • Formato estándar de las APIs REST
  • Soportado por todos los lenguajes modernos

📐 Reglas de sintaxis

  • Las claves siempre van entre comillas dobles
  • Valores: string, number, boolean, array, object, null
  • Sin funciones ni undefined
  • Sin comentarios
  • Sin comas al final del último elemento
{ "nombre": "Ana", "edad": 25, "activo": true, "cursos": ["JS", "CSS"] }
Bloque 3 · JSON

JSON.stringify() y JSON.parse()

const usuario = {
  nombre: "Sofía",
  edad: 25,
  cursos: ["JavaScript", "CSS"]
};

// Objeto → JSON (string)
const json = JSON.stringify(usuario);
console.log(typeof json); // "string"
console.log(json); // '{"nombre":"Sofía","edad":25,"cursos":["JavaScript","CSS"]}'

// JSON (string) → Objeto
const obj = JSON.parse(json);
console.log(typeof obj); // "object"
console.log(obj.nombre); // "Sofía"
console.log(obj.cursos[0]); // "JavaScript"
stringify: objeto → string JSON para enviar · parse: string JSON → objeto para usar en código

Bloque 4 · Ventajas

¿Por qué JSON domina la web?

⚡ Ligero y rápido

Formato muy compacto. Los datos son fáciles de transportar, especialmente en aplicaciones web y móviles donde el ancho de banda importa.

👁️ Legible

Estructura clara que los humanos pueden leer y escribir directamente. Facilita la depuración y revisión de datos.

🌍 Universal

Soportado por prácticamente todos los lenguajes: Python, Java, PHP, Ruby, Go. Permite integración entre sistemas distintos.

🔗 Interoperable

Estándar para APIs REST. Cuando un servidor envía datos a un cliente, casi siempre lo hace en JSON.
