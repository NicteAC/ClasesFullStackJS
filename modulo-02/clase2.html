🌐 ¿Qué es CSS?

CSS (Cascading Style Sheets) define cómo se muestran los elementos HTML: diseño, colores, formato y disposición.

Permite una separación clara entre contenido (HTML) y diseño (CSS), resultando en sitios más atractivos y mantenibles.

Puntos clave:

  • Reglas CSS: selector + propiedades + valores
  • Selectores: eligen qué elementos estilizar
  • Cascada: estilos pueden heredarse y sobreescribirse
  • Enlace externo: <link> en el <head>
/* Anatomía de una regla CSS */
selector {
  propiedad: valor;
}

/* Ejemplo real */
h1 {
  color: #667eea;
  font-size: 2rem;
  text-align: center;
}

/* Vinculación en HTML */
<link rel="stylesheet" href="styles.css">

📄 Tipos de Estilos CSS

⚡ En Línea

Dentro del atributo style="" del elemento.

<p style="color:red;">
Texto</p>

⚠️ Dificulta el mantenimiento

📝 Embebido

Dentro de <style> en el <head>.

<style>
 p { color: blue; }
</style>

🟡 Útil para páginas únicas

✅ Archivo Externo

Archivo .css separado, vinculado con <link>.

<link rel="stylesheet"
  href="styles.css">

🌟 ¡Mejor práctica!

🎯 Selectores CSS

Por Etiqueta

Selecciona todos los elementos de ese tipo en la página.

p {
 color: blue;
 font-weight: bold;
}

Afecta a todos los <p>

Por Clase .clase

Reutilizable en múltiples elementos.

.destacado {
 color: blue;
 font-weight: bold;
}

Múltiples elementos pueden usarla

Por ID #id

Para un único elemento específico.

#miTitulo {
 color: blue;
 font-size: 16px;
}

Solo para elementos únicos

🔤 Tipografía con CSS

.texto {
 font-family: Arial, sans-serif;
 font-size: 16px;
 font-weight: bold; /* normal | bold | 100-900 */
 line-height: 1.8;
 text-align: center; /* left | right | justify */
 color: #ffffff;
}

font-family: Define el tipo de letra. Siempre listar varias como fallback.

font-size: Tamaño: px (fijo), em (relativo al padre), rem (relativo a raíz — mejor para responsividad).

font-weight: Grosor del texto. normal, bold o valores del 100 al 900.

line-height: Espaciado entre líneas. Entre 1.5 y 1.8 es estándar de legibilidad.

⚖️ Orden Jerárquico en CSS

Selectores de menor a mayor peso:

Etiquetas: p, div, h1, ::beforeMenor especificidad
Clases, atributos, pseudoclases: .clase, [attr], :hover
IDs: #id
Estilos en línea: style=""
!important — ¡Máxima prioridad!⚠️ Usar con precaución

Si dos reglas tienen la misma especificidad, gana la que aparece última en el archivo CSS.

🌊 Cascada, Especificidad y Herencia

🌊 Cascada

Estilos aplicados en orden de aparición. Última regla con igual especificidad prevalece.

p { color: red; }
p { color: blue; } ← gana

📐 Especificidad

Sistema de puntuación. Mayor especificidad = mayor prioridad.

.clase1.clase2 {
 color: blue; ← gana
}
p { color: orange; }

🧬 Herencia

Propiedades como color y font-family se heredan de padres a hijos.

body {
 font-family: Arial;
}
/* Todos los hijos heredan */

📱 Diseño Responsivo

El sitio se adapta automáticamente al tamaño de la pantalla.

📲

Móvil

Pantallas pequeñas, orientación vertical. Touch.

≤ 767px

📱

Tablet

Tamaño intermedio. Puede rotar entre vertical y horizontal.

768px – 1023px

💻

Desktop

Pantallas grandes, orientación horizontal. Teclado y mouse.

≥ 1024px

Diseño adaptable: reorganizar elementos, ajustar fuentes, ocultar o mostrar contenido según el dispositivo.

Performance: optimizar recursos para distintas velocidades de conexión y densidades de pantalla.

📲 Mobile First

Diseñar primero para móvil, luego escalar hacia pantallas más grandes.

¿Por qué Mobile First?

  • Simplificación: diseño minimalista, contenido esencial primero
  • Mejora progresiva: más elementos al agrandar la pantalla
  • Performance: carga más rápida en conexiones lentas
  • Estándar actual: más del 60% del tráfico viene de móvil
/* Estilos BASE para móvil */
body { font-size: 14px; }
.nav { flex-direction: column; }

/* Escalar con min-width → tablet */
@media (min-width: 768px) {
 body { font-size: 16px; }
 .nav { flex-direction: row; }
}

/* Escalar con min-width → desktop */
@media (min-width: 1024px) {
 body { font-size: 18px; }
}

📡 Media Queries

Aplicar estilos según características del dispositivo.

⚠️ Obligatorio en el <head>:

<meta name="viewport"
 content="width=device-width, initial-scale=1.0">

Sin este meta tag, las media queries no funcionan en móvil.

Condiciones comunes

  • max-width: aplica hasta ese ancho
  • min-width: aplica desde ese ancho
  • and: combina dos condiciones
/* Móvil: ≤ 767px */
@media (max-width: 767px) {
 header { font-size: 20px; }
 nav ul { flex-direction: column; }
}

/* Tablet: 768px – 1023px */
@media (min-width: 768px) and (max-width: 1023px) {
 header { font-size: 24px; }
}

/* Desktop: ≥ 1024px */
@media (min-width: 1024px) {
 header { font-size: 28px; }
}
