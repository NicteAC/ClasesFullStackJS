¿Qué es el Desarrollo Web?

El desarrollo web es el proceso de crear y mantener sitios web y aplicaciones web que funcionan en internet.

Incluye

  • Diseño de interfaces
  • Programación funcional
  • Gestión de bases de datos
  • Seguridad y rendimiento

Resultado

  • Sitios web informativos
  • Tiendas online
  • Redes sociales
  • Aplicaciones web complejas

Componentes Principales

🖥️ Cliente

El navegador web que usa el usuario para acceder a sitios web

⚙️ Servidor

Computadora que almacena y procesa los archivos del sitio web

🗄️ Base de Datos

Sistema que almacena información de forma organizada (usuarios, productos, contenido, etc.)

💡 Metáfora del Restaurante: Cliente = comensal, Servidor = cocina, Base de datos = despensa

Front-End Development

Todo lo que el usuario VE y con lo que INTERACTÚA

Tecnologías

  • HTML (estructura)
  • CSS (diseño y estilo)
  • JavaScript (interactividad)

Responsabilidades

  • Diseño responsivo (UX/UI)
  • Integración con APIs
  • Accesibilidad web
  • Compatibilidad navegadores
  • Optimización rendimiento

Back-End Development

Todo lo que sucede "detrás de escena" en el servidor

Lenguajes

  • PHP
  • Python
  • Ruby
  • Java
  • Node.js

Responsabilidades

  • Lógica del negocio
  • Gestión de bases de datos
  • APIs y servicios web
  • Autenticación y seguridad
  • Escalabilidad

Fullstack Development

Front-End + Back-End = Fullstack

Un desarrollador fullstack puede trabajar en ambos lados del desarrollo web

Ventajas

  • Visión completa del proyecto
  • Mayor flexibilidad
  • Mejor comunicación en equipos

Desafíos

  • Más tecnologías que dominar
  • Aprendizaje continuo
  • Mantenerse actualizado

Front-End vs Back-End

Front-End

  • Lado del cliente
  • Visual e interactivo
  • HTML, CSS, JavaScript
  • Lo que el usuario ve
  • Diseño y experiencia

Back-End

  • Lado del servidor
  • Lógica y datos
  • PHP, Python, Node.js...
  • Lo que el usuario no ve
  • Funcionalidad y seguridad

¿Qué es Internet?

Una red global de computadoras interconectadas que se comunican usando protocolos estándar

Protocolo TCP/IP

Sistema de reglas que permite que las computadoras se comuniquen entre sí

Internet ≠ World Wide Web

Internet es la infraestructura física. La Web es un servicio que funciona sobre Internet

Navegadores Web

Aplicaciones que permiten acceder y visualizar páginas web

Principales Navegadores

  • Google Chrome
  • Mozilla Firefox
  • Microsoft Edge
  • Safari
  • Opera

Componentes

  • Botones de navegación
  • Barra de dirección (URL)
  • Área de visualización
  • Herramientas de desarrollo

Buscadores (Motores de Búsqueda)

Sistemas que indexan páginas web y permiten encontrar información específica

¿Cómo funcionan?

  • Rastreo: "Arañas" recorren la web siguiendo enlaces
  • Indexación: Almacenan y organizan la información encontrada
  • Búsqueda: Algoritmos encuentran las páginas más relevantes
  • Ranking: Ordenan resultados por relevancia

Ejemplos: Google, Bing, DuckDuckGo, Yahoo

Evolución de HTML

  • HTML 1-2 (1991): Texto básico, imágenes simples, enlaces
  • HTML 3 (1995): Tablas, formularios (nunca fue estándar oficial)
  • HTML 4 (1998): Primera vez que aparece CSS separado, estándar W3C
  • XHTML 1.0 (2000): Basado en XML, versiones Transitional y Strict
  • HTML5 (2004-2008): WHATWG inicia desarrollo, W3C se une en 2008

HTML5 es el estándar actual y continúa evolucionando

HTML - HyperText Markup Language

Lenguaje de marcado que define la estructura y contenido de las páginas web

Conceptos Clave

  • Etiquetas: Definen elementos (<h1>, <p>, <img>)
  • Atributos: Información adicional (class, id, src)
  • Elementos: Etiqueta + contenido + cierre
  • Estructura: Jerarquía anidada

Ejemplo:

<!-- Estructura básica --> <h1 class="titulo">Bienvenidos</h1> <p id="intro"> Este es un párrafo de ejemplo. </p> <img src="foto.jpg" alt="Descripción">

💡 Analogía: HTML es el esqueleto de la página

CSS - Cascading Style Sheets

Lenguaje que controla la presentación visual de las páginas web

¿Qué controla CSS?

  • Colores y fondos
  • Tipografía (fuentes, tamaños)
  • Espaciado y márgenes
  • Posicionamiento
  • Animaciones y transiciones
  • Diseño responsive

Ejemplo:

/* Estilos para el título */ .titulo { color: #2D5BFF; font-size: 32px; text-align: center; margin-bottom: 20px; } #intro { background: #f0f0f0; padding: 15px; border-radius: 8px; }

💡 Analogía: CSS es la piel, ropa y maquillaje

JavaScript - El Lenguaje de la Web

Lenguaje de programación que añade interactividad y funcionalidad dinámica

Capacidades

  • Manipular el contenido (DOM)
  • Responder a eventos (clicks)
  • Validar formularios
  • Comunicarse con servidores
  • Crear animaciones
  • Desarrollar aplicaciones

⚠️ JavaScript ≠ Java

Ejemplo:

// Cambiar contenido al hacer click const boton = document.querySelector('.boton'); boton.addEventListener('click', () => { const titulo = document.querySelector('.titulo'); titulo.textContent = '¡Hiciste click!'; titulo.style.color = '#00D4AA'; });

💡 Analogía: JavaScript es el cerebro y músculos

Inspector de Elementos

Herramienta integrada en los navegadores para examinar y modificar el código de páginas web

Cómo Acceder

  • Tecla F12
  • Ctrl + Shift + I

Funciones

  • Ver código HTML/CSS
  • Modificar en tiempo real
  • Depurar JavaScript
  • Analizar rendimiento
  • Ver peticiones de red

Accesibilidad Web

Hacer que la web sea usable para todos, incluyendo personas con discapacidades

Principios

  • Perceptible
  • Operable
  • Comprensible
  • Robusto

Prácticas

  • Texto alternativo en imágenes
  • Contraste de colores adecuado
  • Navegación por teclado
  • Estructura semántica

Seguridad Web

Amenazas Comunes

  • XSS (Cross-Site Scripting)
  • SQL Injection
  • CSRF (Cross-Site Request Forgery)
  • Ataques DDoS

Buenas Prácticas

  • Validar todas las entradas
  • Usar HTTPS
  • Mantener software actualizado
  • Implementar autenticación fuerte

Responsive Design

  • Flexbox y Grid
  • Imágenes responsive

Dispositivos

  • Tablets (768px+)
  • Desktop (1024px+)
