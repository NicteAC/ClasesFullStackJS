¿Qué es Git?

Sistema de control de versiones distribuido que permite gestionar y colaborar en proyectos manteniendo un historial completo de cambios.

📜 Historial Completo

Cada cambio queda registrado con fecha, autor y descripción

🔄 Reversible

Puedes volver a cualquier versión anterior del proyecto

🌿 Ramas

Trabaja en funciones paralelas sin afectar el código principal

💻 Local

Funciona en tu máquina sin necesidad de conexión a internet

¿Qué es GitHub?

Plataforma web que aloja repositorios Git y proporciona herramientas para colaboración en equipo.

☁️ Almacenamiento

Tu código seguro en la nube, accesible desde cualquier lugar

👥 Pull Requests

Revisión de código antes de fusionar cambios

📋 Issues

Seguimiento de bugs y tareas del proyecto

🚀 GitHub Actions

Automatización de pruebas y despliegue

Git vs GitHub

Git

  • Software instalado en tu computadora
  • Funciona localmente sin internet
  • Línea de comandos (terminal)
  • Control de versiones

GitHub

  • Plataforma web en la nube
  • Requiere conexión a internet
  • Interfaz gráfica amigable
  • Colaboración y almacenamiento

Restaurar y Deshacer Cambios

# Deshacer cambios en archivo (antes de add)
git checkout -- nombre_archivo

# Quitar archivo del staging (después de add)
git reset HEAD nombre_archivo

# Revertir un commit específico
git revert hash_del_commit

Ignorando Archivos: .gitignore

Archivo que indica a Git qué archivos o carpetas NO debe rastrear

# Archivo .gitignore

# Ignorar todos los .txt
*.txt

# Ignorar carpeta node_modules
node_modules/

# Ignorar archivos de sistema
.DS_Store
Thumbs.db

¿Qué ignorar?

  • Dependencias (node_modules)
  • Archivos de configuración local
  • Archivos compilados
  • Credenciales y API keys
  • Archivos del sistema operativo

Trabajando con Ramas

Las ramas permiten desarrollar funciones de forma aislada sin afectar el código principal

# Crear nueva rama
git branch nombre-rama

# Cambiar a la rama
git checkout nombre-rama

# Crear y cambiar en un paso
git checkout -b nombre-rama

# Ver todas las ramas
git branch

Demo: Ramas y Fusiones

# Crear y moverse a nueva rama
git checkout -b feature-estilos

# Hacer cambios y commit
git add . && git commit -m "Agregar estilos CSS"

# Volver a master
git checkout master

# Fusionar rama
git merge feature-estilos

Etiquetas (Tags)

Marcadores inmutables para versiones importantes del proyecto

# Ver historial de commits
git log --oneline

# Crear tag en commit específico
git tag v1.0 abc123

# Crear tag con mensaje
git tag -a v1.0 -m "Primera versión estable"

# Ver tags existentes
git tag

Repositorios Remotos

Copia del repositorio en un servidor (GitHub, GitLab, Bitbucket)

⬆️ Push

Enviar commits locales al repositorio remoto

git push origin master

⬇️ Pull

Descargar y fusionar cambios del remoto

git pull origin master

📥 Clone

Crear copia local de un repositorio remoto

git clone URL

🔍 Fetch

Descargar cambios SIN fusionar automáticamente

git fetch origin

Fetch vs Pull

git fetch

  • Descarga cambios del remoto
  • NO fusiona automáticamente
  • Puedes revisar antes de integrar
  • Más seguro y controlado

git pull

  • Descarga cambios del remoto
  • Fusiona automáticamente
  • Más rápido y conveniente
  • Puede generar conflictos

git pull = git fetch + git merge

Conectando con GitHub

git push -u origin master

git remote -v

Pull Requests

Solicitud para fusionar cambios de una rama a otra con revisión de código

1️⃣ Crear Rama

Desarrolla tu funcionalidad en una rama separada

2️⃣ Push a GitHub

Sube tu rama al repositorio remoto

3️⃣ Crear PR

Abre el Pull Request desde GitHub

4️⃣ Revisar y Merge

Revisión de código y fusión final

🖥️ DEMO EN VIVO

Flujo Completo de Pull Request

# Crear rama para nueva función
git checkout -b feature-navbar

# Hacer cambios, add y commit
git add . && git commit -m "Agregar navbar"

# Subir rama a GitHub
git push -u origin feature-navbar
