Bloque 1 · jQuery

¿Qué es jQuery?

📚 Definición

Biblioteca de JavaScript que simplifica la manipulación del DOM, el manejo de eventos y la implementación de efectos. Lema: "Write less, do more".

🎯 ¿Por qué aprenderla?

  • Gran parte del ecosistema legado la usa
  • Plugins y plantillas todavía dependen de ella
  • Bootstrap 4 la requiere para sus componentes JS
  • Permite mantener y migrar proyectos existentes
Con jQuery lo que en JS vanilla tomaría 5 líneas, se hace en 1 línea.
Bloque 1 · Instalación

Incluir jQuery + Bootstrap 4

<!-- Bootstrap 4 CSS -->
<link rel="stylesheet" href="https://stackpath.bootstrapcdn.com/bootstrap/4.5.2/css/bootstrap.min.css">

<!-- Al final del body: jQuery → Popper → Bootstrap JS -->
<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/popper.js@1.16.0/dist/umd/popper.min.js"></script>
<script src="https://stackpath.bootstrapcdn.com/bootstrap/4.5.2/js/bootstrap.min.js"></script>
El orden importa: jQuery → Popper → Bootstrap. Sin jQuery antes, Bootstrap 4 no funciona.
Bloque 1 · Sintaxis

Sintaxis Fundamental de jQuery

// Selector jQuery → Acción
$("selector").accion();

// Ejemplos de selectores (mismos que CSS)
$("p") // todos los párrafos
$("#mi-id") // elemento por ID
$(".mi-clase") // elementos por clase
$("p.texto") // párrafos con clase .texto

// Esperar que el DOM esté listo
$(document).ready(function() {
  // todo el código jQuery va aquí
});
// Forma abreviada:
$(function() { /* código aquí */ });
Bloque 1 · Visibilidad

Show · Hide · Toggle

👁️ .show()

Muestra un elemento que estaba oculto (display:none).

$("button").on("click", function(){
  $("p.texto").show();
});

🙈 .hide()

Oculta un elemento visible.

$("button").on("click", function(){
  $("p.texto").hide();
});

🔄 .toggle()

Alterna entre mostrar y ocultar según el estado actual.

$("button").click(function(){
  $("p.texto").toggle();
});
Bloque 1 · Animaciones

Efectos: Fade y Slide

🌫️ Fade (con transición suave)

$("p").fadeIn("slow"); // aparece
$("p").fadeOut(2000); // desaparece

// Con callback al terminar:
$("p").fadeOut(2000, function(){
  alert("¡Terminó!");
});

📦 Slide (desplazamiento)

$("p").slideDown(1000); // baja
$("p").slideUp(1000); // sube
$("p").slideToggle(); // alterna
Parámetros de velocidad: "slow", "fast" o un número en milisegundos. El segundo parámetro es un callback.
Bloque 1 · Scroll

Scroll Animado a una Sección

<!-- HTML: anchor y sección destino -->
<a id="ir-contacto" href="#">Ir a Contacto</a>
<div id="seccionContacto">...</div>

// jQuery: animar el scroll
$("#ir-contacto").click(function(e) {
  e.preventDefault();
  $("html, body").animate({
    scrollTop: $("#seccionContacto").offset().top
  }, 2000);
});
Muy usado en landing pages: el usuario navega haciendo clic en el menú y se desplaza suavemente entre secciones.

Bloque 2 · Plugins

Plugins Bootstrap 4 + jQuery

🧩 ¿Qué es un plugin?

Código que extiende Bootstrap/jQuery con funcionalidades extra: carruseles, modales, acordeones, datepickers. Sin escribir todo desde cero.

🎠 Carrusel BS4

  • Clase .carousel + data-ride="carousel"
  • Items con .carousel-item
  • Controles prev/next con data-slide
  • jQuery para inicializar y personalizar
// Inicializar carrusel con jQuery y opciones
$('#miCarrusel').carousel({
  interval: 3000, // ms entre slides
  pause: 'hover' // pausa al pasar el cursor
});
// Pausar con mouseenter
$('#miCarrusel').on('mouseenter', function(){ $(this).carousel('pause'); });

¿Qué es Git?

Permite gestionar y colaborar en proyectos manteniendo un historial completo de todos los cambios realizados en los archivos.

Bloque 3 · Instalación

Instalar y Configurar Git

⬇️ Instalación

  • Descargar desde git-scm.com
  • Seleccionar "Git Bash Here" en la instalación
  • Agregar Git al PATH del sistema

Bloque 3 · Comandos

Comandos Fundamentales

# Crear repositorio nuevo
git init

# Clonar repositorio existente
git clone https://github.com/usuario/repo.git

# Ver estado de los archivos
git status

# Agregar archivos al staging area
git add nombre_archivo # archivo específico
git add . # todos los cambios

# Guardar commit con mensaje
git commit -m "Descripción del cambio"
Bloque 3 · Commits

Restaurar · Revertir · Renombrar

# Deshacer cambios en working directory
git checkout -- nombre_archivo

# Quitar archivo del staging (sin borrar cambios)
git reset HEAD nombre_archivo

# Revertir un commit anterior (crea nuevo commit que lo deshace)
git revert hash_del_commit

# Renombrar un archivo (Git lo trackea correctamente)
git mv archivo_antiguo.txt archivo_nuevo.txt
git add .
git commit -m "Renombrado archivo"

# Ver historial de commits
git log --oneline
Bloque 3 · .gitignore

Ignorar Archivos con .gitignore

📄 ¿Qué es?

Archivo en la raíz del proyecto que lista patrones de archivos que Git debe no rastrear: dependencias, credenciales, builds.

✍️ Ejemplos de patrones

# Extensiones
*.txt
*.log

# Carpetas
node_modules/
.env
dist/
# Crear y commitear .gitignore
git add .gitignore
git commit -m "Agregado .gitignore"
Bloque 3 · Ramas

Ramas (Branches)

# Crear y cambiar a nueva rama
git branch nueva-feature
git checkout nueva-feature
# O en un solo comando:
git checkout -b nueva-feature

# Hacer cambios y commitear en la nueva rama
git add .
git commit -m "Nueva funcionalidad"

# Volver a main y fusionar
git checkout master
git merge nueva-feature
Las ramas permiten trabajar en features o fixes sin tocar la rama principal hasta que esté listo.
Bloque 3 · Tags

Etiquetas (Tags)

# Ver historial para encontrar el hash del commit
git log --oneline

# Crear tag en un commit específico
git tag v1.0 abcdef123456

# Tag anotado (recomendado, con mensaje)
git tag -a v1.0 -m "Versión 1.0 estable"

# Ver todos los tags
git tag

# Subir tags al repositorio remoto
git push origin --tags
Los tags marcan versiones estables del proyecto. Son punteros inmutables a un commit específico.

Bloque 4 · Remoto

Repositorios Remotos

☁️ ¿Qué es un remoto?

Copia del repositorio en un servidor como GitHub. Permite colaborar, compartir código y tener respaldo centralizado.

📥 Clonar

Traer una copia completa del repositorio remoto a tu máquina local, incluyendo todo el historial.

git clone https://github.com/usuario/repo.git

⟶ push ⟶
🐙 GitHub

Bloque 4 · Push

git push — Subir Cambios

# Flujo completo: modificar → agregar → commitear → subir

# 1. Ver estado
git status

# 2. Agregar al staging
git add nombre_archivo

# 3. Commitear
git commit -m "Descripción del cambio"

# 4. Subir al remoto
git push origin master

# Primera vez: establecer rama upstream
git push -u origin master
Bloque 4 · Pull

git pull — Traer Cambios

# Verificar estado local antes de pull
git status

# Traer y fusionar cambios del remoto
git pull origin master

# Si hay conflictos: editar los archivos, luego:
git add .
git commit -m "Resueltos conflictos"
git pull = git fetch + git merge en un solo paso.
Bloque 4 · Fetch vs Pull

fetch vs pull

🔍 git fetch

Descarga los cambios del remoto sin fusionarlos automáticamente. Podés inspeccionar antes de aplicar.

git fetch origin
# Revisar cambios
git diff origin/master
# Fusionar cuando esté listo
git merge origin/master

⬇️ git pull

Descarga y fusiona automáticamente. Más conveniente para uso diario cuando confiás en los cambios.

# fetch + merge en un paso
git pull origin master
Usá fetch en equipos para revisar antes de fusionar. Usá pull cuando trabajás solo o en ramas propias.
