Introducción

¿Qué es Vue Router?

📌 Definición

Biblioteca oficial de enrutamiento para Vue.js. Mapea URLs del navegador a componentes, permitiendo que la app cambie de vista sin recargar la página.

🌐 SPA

Simula el comportamiento de un sitio multi-página dentro de una Single Page Application. El navegador carga el HTML solo una vez.

🗺️ Sin Vue Router

  • Cada vista = nueva petición al servidor
  • La página se recarga completa
  • El estado de la app se pierde
  • Experiencia lenta y fragmentada

✅ Con Vue Router

  • La URL cambia, el HTML no
  • El componente cambia instantáneamente
  • El estado se conserva entre vistas
  • Experiencia fluida de app nativa

Regla: Si la app tiene más de una "pantalla", usamos Vue Router. Una sola vista = no es necesario.

Motivación

¿Para qué y cuándo usarlo?

🖥️ Múltiples vistas

Cuando la app requiere diferentes "pantallas": inicio, productos, contacto, perfil, etc. Cada una tiene su propia URL.

⚡ Sin recarga

En vez de múltiples archivos HTML, un solo archivo con componentes que cambian según la ruta activa.

🔒 Control total

Historial de navegación, navegación programática, guardas de acceso y rutas protegidas.

🕐 ¿Cuándo usar Vue Router?

  • La app tiene más de una vista o sección
  • Necesitás navegación fluida sin recargas
  • Querés mantener el estado entre cambios de vista
  • La app usa componentes reutilizables por sección

✅ Ventajas clave

  • Integración con Vue Devtools para debugging
  • Compatible con la arquitectura basada en componentes
  • Estandariza el manejo de rutas en el proyecto
  • Mejora la experiencia del usuario
Instalación

Instalar y registrar Vue Router

Instalar desde la terminal del proyecto:

npm install vue-router # o con yarn: yarn add vue-router

Registrar en main.js:

import Vue from 'vue' import App from './App.vue' import router from './router' // Conecta el router a la instancia Vue new Vue({ router, render: h => h(App) }).$mount('#app')

¿Qué hace router en la instancia?
Hace que todos los componentes tengan acceso a this.$route (info de la ruta actual) y this.$router (instancia del router para navegar).

Estructura de carpetas recomendada:

src/ ├── router/ │ └── index.js ← configuración del router ├── views/ ← componentes de página (rutas) │ ├── Home.vue │ ├── Productos.vue │ └── NotFound.vue ├── components/ ← componentes reutilizables └── App.vue ← contiene router-view
Configuración

Archivo router/index.js

Cuatro pasos para armar el router:

①Importar Vue, VueRouter y los componentes de vista
②Registrar el plugin con Vue.use(VueRouter)
③Definir el array routes con path y componente
④Crear la instancia con new VueRouter() y exportarla

mode: 'history' — URLs limpias como /productos.
Sin esto, las URLs usan #: /#/productos.

import Vue from 'vue' import VueRouter from 'vue-router' import Home from '../views/Home.vue' import Productos from '../views/Productos.vue' // ② Registrar el plugin Vue.use(VueRouter) // ③ Definir las rutas const routes = [ { path: '/', component: Home }, { path: '/productos', component: Productos } ] // ④ Crear la instancia y exportar const router = new VueRouter({ mode: 'history', routes }) export default router
Componentes clave

router-link y router-view

🔗 <router-link>

Reemplaza al <a href>. Intercepta el clic y navega sin recargar. Vue aplica automáticamente la clase router-link-active al enlace activo.

<nav> <router-link to="/">Inicio</router-link> <router-link to="/productos">Productos</router-link> </nav>

📺 <router-view>

El "marco" donde Vue inyecta el componente de la ruta activa. Va en App.vue. Cambia de contenido cuando cambia la URL.

<template> <div id="app"> <nav> ... router-links ... </nav> <!-- La vista activa se renderiza aquí --> <router-view></router-view> </div> </template>

¿Cómo funcionan juntos?

1Usuario hace clic en <router-link to="/productos">
2Vue Router intercepta el clic y actualiza la URL sin recargar
3Busca en el array routes cuál coincide con /productos
4Renderiza ese componente dentro de <router-view>

Analogía: router-view es un marco en la pared. Los componentes son cuadros que se intercambian dentro del marco.

Rutas

Rutas estáticas

Path fijo → componente fijo. Las más simples de definir.

const routes = [ // path: URL · component: qué renderizar { path: '/', component: Home }, { path: '/about', component: About }, { path: '/productos', component: Productos }, { path: '/contacto', component: Contacto } ]

Convención: las vistas van en src/views/. Son componentes que corresponden a rutas. Los reutilizables van en src/components/.

🏠 Home.vue

Componente para la ruta raíz /. Página de bienvenida.

📦 Productos.vue

Componente para /productos. Lista el catálogo.

📬 Contacto.vue

Componente para /contacto. Formulario de contacto.

Rutas dinámicas

Parámetros con :nombre

El prefijo : convierte un segmento en variable. Una sola ruta captura múltiples URLs.

// router/index.js { path: '/producto/:id', component: DetalleProducto } // Coincide con TODAS estas URLs: // /producto/1 // /producto/42 // /producto/abc // /producto/zapatillas-nike
// Múltiples parámetros en una misma ruta { path: '/blog/:año/:mes/:slug', component: ArticuloBlog } // /blog/2020/03/vue-router-guia

Ventajas de rutas dinámicas:

♻️ Reutilización

Un solo componente maneja el detalle de cualquier producto, sin duplicar código.

🔗 URLs semánticas

Las URLs son descriptivas y amigables para el usuario y los motores de búsqueda.

🎯 Contextual

La interfaz se personaliza según el parámetro recibido, mostrando datos relevantes.

Acceso a datos

$route.params y validación

Acceder al parámetro desde el componente:

// DetalleProducto.vue export default { mounted() { // $route.params contiene los parámetros const id = this.$route.params.id console.log(id) // ⚠️ Siempre llega como STRING // '42' no es lo mismo que 42 this.cargarProducto(id) } }

⚠️ $route.params.id siempre es string. Para operaciones numéricas usar Number(id) o parseInt(id).

Validar parámetros antes de usarlos:

export default { data() { return { producto: null, error: false } }, mounted() { const id = this.$route.params.id // Validar que sea numérico if (!/^\d+$/.test(id)) { this.error = true return // Salir si no es válido } // Convertir a número y buscar this.buscarProducto(Number(id)) }, methods: { buscarProducto(id) { /* ... */ } } }

Jerarquía

Rutas anidadas — children

Una vista padre contiene sub-rutas en su interior. Se definen con children.

const routes = [ { path: '/usuario/:id', component: Usuario, children: [ // → /usuario/:id/perfil { path: 'perfil', component: UsuarioPerfil }, // → /usuario/:id/posts { path: 'posts', component: UsuarioPosts } ] } ]

Los paths de children son relativos — no empiezan con /. Vue Router arma la URL completa automáticamente.

Usuario.vue necesita su propio <router-view>:

<template> <div> <h2>Usuario {{ $route.params.id }}</h2> <nav> <router-link :to="`/usuario/${$route.params.id}/perfil`">Perfil</router-link> <router-link :to="`/usuario/${$route.params.id}/posts`">Posts</router-link> </nav> <!-- Aquí se inyectan las subrutas --> <router-view></router-view> </div> </template>

⚠️ Sin <router-view> en el padre, las subrutas no se renderizan. Es el error más común con rutas anidadas.

Desacoplamiento

Props desde rutas

Por defecto: componente acoplado al router.

// ❌ Acoplado al router export default { mounted() { const id = this.$route.params.id // Depende de $route — difícil de testear } }

Con props: true: componente desacoplado.

// router/index.js { path: '/producto/:id', component: Detalle, props: true } // ✅ Desacoplado — limpio y testeable export default { props: ['id'], // :id llega como prop mounted() { this.cargar(this.id) // No usa $route } }

Tres modos de pasar props:

Boolean mode

props: true

Pasa todos los params como props al componente.

Object mode

props: { premium: true }

Props estáticas que no vienen de la URL.

Function mode

props: route => ({ id: route.params.id })

Transforma o combina parámetros antes de pasarlos.

Navegación

Redirecciones

Vue Router permite definir redirecciones automáticas. La URL cambia en el navegador.

const routes = [ // Path simple → nueva ruta { path: '/inicio', redirect: '/' }, // Redirigir a una ruta con nombre { path: '/home', redirect: { name: 'inicio' } }, // Redirección dinámica con función { path: '/buscar', redirect: to => { return { path: '/productos', query: to.query } }} ]

Casos de uso para redirecciones:

🗑️ URLs obsoletas

Cuando el sitio se reorganiza, redirigir desde rutas antiguas a las nuevas sin romper links existentes.

🔒 Rutas protegidas

Redirigir al login cuando un usuario intenta acceder a contenido que requiere autenticación.

🏠 Ruta por defecto

Redirigir a la página principal cuando el usuario accede a una ruta raíz sin especificar sección.

Rutas alternativas

Alias — misma vista, URL distinta

Los alias permiten acceder a la misma vista por diferentes rutas sin cambiar la URL.

const routes = [ { path: '/productos', component: ListaProductos, // /items también muestra ListaProductos // pero la URL no cambia alias: '/items' }, // Múltiples alias { path: '/sobre-nosotros', component: About, alias: ['/about', '/acerca'] } ]

Diferencias clave:

🔀 Redirect

  • La URL cambia en el navegador
  • Reemplaza la entrada del historial
  • El usuario ve la nueva URL
  • Útil cuando una ruta fue movida

🏷️ Alias

  • La URL no cambia
  • No modifica el historial
  • El usuario ve la URL que escribió
  • Útil para rutas alternativas
Manejo de errores

Ruta 404 — Página no encontrada

El comodín * captura cualquier ruta que no coincida con las definidas. Siempre al final del array.

const routes = [ { path: '/', component: Home }, { path: '/productos', component: Productos }, { path: '/contacto', component: Contacto }, // ⚠️ SIEMPRE AL FINAL DEL ARRAY // Captura todo lo que no coincidió { path: '*', component: NotFound } ]

Si * está al principio, captura todo y ninguna ruta funciona. Siempre va al final.

Componente NotFound.vue:

<template> <div class="not-found"> <h1>404 — Página no encontrada</h1> <p> La ruta {{ $route.path }} no existe. </p> <router-link to="/"> ← Volver al inicio </router-link> </div> </template> <script> export default { name: 'NotFound' } </script>
