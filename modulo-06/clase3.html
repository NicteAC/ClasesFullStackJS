Formulario de Perfil — v-model en todos los tipos de campo

QUÉ VAS A CONSTRUIR

Un formulario de perfil de usuario con todos los tipos de input enlazados con v-model: texto, email, número, radio, checkboxes, select y textarea. Incluye una sección de preview en tiempo real.

PASOS EN ORDEN

1
Crear data() con todos los campos del formulario
2
v-model en text e email · mostrar doble binding en vivo
3
v-model.number en el campo edad · demostrar typeof
4
Radio buttons con v-model compartido · value por opción
5
Checkboxes múltiples enlazados a un array
6
<select> con v-model · valor por defecto
7
Textarea con v-model.trim y contador de chars
8
Sección preview que muestra todos los valores en tiempo real

REFERENCIA RÁPIDA — código clave

// data() — todos los campos
data() {
  return {
    nombre: '',       // v-model.trim
    email:  '',       // v-model.trim
    edad:   null,      // v-model.number
    genero: '',       // radio buttons
    skills: [],        // checkboxes → array
    nivel:  'junior', // select
    bio:    ''        // textarea
  }
}

<!-- radio: mismo v-model, value distinto -->
<input type="radio" v-model="genero"
       value="M"/> Masculino
<input type="radio" v-model="genero"
       value="F"/> Femenino
<!-- checkboxes → agregan al array -->
<input type="checkbox" v-model="skills"
       value="vue"/> Vue
<input type="checkbox" v-model="skills"
       value="react"/> React

<!-- select: v-model en el <select> -->
<select v-model="nivel">
  <option value="junior">Junior</option>
  <option value="senior">Senior</option>
</select>

<!-- textarea: tag de cierre obligatorio -->
<textarea v-model.trim="bio"></textarea>
<small>{{ bio.length }}/150</small>

<!-- preview en tiempo real -->
<pre>{{ $data }}</pre>

Validación en Tiempo Real — computed + @blur + submit

QUÉ VAS A CONSTRUIR

Un formulario de login con validación reactiva completa: email con regex, contraseña con fuerza, confirmación de contraseña. Bordes dinámicos verde/rojo, botón deshabilitado y mensaje de error.

PASOS EN ORDEN

1
Crear data(): form (email, password, confirm) + errores (objeto)
2
Computed emailValido con regex · computed passwordValido (min 6)
3
Computed confirmacionValida (igualdad) · computed formularioValido
4
Validar con @blur en cada campo (solo al salir del input)
5
Bordes dinámicos con :style según estado de validación
6
Botón submit :disabled="!formularioValido"
7
Método enviar() con @submit.prevent

REFERENCIA RÁPIDA — validaciones

// data()
data() {
  return {
    form: {
      email: '',
      password: '',
      confirm: ''
    },
    errores: {
      email: '',
      password: '',
      confirm: ''
    }
  }
},

// computed — reactivos
computed: {
  emailValido() {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/
             .test(this.form.email)
  },
  passwordValido() {
    return this.form.password.length >= 6
  },
  confirmacionValida() {
    return this.form.password
             === this.form.confirm
           && this.form.confirm !== ''
  },
  formularioValido() {
    return this.emailValido
      && this.passwordValido
      && this.confirmacionValida
  }
},
// @blur — valida al salir del campo
validarEmail() {
  this.errores.email =
    this.emailValido
      ? ''
      : 'Email inválido'
},
validarPassword() {
  this.errores.password =
    this.passwordValido
      ? ''
      : 'Mínimo 6 caracteres'
},

<!-- borde dinámico -->
<input
  v-model="form.email"
  @blur="validarEmail"
  :style="{
    borderColor: errores.email
      ? '#FF6B6B'
      : form.email && emailValido
        ? '#00D4AA' : ''
  }"
/>
<p v-if="errores.email">
  {{ errores.email }}
</p>

<!-- botón deshabilitado -->
<button
  :disabled="!formularioValido"
>Entrar</button>

Formulario Multipaso — Wizard de 4 pasos con progreso

QUÉ VAS A CONSTRUIR

Una encuesta de 4 pasos con barra de progreso, navegación Siguiente/Anterior, validación por paso y pantalla de resumen final.

PASOS EN ORDEN

1
Estructura de datos: pasoActual, objeto encuesta, totalPasos = 4
2
Barra de progreso con :style="{ width: progreso + '%' }"
3
Mostrar paso correcto con v-if="pasoActual === 1" etc.
4
Paso 1: datos personales (nombre, email)
5
Paso 2: checkboxes de intereses
6
Paso 3: slider satisfacción + radio recomendaría
7
Paso 4: resumen visual de todo lo ingresado
8
Computed pasoValido con switch · botón Siguiente deshabilitado

REFERENCIA RÁPIDA — lógica de navegación

// data()
data() {
  return {
    pasoActual: 1,
    totalPasos: 4,
    encuesta: {
      nombre: '',
      email: '',
      intereses: [],
      satisfaccion: 5,
      recomendaria: null
    }
  }
},

// computed progreso
progreso() {
  return (
    this.pasoActual
    / this.totalPasos
  ) * 100
},

// methods
siguientePaso() {
  if(this.pasoValido
    && this.pasoActual < this.totalPasos)
    this.pasoActual++
},
anteriorPaso() {
  if(this.pasoActual > 1)
    this.pasoActual--
},
// pasoValido — un caso por paso
pasoValido() {
  switch(this.pasoActual) {
    case 1:
      return this.encuesta.nombre
          && this.encuesta.email
    case 2:
      return this.encuesta
               .intereses.length > 0
    case 3:
      return this.encuesta
               .recomendaria !== null
    default: return true
  }
},

<!-- barra de progreso -->
<div class="barra-bg">
  <div
    class="barra-fill"
    :style="{ width: progreso+'%' }"
  ></div>
</div>

<!-- nav buttons -->
<button
  v-if="pasoActual < totalPasos"
  :disabled="!pasoValido"
  @click="siguientePaso"
>Siguiente →</button>
<button
  v-if="pasoActual === totalPasos"
  @click="enviar"
>✅ Enviar</button>

Arrays Dinámicos + Componente CampoTexto con v-model

QUÉ VAS A CONSTRUIR

Un formulario de pedido con ítems dinámicos (agregar/eliminar filas), total computado automáticamente, y un componente personalizado CampoTexto.vue que usa v-model.

PASOS EN ORDEN

1
Estructura: objeto pedido con cliente + items: []
2
Método agregarItem() con push() · método eliminarItem(i) con splice()
3
Renderizar ítems con v-for · campos editables en cada fila
4
Computed total con reduce() sobre el array de ítems
5
Crear CampoTexto.vue — prop modelValue + emit update:modelValue
6
Usar CampoTexto en el formulario con v-model
7
Mostrar resumen del pedido debajo del formulario

REFERENCIA RÁPIDA — arrays y componente

// data() — pedido con ítems
data() {
  return {
    pedido: {
      cliente: '',
      items: [{
        producto: '',
        cantidad: 1,
        precio: 0
      }]
    }
  }
},

// total = reduce sobre el array
total() {
  return this.pedido.items
    .reduce((sum, item) =>
      sum + (item.cantidad * item.precio)
    , 0)
},

// push y splice mantienen reactividad
agregarItem() {
  this.pedido.items.push({
    producto:'', cantidad:1, precio:0
  })
},
eliminarItem(i) {
  if(this.pedido.items.length > 1)
    this.pedido.items.splice(i, 1)
}
// ── CampoTexto.vue ──
// Componente que acepta v-model
<template>
  <div>
    <label v-if="label">
      {{ label }}
    </label>
    <input
      :value="modelValue"
      @input="$emit(
        'update:modelValue',
        $event.target.value)"
      :placeholder="placeholder"
    />
  </div>
</template>
<script>
export default {
  props: ['modelValue', 'label',
          'placeholder'],
  emits: ['update:modelValue']
}
</script>

<!-- uso idéntico a input nativo -->
<CampoTexto
  v-model="pedido.cliente"
  label="Cliente"
  placeholder="Nombre..."
/>
