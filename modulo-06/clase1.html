¿Qué es Vue.js?

Framework progresivo

Puede adoptarse de forma gradual — desde un simple <script> en HTML hasta una SPA completa.

Basado en componentes

Cada pieza de la UI es un componente con su propio template, script y style encapsulados.

Reactivo por diseño

Cuando los datos cambian, Vue actualiza el DOM automáticamente. Sin manipulación manual.

Estructura de un Componente Vue

📄

<template>

HTML declarativo. Lo que se renderiza en pantalla.

⚙️

<script>

Lógica, datos, métodos, props, lifecycle hooks.

🎨

<style>

CSS encapsulado con scoped.

Comunicación: props (padre→hijo) · $emit (hijo→padre) · slots (contenido dinámico)

MiComponente.vue

<template>
  <div class="mi-componente">
    <h2>{{ titulo }}</h2>
    <button @click="saludar">Hola</button>
  </div>
</template>

<script>
export default {
  props: ['titulo'],
  data() {
    return { contador: 0 }
  },
  methods: {
    saludar() {
      this.$emit('click-btn')
    }
  }
}
</script>

<style scoped>
h2 { color: #00D4AA; }
</style>

Reactividad en Vue.js

La reactividad es la capacidad de Vue para actualizar automáticamente el DOM cuando cambian los datos — sin manipulación manual del DOM.

👁️

1. Observación

Vue intercepta lecturas y escrituras sobre las propiedades del objeto data.

✏️

2. Modificación

Cuando un dato cambia, Vue registra la modificación.

🔔

3. Notificación + Update

Todos los componentes que dependen del dato son notificados y la UI se actualiza.

El patrón MVVM: Model (data) · View (template) · ViewModel (instancia Vue)

  return { contador: 0 }

Lifecycle · Computed · Watchers

Ciclo de Vida

  • beforeCreate — antes de init
  • created — data lista, no DOM
  • mounted — DOM disponible ⭐
  • updated — tras cambio DOM
  • beforeDestroy — cleanup
  • destroyed — fin
mounted() {
  // llamada AJAX aquí
  fetch('/api/data')...
}

Computed

Valores derivados con caché. Solo se recalculan si sus dependencias cambian.

computed: {
  nombreCompleto() {
    return `${this.nombre} 
${this.apellido}`
  },
  tareasCompletas() {
    return this.tareas
      .filter(t => t.done)
      .length
  }
}

Watchers

Reaccionan a cambios en una propiedad. Ideales para operaciones async.

watch: {
  busqueda(nuevo) {
    this.buscar(nuevo)
  },
  'usuario.nombre': {
    handler(val) {
      console.log(val)
    },
    deep: true,
    immediate: true
  }
}

💡 Regla práctica: usá computed para derivar datos → usá watch para reaccionar a cambios y ejecutar lógica async

Vue Router — Navegación SPA

Vue Router permite navegar entre vistas sin recargar la página. La URL cambia, pero el HTML base es siempre el mismo.

router-link

<router-link
  to="/">Inicio</router-link>
<router-link
  :to="{ name:'About' }"
>About</router-link>

router-view

<!-- App.vue -->
<nav>...links...</nav>
<router-view />
<!-- aquí se renderiza -->
<!-- la vista activa     -->

Navegación programática

this.$router.push('/about')
this.$router.push({ name: 'User', params: { id: 42 } })
this.$route.params.id  // leer parámetro

router/index.js

import Vue from 'vue'
import VueRouter from 'vue-router'
import Home from '../views/Home.vue'
import About from '../views/About.vue'

Vue.use(VueRouter)

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
  {
    path: '/user/:id',
    component: () => import('../views/User.vue')
    // lazy-loading ⚡
  }
]

export default new VueRouter({ routes })

// store/index.js
import Vue from 'vue'
import Vuex from 'vuex'
Vue.use(Vuex)

export default new Vuex.Store({
  state: {
    tareas: [],
    contador: 0
  },
  mutations: {
    AGREGAR_TAREA(state, texto) {
      state.tareas.push({ texto, done: false })
    },
    INCREMENTAR(state) {
      state.contador++
    }
  },
  actions: {
    agregarTarea({ commit }, texto) {
      commit('AGREGAR_TAREA', texto)
    }
  },
  getters: {
    totalDone: state =>
      state.tareas.filter(t => t.done).length
  }
})
// En un componente
import { mapState, mapGetters, mapActions } from 'vuex'

export default {
  computed: {
    ...mapState(['tareas', 'contador']),
    ...mapGetters(['totalDone'])
  },
  methods: {
    ...mapActions(['agregarTarea']),
    agregar() {
      this.agregarTarea(this.input)
    }
  }
}

Regla: mutations son síncronas · actions pueden ser async
Solo las mutations modifican el state.

store/index.js — completo

export default new Vuex.Store({
  state: { tareas: [] },
  mutations: {
    AGREGAR(state, texto) {
      state.tareas.push({ texto, done: false })
    },
    ELIMINAR(state, idx) {
      state.tareas.splice(idx, 1)
    },
    TOGGLE(state, idx) {
      state.tareas[idx].done =
        !state.tareas[idx].done
    }
  },
  getters: {
    total: s => s.tareas.length,
    done: s => s.tareas.filter(t => t.done).length
  }
})

Time-travel debugging

En la pestaña Vuex podés hacer click en cualquier mutation anterior y volver el estado de la app a ese momento. Ideal para debuggear flujos complejos.
