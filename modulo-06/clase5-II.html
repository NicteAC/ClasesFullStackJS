📋 Consigna — ≠ Demo

Panel de administración con área pública y privada. El rol del usuario determina qué rutas puede ver. Formulario con guarda de salida. Lazy loading en todo el área privada.
  1. Rutas públicas: /, /login, /404
  2. Rutas privadas con meta: { requiresAuth: true, role: 'admin' }: /admin, /admin/usuarios, /admin/reportes
  3. Función isAuthenticated() y función getUserRole()
  4. beforeEach: si requiere auth sin login → redirigir a /login guardando query: { redirect: to.fullPath }
  5. En Login.vue: tras autenticar, leer $route.query.redirect y navegar allí con $router.push
  6. Si el rol no coincide con to.meta.role → redirigir a /404
  7. Lazy loading en todas las vistas del área /admin
  8. scrollBehavior: si hay hash en la URL (to.hash), hacer scroll al elemento; si no, volver a { x: 0, y: 0 }
  9. En FormularioReporte.vue: implementar beforeRouteLeave que pregunta confirm('¿Descartar cambios?') antes de salir si hay cambios sin guardar
  10. En beforeRouteUpdate en la vista de usuario: recargar datos si cambia el :id

Resumen

Buenas prácticas Vue Router

📁 Organización

Separar rutas en archivos por módulo cuando el proyecto crece. Un solo router/index.js se vuelve difícil de mantener.

💤 Lazy loading

Usar () => import() en rutas que no son críticas para la carga inicial. Reduce el bundle de entrada.

📦 Props vs $route

Preferir props: true para desacoplar componentes del router. Más fácil de reutilizar y testear.

🏷️ Rutas con nombre

Siempre usar name en rutas importantes. Si el path cambia, solo se actualiza en el router — el resto del código usa el nombre.

🛡️ Guardas específicas

Implementar guardas en el nivel más específico posible. Global (beforeEach) para auth general. Por componente para lógica de negocio.
