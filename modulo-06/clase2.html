¿Qué son los Templates en Vue?

Los templates son estructuras declarativas que definen qué ve el usuario. Viven dentro del bloque <template> y son HTML enriquecido con expresiones de Vue.

📋

Declarativo

Describís qué mostrar, no cómo manipular el DOM.

⚡

Reactivo

Cuando cambian los datos, la vista se actualiza sola.

🔧

Componible

Los componentes se usan como etiquetas HTML custom.

Vue transforma los templates en funciones render que manipulan el DOM virtual. Solo actualiza lo que cambió.

Anatomía de un template

<template>
  <div class="tarjeta">

    <!-- interpolación -->
    <h2>{{ titulo }}</h2>

    <!-- directiva de atributo -->
    <img :src="imagen" />

    <!-- directiva condicional -->
    <span v-if="activo">✅</span>

    <!-- evento -->
    <button @click="accion">Click</button>

    <!-- componente hijo -->
    <mi-componente :datos="info" />

  </div>
</template>

Interpolaciones y Expresiones

La sintaxis {{ expresión }} inserta contenido dinámico en el HTML. Se evalúa en el contexto del componente — puede acceder a data, computed y methods.

Datos simples

<p>{{ mensaje }}</p>
<p>{{ usuario.nombre }}</p>
<p>{{ usuario.datos.edad }}</p>

Expresiones JS válidas

<p>{{ nombre + ' ' + apellido }}</p>
<p>{{ precio * 1.21 }}</p>
<p>{{ activo ? 'On' : 'Off' }}</p>
<p>{{ formatearFecha(fecha) }}</p>

⛔ NO permitido en interpolaciones

// No podés poner estructuras de control
{{ if (condicion) { } }}      // ❌
{{ for (let i ...) { } }}   // ❌
{{ var x = 1 }}              // ❌ (declaración)

// Sí podés: expresiones de una sola línea
{{ lista.length > 0 }}         // ✅
{{ items.filter(i => i.a).length }} // ✅

💡 Si la lógica es compleja → movela a una computed property. Los templates deben ser legibles.

Atributos: usá v-bind

<!-- ❌ Incorrecto -->
<img src="{{ imagen }}" />

<!-- ✅ Correcto -->
<img :src="imagen" />

v-if vs v-show — ¿Cuándo usar cada uno?

v-if · v-else-if · v-else

Añade o elimina el nodo del DOM según la condición. El componente se destruye y recrea.

<div v-if="rol === 'admin'">
  Panel admin
</div>
<div v-else-if="rol === 'user'">
  Panel usuario
</div>
<div v-else>
  Sin acceso
</div>

✅ Usalo cuando: la condición cambia pocas veces · el componente es pesado · necesitás controlar permisos

v-show

El nodo siempre está en el DOM. Solo cambia display: none. El componente se mantiene vivo.

<div v-show="mostrarMenu">
  Menú lateral
</div>

<div v-show="cargando">
  <spinner />
</div>

<!-- inspeccioná en DevTools:
     el nodo existe aunque esté oculto -->

✅ Usalo cuando: alterna frecuentemente · el componente es ligero · tooltips, dropdowns, sidebars

v-ifv-show
Nodo en DOMSolo si condición = trueSiempre
Alternancia frecuente❌ Costoso✅ Eficiente
Carga inicial✅ Menor (no renderiza)❌ Mayor (todo cargado)
Lifecycle hooksSe ejecutan al montar/destruirNo se ejecutan

v-for — Renderización de Listas

Array básico

<li v-for="item in lista" :key="item.id">
  {{ item.nombre }}
</li>

Con índice

<li v-for="(item, i) in lista" :key="item.id">
  {{ i + 1 }}. {{ item.nombre }}
</li>

Sobre objetos

<div v-for="(val, clave, i) in objeto"
     :key="clave">
  {{ clave }}: {{ val }}
</div>

Con template (sin wrapper)

<template v-for="item in lista">
  <dt :key="'dt-'+item.id">{{ item.nombre }}</dt>
  <dd :key="'dd-'+item.id">{{ item.desc }}</dd>
</template>

⚠️ :key es obligatorio

Permite a Vue identificar cada elemento durante actualizaciones del DOM virtual. Sin :key, Vue puede reutilizar nodos incorrectamente.

NO mezcles v-if y v-for en el mismo elemento

<!-- ❌ Evitar -->
<li v-for="u in usuarios" v-if="u.activo" />

<!-- ✅ Mejor: filtrar con computed -->
<li v-for="u in usuariosActivos" :key="u.id" />

// computed
usuariosActivos() {
  return this.usuarios.filter(u => u.activo)
}

Si no tenés ID único → usá clave compuesta: :key="`${cat}-${nombre}`"

Listado de Productos — Solución

template

<div v-if="productos.length === 0">
  Sin productos disponibles
</div>

<div v-for="p in productos" :key="p.id"
  :class="{ agotado: p.stock === 0 }">
  <h3>{{ p.nombre }}</h3>
  <p>${{ p.precio.toFixed(2) }}</p>
  <span v-if="p.stock > 0">
    ✅ En stock ({{ p.stock }})
  </span>
  <span v-else>❌ Agotado</span>
  <button @click.once="p.verDetalle = true">
    Ver detalles
  </button>
  <div v-show="p.verDetalle">
    ID: {{ p.id }} · Stock: {{ p.stock }}
  </div>
</div>

<p>En stock: {{ productosEnStock }}</p>

script

export default {
  data() {
    return {
      nuevo: { nombre:'', precio:0, stock:0 },
      productos: [
        { id:1, nombre:'Laptop', precio:999,
          stock:5, verDetalle:false },
        { id:2, nombre:'Mouse', precio:29,
          stock:0, verDetalle:false },
      ]
    }
  },
  computed: {
    productosEnStock() {
      return this.productos
        .filter(p => p.stock > 0).length
    }
  },
  methods: {
    agregar() {
      const id = Date.now()
      this.productos.push({
        ...this.nuevo, id, verDetalle:false
      })
      this.nuevo = { nombre:'', precio:0, stock:0 }
    }
  }
}

Componentes Dinámicos · keep-alive

<component :is=""> renderiza el componente que corresponde al valor de la variable. Perfecto para tabs, wizards y layouts condicionales.

<component :is="vistaActual"></component>

// En data:
vistaActual: 'TabInicio'

// Cambiar de vista:
this.vistaActual = 'TabConfig'

keep-alive — preservar estado

<keep-alive>
  <component :is="vistaActual" />
</keep-alive>

<!-- Sin keep-alive: el componente se destruye
     al cambiar de vista — se pierde el estado.
     Con keep-alive: el componente se "pausa"
     y su estado se conserva. -->

Slots — contenido personalizable

<!-- Componente hijo Tarjeta.vue -->
<template>
  <div class="tarjeta">
    <header>
      <slot name="cabecera">
        Título por defecto
      </slot>
    </header>
    <main>
      <slot>Contenido por defecto</slot>
    </main>
    <footer>
      <slot name="pie"></slot>
    </footer>
  </div>
</template>

<!-- Uso desde el padre -->
<Tarjeta>
  <template v-slot:cabecera>
    <h2>🎉 ¡Nuevo producto!</h2>
  </template>
  <p>Este es el contenido del cuerpo</p>
  <template #pie>
    <button>Comprar</button>
  </template>
</Tarjeta>

Modificadores Avanzados

Teclado

<!-- Teclas específicas -->
<input @keyup.enter="enviar" />
<input @keyup.esc="cancelar" />
<input @keyup.space="accion" />
<input @keydown.tab="navegar" />

<!-- Combinaciones -->
<div @keyup.ctrl.enter="guardar" />

Ratón

<!-- Botones del mouse -->
<div @click.right="contextMenu">
  Click derecho
</div>

<div @click.middle="abrirTab">
  Rueda
</div>

<!-- Exacto (sin otras teclas) -->
<div @click.exact="solo">
  Solo este click
</div>

v-model

<!-- .lazy: on:change en vez de on:input -->
<input v-model.lazy="busqueda" />

<!-- .number: auto-convierte a Number -->
<input v-model.number="precio"
       type="number" />

<!-- .trim: elimina espacios -->
<input v-model.trim="nombre" />

<!-- combinados -->
<button
  @click.stop.prevent="accion"
>Detener todo</button>

💡 Los modificadores se encadenan en orden: @click.stop.prevent primero detiene la propagación, luego previene el default. El orden importa.

Clases y Estilos Dinámicos

:class — OBJETO

<!-- La clase se aplica si el valor es truthy -->
<div :class="{ activo: esActivo, error: tieneError }">

<!-- Combinado con clase estática -->
<div class="btn" :class="{ 'btn-primary': primario }">

<!-- O con array -->
<div :class="[claseBase, condicion ? 'a' : 'b']">

:style — OBJETO

<!-- camelCase para propiedades CSS -->
<div :style="{ color: colorTexto, fontSize: size + 'px' }">

<!-- Array de objetos de estilo -->
<div :style="[estiloBase, { color: colorActivo }]">

// En data:
estiloBase: { padding: '1rem', borderRadius: '8px' }

Atributos dinámicos con v-bind

<!-- Atributos individuales -->
<img :src="imagen" :alt="descripcion" />
<a :href="url" :target="target">Link</a>

<!-- Múltiples atributos de un objeto -->
<div v-bind="todosLosAtributos">

// En data:
todosLosAtributos: {
  id: 'mi-id',
  class: 'mi-clase',
  tabindex: 0
}

Usar v-bind sin argumento para pasar un objeto completo de atributos es muy útil al construir componentes genéricos de alto orden.
