Bloque 1 · Definición

¿Qué es una Base de Datos?

📋 Definición

Colección organizada de información que se almacena y gestiona de manera estructurada para permitir su fácil acceso, manipulación y recuperación.

Sistema que permite almacenar datos eficientemente y acceder a ellos de manera rápida y precisa.

💡 Analogía

Como un armario de archivos perfectamente organizado: cada cajón tiene su categoría, cada carpeta tiene su etiqueta y cada documento ocupa exactamente su lugar.

La diferencia: una base de datos puede manejar millones de registros y encontrar cualquiera en milisegundos.

Sin bases de datos no existirían: redes sociales, bancos, e-commerce, hospitales, ni ninguna aplicación moderna.

Bloque 1 · BD Relacional

¿Qué es una Base de Datos Relacional?

🏗️ Estructura

  • Datos organizados en tablas
  • Cada fila = un registro único (tupla)
  • Cada columna = un atributo o campo
  • Las tablas se relacionan entre sí mediante claves

🔗 Ejemplo de relaciones

  • Clientes → Facturas por ID de cliente
  • Facturas → Ventas por ID de factura
  • Ventas → Productos por ID de producto
  • Productos → Proveedores por ID de proveedor
El modelo relacional = tablas + atributos + tuplas + relaciones. La tabla es el concepto central.
Bloque 1 · Estructura

Tablas · Atributos · Tuplas

📋

Tabla

Estructura que representa una entidad. Conjunto de columnas y filas. Ejemplo: tabla Clientes.

🏷️

Atributo (columna)

Hecho simple que describe o caracteriza una entidad. Ejemplo: nombre, email, fecha_nacimiento.

📝

Tupla (fila)

Conjunto de valores que forman una instancia de la entidad. Ejemplo: los datos de un cliente específico.

-- Tabla Clientes: cada fila es una tupla, cada columna es un atributo
| id_cliente | nombre | email | ciudad |
|------------|------------|--------------------|------------|
| 1 | Ana López | ana@email.com | Santiago |
| 2 | Luis Pérez | luis@email.com | Valparaíso |
Bloque 1 · PK y FK

Clave Primaria y Clave Foránea

🔑 Clave Primaria (PK)

Atributo o conjunto de atributos que identifica de forma única cada registro en una tabla. No puede repetirse ni ser nulo.

| id_cliente (PK) | nombre | email |
| 1 | Ana López | ana@email.com |
| 2 | Luis P. | luis@email.com |

🔗 Clave Foránea (FK)

Campo en una tabla que referencia la PK de otra tabla. Establece la relación entre tablas y garantiza la integridad referencial.

| id_factura (PK) | id_cliente (FK) | total |
| 101 | 1 | $50.00 |
| 102 | 2 | $80.00 |
La FK en Facturas apunta a la PK de Clientes. Esto asegura que no exista una factura de un cliente que no existe.
Bloque 1 · SQL

SQL — El Lenguaje de las BD Relacionales

📖 ¿Qué es SQL?

Structured Query Language. Lenguaje estándar diseñado para trabajar con bases de datos relacionales.

  • Insertar datos nuevos
  • Consultar y filtrar información
  • Actualizar registros existentes
  • Eliminar datos
  • Crear y modificar estructuras

💻 Primeros comandos

-- Consultar todos los clientes
SELECT * FROM clientes;

-- Insertar un cliente nuevo
INSERT INTO clientes (nombre, email)
VALUES ('Ana López', 'ana@email.com');

-- Filtrar por ciudad
SELECT * FROM clientes
WHERE ciudad = 'Santiago';

Bloque 3 · RDBMS

¿Qué es un RDBMS?

⚙️ Definición

Relational Database Management System. Software diseñado para gestionar y organizar datos en estructura tabular.

Permite crear, modificar, almacenar, proteger y recuperar datos de una base de datos de forma eficiente.

✅ Características clave

  • Cada tabla tiene un nombre único
  • Cada fila en una tabla es única
  • Todas las tablas tienen una clave (ID) para identificar filas
  • Cada columna se llama campo
  • Aplica integridad y normalización
Bloque 3 · Motores RDBMS

Los 4 Motores Principales

🪟 SQL Server

  • Ecosistema Microsoft
  • Integración con Azure, Power BI, Visual Studio
  • Herramienta: SSMS
  • Licencia comercial

🏛️ Oracle DB

  • Más robusto del mercado
  • Misión crítica
  • Alta escalabilidad y fiabilidad
  • Usado en banca y telecomunicaciones

🐬 MySQL

  • Más popular en la web
  • Rápido y ligero
  • Open source (GPL)
  • Integra con PHP y Python

🐘 PostgreSQL

  • Avanzado y extensible
  • Soporta JSON y datos complejos
  • Open source (MIT)
  • Integridad referencial avanzada

Bloque 3 · Objetos

Objetos Principales de una Base de Datos

📋

Tablas

Objeto central. Colección de datos en filas y columnas. Cada tabla representa una entidad.

🔍

Vistas (Views)

Consultas almacenadas que generan resultados dinámicos. Ocultan complejidad o datos sensibles.

📑

Índices

Estructuras auxiliares que aceleran las búsquedas. Como el índice de un libro — van directo a la página.

⚙️

Procedimientos Almacenados

Scripts guardados en la BD con lógica de negocio compleja. Reutilizables y eficientes.

Bloque 3 · Objetos (cont.)

Más Objetos de una BD

⚡

Disparadores (Triggers)

Se ejecutan automáticamente ante eventos: INSERT, UPDATE, DELETE. Como alarmas que reaccionan a cambios.

🔑

Claves Primarias y Foráneas

PK identifica registros únicos. FK vincula tablas y garantiza integridad referencial.

📁

Esquemas

Contenedores lógicos que agrupan objetos relacionados. Como carpetas que organizan tablas, vistas, etc.

🔢

Secuencias / Auto-increment

Generan valores únicos automáticamente para las claves primarias. Se incrementan solos con cada INSERT.

Ejercicio: BD
Take Away

Un cliente necesita organizar los datos de su emprendimiento gastronómico:
✅ Crear tabla Clientes con: id (PK), nombre, email, telefono
✅ Crear tabla Productos con: id (PK), nombre, precio, categoria
✅ Crear tabla Ordenes con: id (PK), id_cliente (FK), id_producto (FK), cantidad, fecha
✅ Insertar al menos 5 registros en cada tabla
⭐ Extra: consultar las órdenes mostrando nombre del cliente y producto

Referencia · SQL Take Away

Estructura Base del Ejercicio

-- Crear tabla Clientes
CREATE TABLE Clientes (
  id_cliente INTEGER PRIMARY KEY AUTOINCREMENT,
  nombre TEXT NOT NULL,
  email TEXT UNIQUE,
  telefono TEXT
);

-- Crear tabla Productos
CREATE TABLE Productos (
  id_producto INTEGER PRIMARY KEY AUTOINCREMENT,
  nombre TEXT NOT NULL,
  precio REAL NOT NULL,
  categoria TEXT
);

-- Crear tabla Ordenes (con FK)
CREATE TABLE Ordenes (
  id_orden INTEGER PRIMARY KEY AUTOINCREMENT,
  id_cliente INTEGER REFERENCES Clientes(id_cliente),
  id_producto INTEGER REFERENCES Productos(id_producto),
  cantidad INTEGER DEFAULT 1,
  fecha TEXT DEFAULT (date('now'))
);
