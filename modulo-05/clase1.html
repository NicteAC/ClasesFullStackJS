🌟 Introducción a la POO

La POO es un paradigma de programación basado en organizar el código en objetos

¿Por qué es importante?

  • Permite estructurar mejor los programas
  • Facilita la reutilización del código
  • Hace el mantenimiento más sencillo
  • Modela el software como objetos del mundo real

Se usa en: Desarrollo web, aplicaciones de escritorio, videojuegos

🔀 Paradigmas de Programación

Imperativa

Instrucciones paso a paso de cómo realizar una tarea

Ejemplo: C, Pascal

Declarativa

Describe qué se quiere hacer, no cómo hacerlo

Ejemplo: SQL, Haskell

Funcional

Basada en funciones matemáticas sin efectos secundarios

Ejemplo: Lisp, Scala

Orientada a Objetos

Modela entidades del mundo real mediante clases y objetos

Ejemplo: Java, Python, JavaScript

🏛️ Los 4 Pilares de la POO

🔒 Encapsulamiento

Protege los datos de acceso indebido

🧬 Herencia

Una clase deriva propiedades y métodos de otra

Oculta complejidad interna del objeto

Muestra solo lo esencial

🔒 Encapsulamiento

Controla el acceso a las propiedades de un objeto

¿Qué es?

  • Oculta detalles internos de un objeto
  • Expone solo lo necesario
  • Protege la integridad de los datos
  • Evita modificaciones indebidas

Ejemplo en la vida real

Clase: Automóvil

Encapsulamiento: No accedemos directamente al motor, solo usamos el volante y pedales

En JavaScript: variables privadas (#) y métodos get/set

🧬 Herencia

Una clase puede heredar propiedades y métodos de otra

Conceptos

  • Clase Padre: Define atributos y métodos base
  • Clase Hija: Hereda de la clase padre
  • Puede agregar nuevas funcionalidades
  • Puede modificar comportamientos heredados

Ejemplo en la vida real

Clase Padre: Vehículo (marca, arrancar())

Clase Hija: Coche (hereda marca y arrancar(), añade puertas)

En JavaScript: palabra clave extends

🎭 Polimorfismo

Permite redefinir métodos en clases hijas sin cambiar su interfaz

¿Qué es?

  • Mismo nombre de método, diferente implementación
  • Facilita la extensibilidad del código
  • Cada clase puede tener su propia versión del método

Ejemplo en la vida real

Un pintor, un dibujante y un escritor pueden tener un método crear()

Cada uno lo ejecuta de manera diferente, pero todos "crean" algo

🎨 Abstracción

Oculta detalles de implementación y muestra solo lo necesario

¿Qué es?

  • Se centra en lo esencial
  • Oculta la complejidad interna
  • Se logra con clases y métodos abstractos

Ejemplo en la vida real

Automóvil:

Sabemos usar el volante y los pedales

No necesitamos conocer el funcionamiento interno del motor

📦 Objetos Literales

Características

  • Forma más simple de definir un objeto
  • Se crean con { } y pares clave-valor
  • Fácil creación y acceso
  • Pueden incluir métodos

Limitaciones

  • No permiten reutilización eficiente
  • Cada objeto debe definirse por separado
  • No hay plantilla reutilizable

Acceso: obj.propiedad o obj["propiedad"]

🔧 Funciones Constructoras

Antes de ES6, se usaban para crear múltiples objetos con la misma estructura

Características

  • Se define una función que actúa como plantilla
  • Se usa this para asignar propiedades
  • Se usa new para instanciar objetos
  • Permite crear múltiples instancias

✨ Clases (ES6)

Las clases ofrecen una forma más estructurada y legible de definir objetos

Ventajas

  • Más legibles y organizadas
  • Sintaxis clara y moderna
  • Métodos definidos directamente
  • Facilitan la herencia con extends

Elementos clave

  • class - Define la clase
  • constructor() - Inicializa propiedades
  • extends - Herencia
  • super() - Llama al constructor padre

🔗 Uso del Prototipo (prototype)

El prototype permite a los objetos heredar propiedades y métodos de otros objetos

¿Qué es prototype?

  • Mecanismo de herencia en JavaScript
  • Antes de ES6, era la principal forma de aplicar herencia
  • Todos los objetos en JS tienen un prototipo
  • Se pueden extender con nuevos métodos

Las clases de ES6 usan prototype internamente de forma automática
