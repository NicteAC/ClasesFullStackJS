🌳 ¿Qué es el DOM?

El Document Object Model (DOM) es una interfaz de programación que representa la estructura de un documento HTML o XML en forma de árbol jerárquico

¿Para qué sirve?

  • Permite a JavaScript acceder y modificar la estructura de una página
  • Modificar contenido sin recargar la página
  • Agregar o eliminar elementos dinámicamente
  • Cambiar estilos CSS en respuesta a eventos
  • Escuchar eventos del usuario (clics, input, scroll, etc.)

📊 Jerarquía de Elementos del DOM

El DOM sigue una estructura jerárquica en forma de árbol

Tipos de nodos:

  • Nodo raíz: document
  • Nodos de elementos: Etiquetas HTML
  • Nodos de texto: Contenido de texto
  • Nodos de atributos: class, id, src, etc.

Relaciones:

  • Padre (parent): Contiene otros nodos
  • Hijos (children): Elementos dentro del padre
  • Hermanos (siblings): Comparten mismo padre

🎯 Métodos de Selección de Elementos

getElementById()

Selecciona UN elemento por ID

document.getElementById("titulo")

getElementsByClassName()

Selecciona MÚLTIPLES por clase

document.getElementsByClassName("item")

getElementsByTagName()

Selecciona por etiqueta HTML

document.getElementsByTagName("p")

querySelector()

Primer elemento con selector CSS

document.querySelector(".item")

querySelectorAll()

TODOS los elementos con selector CSS (más versátil)

document.querySelectorAll(".item")

✏️ Modificación de Contenido

textContent

Reemplaza solo el texto

✅ Seguro (no ejecuta HTML)

element.textContent = "Nuevo texto"

innerHTML

Permite incluir etiquetas HTML

⚠️ Riesgo XSS si no se valida

element.innerHTML = "<strong>Texto</strong>"

Recomendación: Usa textContent cuando no necesites HTML, usa innerHTML solo con datos seguros

🔧 Modificación de Atributos

Directo

element.src = "imagen.jpg"

element.href = "url.com"

✅ Rápido y simple

setAttribute()

element.setAttribute("href", "url")

✅ Funciona con atributos personalizados

getAttribute()

element.getAttribute("href")

✅ Lee cualquier atributo

Atributos data-* personalizados

element.dataset.id = "123"

console.log(element.dataset.estado)

➕ Agregar Elementos al DOM

Proceso en 3 pasos:

  • 1. Crear: document.createElement("p")
  • 2. Configurar: element.textContent = "Texto"
  • 3. Agregar: parent.appendChild(element)

appendChild()

Agrega al final del contenedor

parent.appendChild(newElement)

insertBefore()

Inserta antes de un elemento

parent.insertBefore(new, ref)

🗑️ Eliminar Elementos del DOM

remove()

Elimina el elemento directamente

element.remove()

✅ Moderno y simple

⚠️ No funciona en IE

removeChild()

Elimina desde el padre

parent.removeChild(child)

✅ Compatible con navegadores antiguos

🎨 Modificar Estilos CSS

Opción 1: Propiedad style

element.style.color = "red"

element.style.fontSize = "20px"

element.style.backgroundColor = "blue"

✅ Cambios específicos inmediatos

⚠️ Difícil de mantener con muchos estilos

Opción 2: classList (Recomendado)

element.classList.add("clase")

element.classList.remove("clase")

element.classList.toggle("activo")

✅ Mantiene separación estructura/estilo

✅ Más fácil de mantener

⚡ ¿Qué son los Eventos?

Los eventos son acciones o sucesos que ocurren en el navegador y que pueden ser detectados y manejados mediante código JavaScript

Permiten:

  • Responder a interacciones del usuario
  • Crear páginas web dinámicas e interactivas
  • Ejecutar código en respuesta a acciones específicas
  • Mejorar la experiencia del usuario

🔄 Modelo de Eventos

Event Bubbling (Burbujeo)

El evento se propaga desde el elemento objetivo hacia sus padres

Button → Div → Body → Document

↑ ↑ ↑

Comportamiento por defecto

Event Capturing (Captura)

El evento se propaga desde la raíz hacia el objetivo

Document → Body → Div → Button

↓ ↓ ↓

Se activa con addEventListener(event, handler, true)

event.stopPropagation() - Detiene la propagación del evento

🎮 Formas de Manejar Eventos

1. Atributos HTML

<button onclick="fn()">

❌ No recomendado

Mezcla HTML y JS

2. Propiedades DOM

element.onclick = fn

⚠️ Solo un handler

Sobrescribe anteriores

3. addEventListener

element.addEventListener("click", fn)

✅ Recomendado

Múltiples handlers, mejor control

🎯 Delegación de Eventos

Técnica donde un solo event listener en un elemento padre maneja eventos de sus hijos

Ventajas:

  • ✅ Reduce el uso de memoria
  • ✅ Mejora el rendimiento
  • ✅ Funciona con elementos agregados dinámicamente
  • ✅ Código más limpio y mantenible

En lugar de: Agregar listener a cada <li>

Mejor: Agregar listener al <ul> padre
