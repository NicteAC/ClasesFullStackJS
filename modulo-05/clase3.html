AE 3.1 · Bloque 1

Data Manipulation
Language (DML)

Parte de SQL que permite manipular los datos: insertar, actualizar y eliminar registros dentro de las tablas.

Los 4 comandos DML

➕

INSERT

Agrega nuevos registros (filas) a una tabla

✏️

UPDATE

Modifica valores existentes en una o varias filas

🗑️

DELETE

Elimina filas de una tabla

🔍

SELECT

Recupera datos (también es parte de DML)

⚡ AutoCommit

Característica que determina si las operaciones DML se confirman automáticamente o requieren un COMMIT manual.

✅ AutoCommit = ON

Cada instrucción se guarda de inmediato en la base de datos.

⚠️ Difícil de revertir cambios.

🔒 AutoCommit = OFF

Los cambios quedan pendientes hasta ejecutar COMMIT o ROLLBACK.

✅ Útil para transacciones multi-paso.

Bloque 2 · INSERT

Agregar información
en una tabla

INSERT INTO nombre_tabla (columna1, columna2, columna3) VALUES (valor1, valor2, valor3);

Ejemplo INSERT

-- Un solo registro INSERT INTO productos (nombre, precio, cantidad) VALUES ('Laptop HP', 900000, 15); -- Varios registros a la vez INSERT INTO productos (nombre, precio, cantidad) VALUES ('Mouse Logitech', 25000, 50), ('Teclado Redragon', 45000, 30);

🔢 ID Autogenerado

Cada vez que insertamos un registro, el ID se genera automáticamente aumentando de uno en uno.

MySQL / MariaDB

id INT AUTO_INCREMENT PRIMARY KEY

SQL Server

id INT IDENTITY(1,1) PRIMARY KEY

PostgreSQL

id SERIAL PRIMARY KEY

SQLite

id INTEGER PRIMARY KEY AUTOINCREMENT

Bloque 2 · DELETE

Borrar información
de una tabla

DELETE FROM nombre_tabla WHERE condicion;

⚠️ ¡CUIDADO! Si omites la cláusula WHERE, se eliminarán TODAS las filas de la tabla.

Ejemplo DELETE

-- Eliminar un producto específico DELETE FROM productos WHERE nombre = 'Laptop Dell'; -- Eliminar clientes inactivos (más de 12 meses sin compras) DELETE FROM clientes WHERE ultima_compra IS NULL OR ultima_compra < DATE_SUB(NOW(), INTERVAL 12 MONTH);

💡 Antes de un DELETE, siempre ejecuta el SELECT equivalente para verificar qué filas afectará.

Bloque 2 · UPDATE

Actualizar información
en una tabla

UPDATE nombre_tabla SET columna1 = valor1, columna2 = valor2 WHERE condicion;

⚠️ Igual que DELETE: si omites WHERE, se actualizan TODAS las filas.

Ejemplo UPDATE

-- Aumentar 10% a empleados con más de 5 años UPDATE empleados SET salario = salario * 1.10 WHERE anios_experiencia > 5; -- Actualizar múltiples columnas de una vez UPDATE productos SET precio = 1200, cantidad = 30, actualizado_en = NOW() WHERE id_producto = 5;

AE 3.2 · Bloque 3

Integridad referencial

Concepto que garantiza que las relaciones entre tablas se mantengan consistentes. Los valores en una FK deben coincidir con valores existentes en la PK de la tabla relacionada.

¿Para qué sirve?

🎯

Coherencia

Evita relaciones incorrectas entre tablas. Datos siempre precisos.

🛡️

Prevención

Bloquea eliminaciones o updates que rompan la coherencia.

🏗️

Estructura

Define cómo se relacionan las tablas de manera clara y automática.

Insertar con integridad referencial

⚡ Regla de oro: primero el padre, luego el hijo

-- Paso 1: primero insertar el Cliente (tabla principal) INSERT INTO Clientes (nombre) VALUES ('Sofía Ruiz'); -- Supongamos que MySQL le asigna id_cliente = 3 -- Paso 2: luego insertar el Pedido con el id_cliente EXISTENTE INSERT INTO Pedidos (id_cliente, fecha) VALUES (3, '2024-05-20');

Si intentas insertar un pedido con id_cliente = 999 y no existe, MySQL lanza error.

🗑️ Eliminar con integridad referencial

Regla inversa: primero el hijo, luego el padre

-- Paso 1: eliminar los libros del autor primero DELETE FROM Libros WHERE id_autor = 1; -- Paso 2: ahora sí eliminar el autor DELETE FROM Autores WHERE id_autor = 1;

Si intentas eliminar el autor primero, MySQL rechaza la operación con un error de FK.

⚡ ON DELETE CASCADE

Al eliminar registros en la tabla principal, las filas relacionadas se eliminan automáticamente.

CREATE TABLE Libros ( id_libro INT PRIMARY KEY, titulo VARCHAR(100), id_autor INT, FOREIGN KEY (id_autor) REFERENCES Autores(id_autor) ON DELETE CASCADE ); -- Ahora eliminar el autor borra sus libros automáticamente DELETE FROM Autores WHERE id_autor = 1;

Bloque 4

Principios ACID

Conjunto de propiedades que garantizan la integridad y coherencia de los datos en las transacciones.

A C I D
A

Atomicidad

La transacción es una unidad indivisible. Se ejecuta todo o nada.

Si alguna operación falla, todas las anteriores se deshacen (rollback). Ejemplo: si retirar $100 falla, no se deposita nada en la otra cuenta.

C

Consistencia

Las transacciones llevan la base de datos de un estado válido a otro estado válido.

Se respetan todas las reglas de integridad, constraints y restricciones. Si algo las viola, la transacción se cancela.

I

Aislamiento

Las transacciones en ejecución son independientes, no se afectan entre sí.

Evita problemas como lecturas sucias, lecturas no repetibles y escrituras fantasma cuando muchos usuarios operan al mismo tiempo.

D

Durabilidad

Una vez confirmada la transacción, los cambios persisten para siempre.

Incluso si el servidor se apaga o hay un fallo eléctrico después del COMMIT, los datos siguen allí al reiniciar.

Bloque 5

Transaccionalidad
en operaciones SQL

Capacidad de agrupar varias operaciones en una unidad lógica indivisible llamada transacción.

Flujo de una transacción

1
START TRANSACTION → Inicia la transacción
2
Operaciones DML → INSERT, UPDATE, DELETE, SELECT
3
COMMIT → Confirma los cambios (los hace permanentes)
4
ROLLBACK → Si algo falla, revierte todo al estado inicial

Sintaxis de una transacción

START TRANSACTION; -- Operaciones DML UPDATE Cuentas SET Saldo = Saldo - 200 WHERE ID = 1; UPDATE Cuentas SET Saldo = Saldo + 200 WHERE ID = 2; -- Si todo salió bien, confirmar COMMIT; -- Si hubo error, deshacer todo ROLLBACK;
