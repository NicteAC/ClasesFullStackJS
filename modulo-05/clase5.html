🔨 Construcción de una Promesa

Usamos el constructor new Promise() con dos parámetros: resolve y reject

new Promise((resolve, reject) => {
  // Código asíncrono aquí
  if (exitoso) {
    resolve(resultado);
  } else {
    reject(error);
  }
});

Parámetros:

  • resolve: Función para indicar éxito
  • reject: Función para indicar error

✅ Resolviendo y Rechazando Promesas

resolve()

Se llama cuando la operación es exitosa

resolve("¡Éxito!");
resolve({data: valores});
resolve([items]);

Cambia el estado a Fulfilled

reject()

Se llama cuando la operación falla

reject("Error!");
reject(new Error("Falló"));
reject({code: 404});

Cambia el estado a Rejected

⛓️ .then() y .catch()

.then() se ejecuta si la promesa se resuelve correctamente

.catch() captura errores si la promesa es rechazada

promesa
  .then(resultado => {
    console.log("Éxito:", resultado);
  return procesarResultado(resultado);
  })
  .then(procesado => {
    console.log("Procesado:", procesado);
  })
  .catch(error => {
    console.error("Error:", error);
  });

✅ Permite encadenar múltiples operaciones

✅ Un solo .catch() maneja errores de toda la cadena

🎯 Async/Await

async/await es una forma más clara y estructurada de trabajar con código asíncrono

Con .then()/.catch()

obtenerDatos()
  .then(datos => {
    return procesar(datos);
  })
  .then(resultado => {
    console.log(resultado);
  })
  .catch(error => {
    console.error(error);
  });

Con async/await

async function obtener() {
  try {
    const datos = await obtenerDatos();
    const resultado = await procesar(datos);
    console.log(resultado);
  } catch (error) {
    console.error(error);
  }
}

⏸️ Uso de await

await detiene la ejecución de la función hasta que la promesa se resuelva

Ventajas:

  • Hace que el código se vea más estructurado
  • Más fácil de leer y mantener
  • Debugging más sencillo
  • Manejo de errores con try/catch familiar
async function procesarDatos() {
  const usuario = await obtenerUsuario();
  const posts = await obtenerPosts(usuario.id);
  const comentarios = await obtenerComentarios(posts[0].id);
  return comentarios;
}

🛡️ Manejo de Errores con try/catch

Evita tener que encadenar .catch(), haciendo el código más limpio

async function obtenerDatos() {
  try {
    const respuesta = await fetch(url);
    const datos = await respuesta.json();
    return datos;
  } catch (error) {
    console.error("Error al obtener datos:", error);
    throw error; // Re-lanzar si es necesario
  }
}

Ventajas:

  • Sintaxis familiar para manejar errores
  • Captura errores de múltiples await en un solo bloque
  • Más limpio que múltiples .catch()

🔀 Promise.all() - Múltiples Promesas

Promise.all() ejecuta múltiples promesas en paralelo y espera a que todas terminen

const promesa1 = obtenerUsuario();
const promesa2 = obtenerPosts();
const promesa3 = obtenerComentarios();

Promise.all([promesa1, promesa2, promesa3])
  .then(([usuario, posts, comentarios]) => {
    console.log("Todo cargado:", {usuario, posts, comentarios});
  })
  .catch(error => {
    console.error("Alguna falló:", error);
  });

⚠️ Si UNA promesa falla, Promise.all() rechaza toda la operación
