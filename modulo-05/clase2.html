de Consultas (SQL)

Structured Query Language

Lenguaje de programación utilizado para gestionar y manipular bases de datos relacionales. Desarrollado originalmente por IBM en los años 70, hoy es el estándar de la industria.

Las 4 familias de sentencias SQL

🏗️ DDL (Data Definition Language)

Crear y modificar objetos de la base de datos.

CREATE, ALTER, DROP

📝 DML (Data Manipulation Language)

Consultar y manipular los datos.

SELECT, INSERT, UPDATE, DELETE

🔐 DCL (Data Control Language)

Controlar el acceso a los objetos.

GRANT, REVOKE

💼 TCL (Transaction Control Language)

Gestionar transacciones.

COMMIT, ROLLBACK, SAVEPOINT

Recuperar información
de una tabla

Solicitar y obtener información específica almacenada en una base de datos relacional mediante la ejecución de una consulta SQL.

Proceso para recuperar datos

1. Escribir la consulta

Redactar SQL que especifique qué datos deseas obtener.

2. Cláusula SELECT

Define las columnas específicas que deseas recuperar.

3. Especificar tabla (FROM)

Indica la tabla de la cual recuperar datos.

4. Filtrar y ordenar

Usa WHERE para filtrar y ORDER BY para ordenar.

Cláusulas SQL principales

SELECT nombre, precio FROM productos WHERE precio > 100 ORDER BY precio DESC LIMIT 10;
  • SELECT → columnas a recuperar
  • FROM → tabla origen
  • WHERE → filtro de registros
  • ORDER BY → orden ASC / DESC
  • LIMIT → cantidad máxima de filas
  • EXISTS → verifica existencia en subconsulta

    ¿Qué es un modelo de datos?

    La estructura y organización de los datos en una base de datos. Define cómo se almacenan, acceden y relacionan los datos.

    Modelos de datos más comunes

    🔗

    Modelo Relacional

    Tablas con filas y columnas. Relaciones vía claves primarias y foráneas. (El más usado en SQL)

    🕸️

    Entidad-Relación (ER)

    Entidades representan objetos del mundo real. Relaciones conectan esas entidades.

    📄

    Orientado a Documentos

    Datos en formatos como JSON o XML. Mayor flexibilidad estructural.

    🔑 Clave primaria (PK)

    Campo o conjunto de campos que identifica de manera única cada registro dentro de una tabla.

    Unicidad

    Cada valor debe ser único en toda la tabla.

    No nulidad

    Los valores no pueden ser NULL.

    Identificación

    Identifica cada registro de forma inequívoca.

    CREATE TABLE clientes ( id_cliente INT PRIMARY KEY, nombre VARCHAR(100) );

    🔗 Clave externa (FK)

    Campo que establece una relación entre dos tablas. Referencia a la clave primaria de otra tabla.

    CREATE TABLE ordenes ( id_orden INT PRIMARY KEY, fecha DATE, id_cliente INT, FOREIGN KEY (id_cliente) REFERENCES clientes(id_cliente) );

    Las FK garantizan la integridad referencial y permiten realizar uniones entre tablas.

    Consultas con PK y FK

    Usando PK

    Obtener info de un cliente específico:

    SELECT * FROM clientes WHERE id_cliente = 5;

    Usando FK

    Obtener órdenes de ese cliente:

    SELECT * FROM ordenes WHERE id_cliente = 5;

    Consultas anidadas
    (Subqueries)

    Consultas que se encuentran dentro de otra consulta principal. La subconsulta se ejecuta primero, y su resultado se usa en la consulta externa.

    Anatomía de una subquery

    -- Consulta principal (externa) SELECT nombre FROM empleados WHERE salario > ( -- Subconsulta (interna): se ejecuta PRIMERO SELECT AVG(salario) FROM empleados );

    🔹 La subconsulta calcula el promedio de todos los salarios.

    🔹 La consulta principal devuelve empleados con salario mayor a ese promedio.

    ¿Dónde pueden ir las subconsultas?

    SELECT

    Como valor calculado dentro de una columna.

    FROM

    Como tabla derivada (temporal).

    WHERE

    Como valor de comparación en el filtro.

    HAVING

    Como filtro después de un GROUP BY.

    IN / NOT IN

    Como lista de valores permitidos.

    EXISTS

    Para verificar si retornó filas.

    Ejemplo práctico

    Estudiantes inscritos en "Matemáticas Avanzadas"

    SELECT nombre FROM estudiantes WHERE id_estudiante IN ( SELECT id_estudiante FROM inscripciones WHERE id_curso = ( SELECT id_curso FROM cursos WHERE nombre = 'Matemáticas Avanzadas' ) );

    Consultas con funciones
    de agregación + GROUP BY

    SELECT categoria, COUNT(*) AS total, AVG(precio) AS promedio FROM productos GROUP BY categoria HAVING COUNT(*) > 5;

    GROUP BY agrupa registros · HAVING filtra grupos (no filas)

    ¿Qué son las operaciones JOIN?

    Combinan datos de dos o más tablas en función de una relación entre ellas. Permiten unir información dividida en distintos conjuntos.

    INNER JOIN

    Solo registros que coinciden en ambas tablas.

    LEFT JOIN

    Todos los de la izquierda + coincidencias derecha.

    RIGHT JOIN

    Todos los de la derecha + coincidencias izquierda.

    FULL JOIN

    Todos los registros de ambas tablas.

    Ejemplo: INNER JOIN

    SELECT c.nombre, p.numero_pedido FROM clientes c INNER JOIN pedidos p ON c.id_cliente = p.id_cliente;

    🔹 Combina las tablas clientes y pedidos por id_cliente.

    🔹 El resultado: cada cliente con los números de pedido correspondientes.

    🔹 Los clientes sin pedidos NO aparecen.

    JOIN + Agregación + HAVING

    Autores con más de 3 libros publicados desde 2020

    SELECT a.nombre, a.apellido, COUNT(*) AS total_libros FROM autores a INNER JOIN libros l ON a.id_autor = l.id_autor WHERE l.anio_publicacion >= 2020 GROUP BY a.id_autor, a.nombre, a.apellido HAVING COUNT(*) > 3 ORDER BY total_libros DESC;
    </>
