✨ JavaScript ES6+: La Evolución del Lenguaje

ECMAScript 6 (ES6), también conocido como ECMAScript 2015, marcó un antes y después en JavaScript

Principales mejoras introducidas:

  • Variables con let y const
  • Funciones flecha (=>)
  • Template literals
  • Destructuring
  • Módulos (import/export)
  • Clases y herencia
  • Promesas
  • Spread y Rest operators

🌐 Compatibilidad en Navegadores

✅ Navegadores Modernos

  • Chrome, Firefox, Safari, Edge
  • Soportan la mayoría de ES6+
  • Actualizaciones constantes
  • let, const, arrow functions
  • Promises, async/await

⚠️ Navegadores Antiguos

  • Internet Explorer (sin soporte)
  • Versiones antiguas de otros
  • Dispositivos desactualizados
  • Necesitan transpilación
  • Requieren polyfills

🔧 Babel - Transpilador de JavaScript

Babel convierte código ES6+ a ES5 compatible con navegadores antiguos

¿Qué hace Babel?

  • Transpila código moderno
  • Garantiza compatibilidad
  • Permite usar últimas features
  • Integra con build tools

Ejemplo

ES6:

const saludar = (n) => `Hola ${n}`

ES5:

var saludar = function(n) { return "Hola " + n }

➡️ Arrow Functions

Sintaxis más corta y clara para funciones, sin su propio this

Ventajas

  • Sintaxis concisa
  • Return implícito en una línea
  • No tienen su propio this (heredan del contexto)
  • Ideales para callbacks y funciones cortas

Sintaxis: const sumar = (a, b) => a + b

💬 Template Literals

Facilitan la manipulación de strings con backticks ` y permite interpolación

Características

  • Interpolación con ${}
  • Strings multilínea
  • Más legible que concatenación
  • Permite expresiones

Ejemplo

const nombre = "Ana";

console.log(`Hola, ${nombre}!`);

📦 Destructuring

Extrae valores de objetos y arrays de forma rápida y elegante

Objetos

const persona = { nombre: "Juan", edad: 30 };

const { nombre, edad } = persona;

Arrays

const numeros = [1, 2, 3];

const [primero, segundo] = numeros;

... Spread y Rest Operators

Spread (...)

Expande elementos de un array u objeto

const arr = [1, 2, 3];

const nuevo = [...arr, 4, 5];

Rest (...)

Agrupa múltiples elementos en uno

function sumar(...nums) {

return nums.reduce((a,b) => a+b);

}

🎯 Parámetros por Defecto

Permiten valores predeterminados en funciones, evitando errores por parámetros undefined

Ejemplo

function saludar(nombre = "Invitado") {

console.log(`Hola, ${nombre}!`);

}

Si no se pasa argumento, usa "Invitado" por defecto

🏗️ Clases en ES6

Sintaxis clara para programación orientada a objetos

Características

  • constructor() para inicializar
  • Métodos dentro de la clase
  • extends para herencia
  • super() para clase padre

Ventajas

  • Más legible que prototypes
  • Sintaxis familiar de POO
  • Herencia simplificada
  • Código organizado

📦 Módulos: import y export

Permiten organizar y reutilizar código en archivos separados

Exportar

export const PI = 3.14;

export function suma(a, b) {

return a + b;

}

Importar

import { PI, suma } from './math.js';

console.log(suma(2, 3));

🗂️ Sets y Maps

Set

Colección de valores únicos

const set = new Set([1, 2, 3, 3]);

console.log(set); // {1, 2, 3}

Útil para eliminar duplicados

Map

Estructura de pares clave-valor

const map = new Map();

map.set("nombre", "Ana");

Más eficiente que objetos para búsquedas

🤝 Promesas

Representan una operación que puede completar con éxito (resolve) o fallar (reject) en el futuro

Características

  • Código más limpio que callbacks
  • Mejor manejo de errores
  • Encadenamiento con .then()
  • Captura de errores con .catch()

Estados: Pending → Fulfilled o Rejected

⏳ Async y Await

Sintaxis moderna (ES8) que simplifica el trabajo con Promesas

Ventajas

  • Código más simple y lineal
  • Parece código síncrono
  • Manejo de errores con try...catch
  • Más fácil de leer y mantener

Uso

async - Declara función asíncrona

await - Espera resultado de promesa

Compatible con fetch, APIs, timers, etc.
