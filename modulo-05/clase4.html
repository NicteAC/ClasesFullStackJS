🔄 ¿Qué es la Programación Asíncrona?

Es un modelo de programación donde las operaciones no se ejecutan secuencialmente, sino que pueden continuar sin bloquear el flujo del programa

Características:

  • Permite manejar tareas que requieren esperar
  • No bloquea la ejecución del código
  • Mejora la eficiencia y experiencia del usuario
  • Fundamental para aplicaciones web modernas

Casos de uso comunes:

Llamadas a APIs • Lectura de archivos • Temporizadores • Operaciones de base de datos

🧵 Hilos de Ejecución (Threads)

JavaScript es monohilo (single-threaded) - ejecuta una tarea a la vez en el Event Loop principal

Event Loop

Mecanismo que permite a JavaScript realizar operaciones asíncronas

  • Call Stack (pila de llamadas)
  • Task Queue (cola de tareas)
  • Microtask Queue (promesas)

¿Por qué es importante?

  • Evita bloquear la interfaz de usuario
  • Permite múltiples operaciones "simultáneas"
  • Optimiza el rendimiento

⚔️ Código Bloqueante vs No Bloqueante

⏳ Código Bloqueante (Síncrono)

Se ejecuta línea por línea

No continúa hasta terminar la tarea actual

console.log("Inicio");
for (let i = 0; i < 1000000000; i++) {}
console.log("Fin");

❌ Bloquea la ejecución

⚡ Código No Bloqueante (Asíncrono)

Permite que el código continúe

Ejecuta tareas mientras espera

console.log("Inicio");
setTimeout(() => {...}, 2000);
console.log("Fin");

✅ No bloquea

📞 Callbacks Asíncronos

Un callback es una función que se pasa como argumento a otra función para ejecutarse después de una operación

Características:

  • Primera forma de manejar asincronía en JavaScript
  • Se ejecuta cuando la operación asíncrona termina
  • Permite continuar la ejecución sin esperar

Casos de uso:

Event listeners • setTimeout/setInterval • Lectura de archivos • Peticiones HTTP

🔥 El Problema: Callback Hell

Cuando anidamos demasiados callbacks, el código se vuelve difícil de leer y mantener

Problemas del Callback Hell:

  • ❌ Código con forma de "pirámide" (anidación excesiva)
  • ❌ Difícil de leer y entender
  • ❌ Complicado de mantener y depurar
  • ❌ Manejo de errores tedioso

También conocido como "Pyramid of Doom" 🏔️

🤝 Promesas (Promises)

Una Promesa es un objeto que representa una operación que puede completarse o fallar en el futuro

⏳

Pending

Operación en curso

✅

Resolved

Operación exitosa

❌

Rejected

Operación fallida

✨ Ventajas de las Promesas

✅ Mejor Legibilidad

Menos anidamiento que callbacks

Código más plano y fácil de seguir

✅ Manejo de Errores

Manejo estructurado con .catch()

Un solo catch para toda la cadena

✅ Encadenamiento

Flujo de datos con .then()

Composición de operaciones asíncronas

✅ Estándar Moderno

Introducidas en ES6 (2015)

Base para async/await

🎯 Async/Await - Sintaxis Moderna

async/await es azúcar sintáctica sobre promesas que permite escribir código asíncrono como si fuera síncrono

async

  • Declara una función asíncrona
  • Siempre retorna una Promise
  • Permite usar await dentro

await

  • Pausa la ejecución hasta que la Promise se resuelva
  • Solo funciona dentro de funciones async
  • Retorna el valor resuelto

Ventajas:

✅ Código más legible • ✅ Manejo de errores con try/catch • ✅ Debugging más fácil
