🔌 ¿Qué es una API?

API (Application Programming Interface) es un conjunto de reglas que permite la comunicación entre diferentes sistemas o aplicaciones

¿Para qué sirve?

  • Facilita el acceso a funcionalidades o datos
  • No necesitas conocer los detalles internos
  • Conectar con bases de datos y servicios externos
  • Acceder a funcionalidades del navegador

Piensa en una API como un menú de restaurante: te dice qué puedes pedir, pero no cómo se cocina

📦 Tipos de APIs en JavaScript

🔹 APIs del Browser

Funciones integradas en navegadores

  • API DOM (Document Object Model)
  • API Fetch (solicitudes HTTP)
  • API Geolocation (ubicación)
  • API Web Storage (localStorage)
  • API Clipboard (portapapeles)
  • API Notification (notificaciones)

✅ No requieren instalación

🔹 APIs de Terceros

Servicios externos que proporcionan datos

  • OpenWeather (clima)
  • Google Maps (mapas)
  • Twitter API (tweets)
  • Stripe (pagos)
  • GitHub API (repositorios)
  • JSONPlaceholder (datos de prueba)

⚠️ Muchas requieren autenticación

⚡ ¿Qué es AJAX?

AJAX (Asynchronous JavaScript and XML) permite actualizar partes de una web sin recargar la página

Antes de AJAX:

  • Cada interacción recargaba toda la página
  • Experiencia lenta y molesta
  • Mayor consumo de datos
  • Pérdida del estado de la aplicación

Con AJAX:

  • Actualizaciones parciales de la página
  • Experiencia fluida y rápida
  • Menor consumo de datos
  • Aplicaciones web interactivas

📜 XMLHttpRequest (XHR)

Método tradicional para hacer peticiones HTTP en JavaScript

Características:

  • Eventos para respuestas
  • Soporte navegadores antiguos
  • Sintaxis verbosa
  • Usa callbacks

Ventajas y Desventajas:

✅ Compatible navegadores antiguos

❌ Difícil de leer

❌ Sin promesas nativas

Método usado desde 2005 - Hoy reemplazado por Fetch

🚀 Fetch API

Alternativa moderna y sencilla para hacer peticiones HTTP

Características:

  • Basada en Promesas
  • Sintaxis limpia
  • Compatible async/await
  • Soporte para streams

Ventajas y Desventajas:

✅ Código limpio y moderno

✅ Mejor manejo errores

❌ No funciona en IE

Estándar desde 2015 - Recomendado para código nuevo
