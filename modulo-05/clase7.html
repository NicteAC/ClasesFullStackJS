🌍 APIs de Terceros

Una API de terceros es un servicio externo desarrollado por otra empresa que permite integrar funcionalidades o datos en tu aplicación

Características:

  • Desarrolladas por empresas externas
  • Permiten acceder a datos específicos
  • Soluciones predefinidas y listas para usar
  • Ahorran tiempo y recursos de desarrollo
  • Actúan como intermediarias entre aplicaciones

💡 En lugar de construir todo desde cero, usas servicios ya existentes

🔑 Conectando con API Keys

La mayoría de APIs requieren una API Key (clave) única para autenticarse

1 Registrarse en el servicio de la API
2 Obtener la API Key en el panel de usuario/desarrollador
3 Usar la Key en las solicitudes para autenticarse

⚠️ Importante: Nunca compartas tu API Key públicamente

⚠️ No la subas a repositorios públicos (GitHub)

⏱️ Límites y Restricciones

Las APIs imponen límites para evitar abusos y mantener su servicio estable

Tipos de Restricciones:

  • Requests por tiempo (ej: 1000/hora)
  • Tamaño de datos (ej: max 10MB)
  • Rate limiting (bloqueo temporal)
  • Planes de pago (free vs premium)

Ejemplo: GitHub API

Sin autenticación:

60 requests/hora

Con OAuth:

5,000 requests/hora

📊 Códigos de Estado HTTP

200 OK

Solicitud exitosa

201 Created

Recurso creado (POST exitoso)

400 Bad Request

Error en formato de solicitud

401 Unauthorized

Falta o es inválida la autenticación

403 Forbidden

Sin permiso para acceder

404 Not Found

Recurso no existe

429 Too Many Requests

Límite de peticiones alcanzado

500 Server Error

Problema en el servidor

🛡️ Manejo de Errores en APIs

Cuando una API falla, es clave manejarlo correctamente para que la aplicación no se rompa

Estrategias:

  • Capturar respuestas erróneas
  • Mostrar mensajes al usuario
  • Reintentar en fallos temporales
  • Validar datos antes de enviar
  • Monitorear logs de errores

Buenas Prácticas:

  • Siempre usar try/catch
  • Verificar response.ok
  • Mensajes claros al usuario
  • Tener plan de contingencia
